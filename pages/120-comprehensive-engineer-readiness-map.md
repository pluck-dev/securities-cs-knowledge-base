# 120. 종합 개발자 역량 맵과 자가 진단

> 방대한 지식을 실제 성장 계획으로 바꾸려면 역할별 역량과 점검 기준이 필요합니다.

## 1. 역량 축

| 축 | 설명 |
|---|---|
| CS 기본기 | 자료구조, 알고리즘, OS, 네트워크 |
| 애플리케이션 | 백엔드, 프론트엔드, 모바일, API |
| 데이터 | DB, 캐시, 검색, 분석, 정합성 |
| 인프라 | Linux, Kubernetes, cloud, IaC, DR |
| 운영 | SRE, 관측성, 장애 대응, 성능 |
| 보안 | IAM, 암호학, threat modeling, 공급망 |
| 설계 | DDD, 분산 시스템, 모듈, 마이그레이션 |
| 협업 | Git, 리뷰, 문서, 요구사항, 리딩 |

## 2. Junior 기준

- 한 서비스의 기능을 안전하게 수정한다.
- 테스트와 빌드를 실행한다.
- API와 DB 기본을 이해한다.
- 로그로 단순 오류를 찾는다.
- 코드 리뷰 피드백을 반영한다.
- 모르는 것을 문서화하고 질문한다.

## 3. Mid-level 기준

- 기능 설계와 edge case를 스스로 정리한다.
- 트랜잭션/동시성/성능 문제를 고려한다.
- 운영 지표와 알림을 추가한다.
- 장애를 재현하고 원인을 좁힌다.
- 작은 마이그레이션을 안전하게 수행한다.
- 팀 표준을 따른다.

## 4. Senior 기준

- 시스템 경계와 trade-off를 설계한다.
- 장애 모드와 롤백을 먼저 생각한다.
- 팀의 기술 부채를 구조적으로 줄인다.
- 보안/개인정보/운영 리스크를 사전에 발견한다.
- 여러 팀과 API/데이터 계약을 조율한다.
- 후배가 성장할 수 있는 문서와 도구를 만든다.

## 5. Staff+ 기준

- 조직 전체의 아키텍처 방향을 제시한다.
- 플랫폼과 표준으로 반복 문제를 해결한다.
- 큰 마이그레이션을 단계적으로 이끈다.
- SLO, 비용, 보안, 생산성을 함께 최적화한다.
- 기술 결정의 장기 비용을 설명한다.
- 여러 팀의 의사결정을 정렬한다.

## 6. 자가 진단 체크리스트

- [ ] 내가 맡은 서비스의 핵심 불변조건을 설명할 수 있다.
- [ ] 장애 시 어떤 대시보드를 볼지 안다.
- [ ] 데이터가 어디서 생성되고 어디로 흘러가는지 안다.
- [ ] 배포와 롤백 절차를 직접 실행할 수 있다.
- [ ] 보안 경계와 권한 모델을 설명할 수 있다.
- [ ] 성능 병목을 측정으로 증명할 수 있다.
- [ ] 다음 사람을 위해 결정과 운영 지식을 문서화한다.
