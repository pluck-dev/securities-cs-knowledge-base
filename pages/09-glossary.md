# 09. 증권 용어집 (완전판)

> 증권사 백엔드 개발자가 실무에서 만나는 용어를 **카테고리별로 망라**했습니다.
> 각 표는 `용어 | 영문/약어 | 뜻 | 개발 포인트` 형식입니다.

> ⚠️ **완전성에 대하여**: 증권 용어는 매우 방대하고, 회사마다 쓰는 **내부 약어/시스템 명칭**이 따로 있습니다. 이 문서는 시장 표준 용어를 빠짐없이 담는 것을 목표로 하지만, 입사 후 마주치는 **사내 전용 용어는 이 문서에 직접 추가**하며 본인만의 사전으로 키우세요. 빠진 용어를 발견하면 해당 섹션 표에 한 줄 추가하면 됩니다.

## 1. 시장·거래소·기관

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 한국거래소 | KRX (Korea Exchange) | 유가증권·파생상품 등의 매매 및 청산결제를 담당하는 국내 유일의 종합거래소. 코스피·코스닥·코넥스·파생상품시장을 운영한다. | KRX OPEN API 및 FTP를 통해 시세·공시 데이터를 수신. 시장 코드 체계(STK=유가증권, KSQ=코스닥, KNX=코넥스)를 구분해야 한다. |
| 코스피시장 | KOSPI Market / 유가증권시장 | KRX가 운영하는 대형·주요 기업 중심의 주식시장. 시가총액 기준 국내 최대 주식시장이다. | 시장구분 코드 'STK' 또는 'KR7'. 종목코드 6자리 앞 2자리가 보통 숫자. |
| 코스닥시장 | KOSDAQ Market | 중소·벤처기업 중심의 주식시장. KRX가 운영하며 코스피와 별도의 종목코드 체계를 사용한다. | 시장구분 코드 'KSQ'. 코스피와 가격제한폭, 매매수량단위 동일하나 지수 산출 기준이 다름. |
| 코넥스시장 | KONEX Market | 초기 중소기업·스타트업을 위한 중소기업 전용 주식시장. 일반 투자자 진입 요건이 코스닥보다 엄격하다. | 시장구분 코드 'KNX'. 거래량이 적어 시세 공백(null 체결가) 처리 로직 필요. |
| K-OTC | K-OTC (Korea OTC) | KOFIA가 운영하는 비상장 주식 장외거래 플랫폼. 상장 전 기업의 주식을 거래할 수 있다. | KRX 시스템과 별도 연동. KOFIA API를 통해 호가·체결 데이터 수신. |
| 장외시장 | OTC (Over-The-Counter Market) | 거래소를 통하지 않고 당사자 간 직접 매매가 이루어지는 시장. 비상장 주식, 채권, 파생상품 등이 거래된다. | 거래소 매칭 시스템 없이 양자 간 합의가격으로 체결. 시스템 내 별도 OTC 체결 프로세스 구현 필요. |
| 한국예탁결제원 | KSD (Korea Securities Depository) | 유가증권의 예탁·결제·명의개서를 담당하는 기관. 매매 후 실물 주권 없이 전자적으로 권리를 이전한다. | 결제지시(DI) 전문 형식 및 FTP/SFTP 연동. T+2 결제 사이클에 맞춘 배치 처리 필요. |
| 금융투자협회 | KOFIA (Korea Financial Investment Association) | 금융투자업자의 자율규제기관. K-OTC 운영, 투자자 보호, 업무 표준화 등을 담당한다. | K-OTC 시세 데이터, 표준코드 조회 등을 KOFIA API를 통해 수신. |
| 금융위원회 | FSC (Financial Services Commission) | 금융정책 수립 및 금융산업 감독을 담당하는 행정부 위원회. 증권·파생 관련 규정 제·개정 권한을 보유한다. | 규정 변경 시(예: 공매도 제한, 가격제한폭 조정) 시스템 파라미터를 즉시 반영할 수 있는 설정 관리 구조 필요. |
| 금융감독원 | FSS (Financial Supervisory Service) | 금융위원회의 지도하에 금융기관을 검사·감독하는 집행기관. 증권사 내부통제 및 규정 준수를 감독한다. | 감독 보고용 데이터(예: 주문 내역, 잔고, 고객 정보) 보관 의무 기간(통상 5~10년) 및 보고 형식 준수. |
| 한국증권금융 | KSFC (Korea Securities Finance Corporation) | 증권담보대출(스탁론), 유가증권 대여, 공매도 담보 등 증권 관련 금융 서비스를 제공하는 기관. | 신용거래 담보 비율, 대주(대여) 잔고 연동 시 KSFC 인터페이스 규격 사용. |
| 회원사(증권사) | Member Firm / Securities Firm | KRX에 가입하여 매매 참여 자격을 갖춘 금융투자업자. 고객 주문을 접수하여 거래소에 제출한다. | 증권사별 고유 회원번호(예: 3자리) 존재. 주문 전문에 회원번호 필드 포함 필수. |
| 거래소회원 | Exchange Member | 거래소에 가입하여 자기 또는 위탁 명의로 매매할 수 있는 자격을 가진 증권사·선물사 등. | 회원 자격별 허용 주문 유형 상이. 비회원은 회원사를 통해 간접 참여. |
| 청산회사(중앙청산소) | CCP (Central Counterparty Clearing House) | 매매 체결 후 매도자와 매수자 사이에 개입하여 결제이행을 보증하는 기관. 한국에서는 KRX가 CCP 기능을 겸한다. | 청산 실패(결제불이행) 시 처리 프로세스 구현. 증거금 산출 모델(SPAN, TIMS 등) 연동 고려. |
| 예탁기관 | Depository | 유가증권을 실물 보관·관리하는 기관. 한국에서는 KSD가 중앙예탁기관 역할을 한다. | 예탁 잔고와 사내 잔고의 일치 여부를 주기적(일 단위) 대사(reconciliation) 처리 필요. |
| 명의개서대행 | Transfer Agent | 발행회사를 대신하여 주주명부 관리, 주권 이전 등 명의개서 업무를 처리하는 기관. | 배당, 유·무상증자 등 권리 처리 시 명의개서 기준일(record date) 데이터 연동 필요. |
| 한국은행 | BOK (Bank of Korea) | 대한민국 중앙은행. 통화정책 결정, 기준금리 설정, 결제망(BOK-Wire+) 운영을 담당한다. | 채권 금리 기준, RP 거래 등에서 BOK-Wire+ 연동 필요. 기준금리 변경 이벤트 수신 처리. |
| 시장조성자 | Market Maker | 특정 종목에 대해 지속적으로 매수·매도 호가를 제시하여 유동성을 공급하는 의무를 진 증권사. | 시장조성 대상 종목 리스트 및 호가 제출 의무(예: 스프레드 ±3% 이내) 관리 로직 필요. |
| 유동성공급자 | LP (Liquidity Provider) | ETF·ELW 등 파생결합상품에서 유동성 공급 의무를 부담하는 증권사. 매수·매도 양방향 호가를 상시 제출한다. | ETF LP 의무 호가 모니터링, 스프레드·호가잔량 기준 위반 여부 자동 경고 시스템 필요. |
| 매매전문회원 | Trading Specialist Member | 고객 위탁 없이 자기매매만 허용된 거래소 회원. 주로 알고리즘 트레이딩 전문 법인이 해당된다. | 위탁주문 수신 기능 불필요. 자기매매 계좌와 위탁계좌 혼용 방지 로직 구현. |
| MTS | MTS (Mobile Trading System) | 스마트폰 앱으로 주식 매매를 할 수 있는 모바일 트레이딩 시스템. | 네트워크 불안정 환경 대비 주문 중복 방지(idempotency key), 푸시 알림 체결 통보 구현 필요. |
| HTS | HTS (Home Trading System) | PC 기반 개인 투자자용 주식 매매 프로그램. 거래소 시세·차트·주문을 통합 제공한다. | FIX 프로토콜 또는 자체 바이너리 프로토콜로 시세 수신. 고대역폭 실시간 데이터 스트리밍 처리. |
| WTS | WTS (Web Trading System) | 별도 설치 없이 웹 브라우저에서 주식 매매가 가능한 웹 트레이딩 시스템. | WebSocket 기반 실시간 시세 수신. 크로스브라우저 호환성, 세션 타임아웃 처리 필수. |
| 영업점 | Branch Office | 증권사의 오프라인 고객 접점. 창구직원이 고객 대신 주문을 입력하거나 계좌 개설을 처리한다. | 영업점 단말기는 내부망 전용 시스템 사용. 지점 코드 관리 및 영업점별 권한 분리 필요. |
| 비대면계좌개설 | Non-face-to-face Account Opening | 영업점 방문 없이 앱·웹으로 본인 인증 후 계좌를 개설하는 방식. 금융당국 가이드라인에 따라 운영된다. | eKYC(전자신원확인) 연동, 신분증 OCR·안면인식 API, 전자서명법 준수, 계좌개설 이력 보관 의무. |
| 투자자예탁금 | Customer Deposits | 고객이 주식 매매를 위해 증권사에 예치한 현금. 증권사 고유 자산과 분리 보관 의무가 있다. | 고객 예탁금은 별도 신탁계정으로 분리. 출금 한도 및 매매 가능금액 실시간 계산 필수. |
| 자기매매 | Proprietary Trading | 증권사가 고객 위탁 없이 자사 자금으로 직접 유가증권을 매매하는 행위. | 자기 계좌와 위탁 계좌의 물리적 분리. 이해충돌 방지를 위한 정보 장벽(Chinese Wall) 시스템 구현. |
| 위탁매매 | Agency Trading (Brokerage) | 고객의 주문을 받아 거래소에 제출하고 체결 결과를 고객에게 전달하는 중개 행위. | 고객 주문과 자기 주문의 처리 경로 완전 분리. 주문 상태 추적(접수→전송→체결) 실시간 업데이트. |

| 인터넷전문은행 증권계좌 연동 | Internet-only Bank Brokerage Account Link | 카카오뱅크·토스뱅크 등 인터넷전문은행이 증권사와 연계하여 제공하는 주식 거래 서비스. | 은행-증권 간 API 연동(OAuth2, Open Banking API). 실시간 잔액 동기화 및 이체 지연 처리. |

## 2. 시장 운영·매매제도

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 정규장 | Regular Trading Session | 오전 9시~오후 3시 30분(예시)의 정규 매매 시간대. 접속매매 방식으로 운영된다. | 정규장 여부 플래그를 시간 기반으로 판별. 정규장 외 주문은 별도 큐로 분리. |
| 시가단일가(장개시동시호가) | Opening Call Auction | 장 개시 전(예: 오전 8:30~9:00) 접수된 호가를 모아 단일 가격으로 시가를 결정하는 방식. | 동시호가 시간대 주문은 즉시 체결되지 않음. 체결 예상가 표시 로직 별도 구현 필요. |
| 종가단일가 | Closing Call Auction | 장 마감 직전(예: 오후 3:20~3:30) 단일가 방식으로 종가를 결정하는 매매. | 종가단일가 진입 시 기존 미체결 지정가 주문의 처리 방침(자동 취소 여부) 확인 필요. |
| 시간외종가매매 | After-hours Closing Price Trading | 장 마감 후 당일 종가로만 매매되는 시간외 거래(예: 오후 3:40~4:00). | 체결 가격이 종가로 고정되어 호가 협상 없음. 종가 확정 후 해당 가격을 시스템에 락인. |
| 시간외단일가매매 | After-hours Single Price Auction | 장 마감 후 10분 단위 단일가 방식으로 매매하는 시간외 거래(예: 오후 4:00~6:00). | 10분 주기 배치 체결. 가격은 당일 종가 ±10% 범위 내(예시). 별도 세션 관리 필요. |
| 장전시간외 | Pre-market After-hours Trading | 정규장 개시 전(예: 오전 7:30~8:30) 전일 종가로 매매하는 시간외 거래. | 전일 종가를 기준 가격으로 사용. 호가 유효 기간 관리(당일 장전 세션 한정). |
| 장후시간외 | Post-market After-hours Trading | 정규장 종료 후 당일 종가로 매매하는 시간외 거래. 시간외종가매매와 시간외단일가매매를 포괄하는 개념. | 장후시간외 전체를 하나의 세션으로 통합 관리하거나 세부 세션으로 분리 관리. |
| 접속매매 | Continuous Auction (Order-driven) | 호가가 접수되는 즉시 상대 호가와 매칭하여 체결하는 방식. 정규장의 기본 매매 방식이다. | 주문 매칭 엔진(order matching engine) 구현 시 가격·시간 우선 원칙 적용. |

| 기준가 | Base Price | 당일 가격제한폭 산정의 기준이 되는 가격. 통상 전일 종가를 사용하며 권리락 등 사유 발생 시 조정된다. | 권리락·배당락 사유 발생 시 기준가 재산출 로직 필요. 매일 장 개시 전 갱신 배치. |
| 시가 | Opening Price | 정규장 개시 시가단일가 체결로 결정된 당일 첫 거래 가격. | 시가 확정 전까지는 null 또는 예상가 표시. 시가 결정 이벤트 수신 후 확정값 저장. |
| 종가 | Closing Price | 종가단일가 또는 마지막 체결 가격으로 결정된 당일 마지막 거래 가격. | 종가 = 다음 영업일 기준가의 기초. 종가 미체결 시 직전 체결가를 종가로 사용하는 규칙 확인. |
| 고가 | High Price | 당일 정규장에서 체결된 가격 중 가장 높은 가격. | 실시간으로 체결가와 비교하여 갱신. 시간외 체결가는 별도 고가 계산에서 제외 여부 확인. |
| 저가 | Low Price | 당일 정규장에서 체결된 가격 중 가장 낮은 가격. | 실시간으로 체결가와 비교하여 갱신. 시가 결정 이전 시간외 거래 포함 여부 규칙 확인. |
| 변동성완화장치 | VI (Volatility Interruption) | 단기간 급격한 가격 변동 시 2분간 단일가 매매로 전환하여 변동성을 완화하는 제도. | VI 발동 이벤트 수신 시 해당 종목을 단일가 모드로 전환. 2분 후 자동 접속매매 복귀. |
| 정적VI | Static VI | 예상 체결 가격이 전일 종가 대비 일정 비율(예: ±10%) 이상 변동 시 발동되는 VI. | 발동 기준가가 전일 종가이므로 매일 갱신 필요. 발동 횟수 제한 없음. |
| 동적VI | Dynamic VI | 예상 체결 가격이 직전 체결가 대비 일정 비율(예: ±3%) 이상 급변 시 발동되는 VI. | 직전 체결가 기반이므로 실시간 체결가 추적 필수. 발동 후 기준가 재설정. |
| 서킷브레이커 | CB (Circuit Breaker) | 주가지수가 급락할 때 시장 전체 거래를 일시 중단시키는 제도. 1단계(8% 이상), 2단계(15% 이상), 3단계(20% 이상, 당일 매매 종료, 예시). | CB 발동 시 전 종목 주문 접수 중단. 단계별 재개 시간(예: 20분 후) 및 종료 조건 처리. |
| 사이드카 | Sidecar (프로그램매매호가효력정지) | 선물 가격이 기준 대비 5%(예시) 이상 급변 시 5분간 프로그램매매 호가 효력을 정지시키는 제도. | 사이드카 발동 여부를 지수 선물 가격 변동으로 판별. 해당 기간 프로그램 바스켓 주문 억제. |
| 프로그램매매 | Program Trading | 여러 종목으로 구성된 바스켓을 알고리즘을 통해 일괄 매매하는 방식. 차익거래·비차익거래로 구분된다. | 바스켓 주문 일괄 전송, 개별 종목 체결 결과 집계. KRX 프로그램매매 신고 의무 데이터 생성. |
| 매매수량단위 | Trading Unit / Board Lot | 거래소에서 거래 가능한 최소 주식 수량 단위. 현행 코스피·코스닥 모두 1주(예시). | 주문 수량이 매매수량단위의 배수인지 검증. 단주(1주 미만) 처리 별도 로직 필요. |
| 호가가격단위(틱사이즈) | Tick Size (Minimum Price Increment) | 호가를 제출할 수 있는 최소 가격 간격. 주가 구간별로 다르게 적용된다(예: 1,000원 미만→1원, 1,000원~5,000원→5원 등). | 주문 가격을 해당 종목의 틱사이즈로 반올림/절사. 주가 구간 변경 시 틱사이즈 갱신 필요. |
| 호가 | Quote / Order | 매수 또는 매도 의사를 거래소에 제출한 가격과 수량의 쌍. | 호가 유형(지정가·시장가·조건부지정가 등) 코드 체계 관리. 호가 유효 기간(당일·GTC·GTD) 구분. |

| 시간우선 원칙 | Time Priority | 동일 가격의 호가는 먼저 접수된 순서대로 체결된다. | 나노초 단위 타임스탬프 기록 및 정렬. 거래소 수신 시각과 내부 접수 시각 구분 관리. |
| 수량우선 원칙 | Volume Priority | 동일 가격·시간의 호가는 수량이 많은 쪽을 우선 체결한다. 동시호가 단일가 결정 시 적용. | 동시호가 체결 알고리즘에서 수량 기준 정렬 로직 추가. |

| 상장 | Listing | 주식이 거래소의 매매 대상으로 등록되어 공개 시장에서 거래될 수 있게 되는 것. | 신규 상장 종목 마스터 데이터(종목코드, 액면가, 시장구분 등) 사전 적재 배치. |

| 거래대금 | Trading Value / Turnover | 특정 기간 동안 체결된 거래의 금액 합계(= 체결가 × 체결 수량의 합). | 거래대금 = Σ(체결가 × 수량). 정밀도를 위해 BigDecimal 또는 정수형 원화 단위 사용. |
| 시가총액 | Market Capitalization | 상장 주식 수 × 현재 주가로 산출된 기업의 시장 가치. | 실시간 현재가 변경 시 시총 갱신. 전체 시장 시총 합산으로 시장 지수 가중치 산출 연동. |
| 유통주식수 | Float (Floating Shares) | 전체 상장 주식 중 대주주·자기주식 등 고정 보유분을 제외한 실제 시장 유통 주식 수. | 유통비율 변동 시(자사주 취득·처분, 보호예수 해제 등) 갱신. 지수 산출 시 유통비율 가중치 적용. |
| 액면가 | Par Value / Face Value | 주권에 표시된 1주당 명목 가치. 국내 주식은 100원, 200원, 500원, 1,000원, 2,500원, 5,000원 등이 있다. | 액면 분할·병합 이벤트 수신 시 종목 마스터 갱신. 권리락 기준가 계산에 사용. |
| 액면분할 | Stock Split | 주당 액면가를 낮추고 주식 수를 늘리는 것(예: 5,000원→500원, 10주→100주). | 액면분할 기준일 전날 종가로 분할 후 기준가 계산. 과거 OHLCV 데이터 소급 수정 배치 필요. |
| 액면병합 | Reverse Stock Split | 주당 액면가를 높이고 주식 수를 줄이는 것(예: 500원→5,000원, 100주→10주). | 병합 후 단주 발생 시 처리 규칙(현금 보상 등) 구현. 과거 데이터 소급 수정 배치. |
| 자기주식 | Treasury Stock (자사주) | 기업이 발행한 주식 중 기업 자신이 취득하여 보유하고 있는 주식. 의결권이 없다. | 자사주 취득·처분 공시 수신 후 유통주식수 갱신. 자사주 소각 처리 시 총 발행주식수 감소. |
| 주식배당 | Stock Dividend | 현금 대신 주식으로 배당을 지급하는 것. 주식 수가 증가하고 주당 가격이 낮아진다. | 주식배당 지급 기준일 후 잔고 자동 증가 처리. 세금(배당소득세) 원천징수 계산 적용. |
| 현금배당 | Cash Dividend | 기업이 이익의 일부를 주주에게 현금으로 지급하는 것. | 배당 기준일·지급일 관리. 배당금 자동 입금 배치, 배당소득세(15.4%) 원천징수 자동화. |
| 중간배당 | Interim Dividend | 결산 전 중간에 지급하는 배당. 기말 배당과 별도로 지급된다. | 기말 배당락과 별도로 중간배당락 기준가 조정 처리. |
| 무상증자 | Bonus Issue / Gratis Allotment | 기업이 잉여금을 자본금으로 전입하여 대가 없이 신주를 주주에게 배정하는 것. | 무상증자 비율만큼 주식 수 증가, 권리락 기준가 = 전일 종가 / (1 + 무상 비율). |
| 유상증자 | Rights Offering / Paid-in Capital Increase | 기업이 신주를 발행하여 자금을 조달하는 방식. 기존 주주에게 우선 배정권을 부여한다. | 신주인수권 배정 비율, 납입 단가, 권리락 기준가 계산. 청약 미달 시 실권주 처리 로직. |
| 조건부지정가 | Market-on-Close / Conditional Limit Order | 정규장에서는 지정가로 운용되다가 종가단일가 시간에 미체결 시 시장가로 전환되는 주문 유형. | 종가단일가 진입 시 해당 주문 자동 시장가 전환 로직. 주문 유형 코드 관리. |
| 최유리지정가 | Best Limit Order | 주문 시점의 상대방 최우선 호가를 지정가로 설정하는 주문 유형. | 주문 접수 시점의 BBO(매수/매도 최우선 호가) 조회 후 가격 고정. 이후 가격 변동 미반영. |
| 시장가 주문 | Market Order | 가격을 지정하지 않고 현재 시장 가격으로 즉시 체결되기를 원하는 주문. | 시장가 주문은 상·하한가 도달 시에도 체결 시도. 슬리피지 위험 안내 로직 필요. |
| 지정가 주문 | Limit Order | 투자자가 지정한 가격 이하(매수) 또는 이상(매도)에서만 체결되는 주문. | 지정가가 상·하한가 범위 내인지 사전 검증. 가격 미달 시 오더북에 대기. |
| IOC 주문 | IOC (Immediate or Cancel) | 주문 후 즉시 체결 가능한 수량만 체결하고 나머지는 즉시 취소되는 주문 조건. | 부분 체결 후 잔여 수량 자동 취소 처리. 체결 결과 즉시 반환. |
| FOK 주문 | FOK (Fill or Kill) | 주문 전량이 즉시 체결되지 않으면 전체를 취소하는 주문 조건. | 전량 체결 가능 여부 사전 확인 후 주문 제출. 부분 체결 불가. |
| GTC 주문 | GTC (Good Till Cancelled) | 투자자가 취소하기 전까지 유효한 주문. 국내에서는 일반적으로 당일 유효가 기본이다. | 국내 거래소는 기본적으로 당일 주문만 허용. GTC는 증권사 내부에서 매일 재제출 방식으로 구현. |
| 거래정지 해제 | Trading Halt Lift | 거래정지 후 다시 매매를 허용하는 조치. 단일가 매매로 재개되는 경우가 많다. | 거래정지 해제 이벤트 수신 후 주문 접수 재활성화. 재개 직후 단일가 처리 여부 확인. |
| 매도호가 | Ask Price / Offer Price | 매도자가 팔고자 하는 가격. 오더북에서 상단에 표시된다. | 매도호가 1~10단계 실시간 갱신. 잔량 합계(매도잔량) 및 최우선 매도 호가(best ask) 관리. |
| 매수호가 | Bid Price | 매수자가 사고자 하는 가격. 오더북에서 하단에 표시된다. | 매수호가 1~10단계 실시간 갱신. 잔량 합계(매수잔량) 및 최우선 매수 호가(best bid) 관리. |
| 호가잔량 | Order Queue Depth | 각 호가 단계별로 매매 대기 중인 수량의 합계. | KRX UDP 멀티캐스트로 수신되는 호가 잔량 전문 파싱. 증분 업데이트 방식 지원 필요. |
| 예상체결가 | Indicative Price / Expected Price | 단일가 매매 시간대에 현재 호가 집계를 기반으로 계산된 예상 체결 가격. | 동시호가 시간대에 매 틱마다 예상 체결가를 재계산. 화면 표시용으로만 사용, 확정 체결가 아님. |
| 수정주가 | Adjusted Price | 주식 분할·배당·증자 등 권리 발생으로 주가가 불연속적으로 변동할 때 과거 주가를 소급 조정한 주가. | 차트 및 수익률 계산에는 수정주가 사용. 수정계수(Adjustment Factor) 관리 및 소급 적용 배치. |
| 신용거래 | Margin Trading | 증권사로부터 자금이나 주식을 빌려 투자하는 거래. 신용매수(융자)와 신용매도(대주)로 구분된다. | 신용 한도 실시간 관리, 담보 비율 모니터링, 반대매매 자동화 로직. 종목별 신용 허용 여부 조회. |
| 반대매매 | Forced Liquidation | 신용거래에서 담보 비율이 기준 이하로 떨어질 때 증권사가 강제로 매도하는 것. | 담보 유지 비율(예: 140%) 실시간 모니터링. 기준 미달 시 익일 장 개시 후 시장가 자동 매도. |
| 담보비율 | Collateral Ratio / Margin Ratio | 신용거래 시 대출금에 대한 담보 가치의 비율. | 담보 = 주식 평가액 × 대용가 + 현금. 비율 = 담보 / 대출. 매 체결마다 재계산. |
| 대용가 | Substitution Value | 주식을 현금 대신 담보로 사용할 때 인정하는 평가 가치(통상 전일 종가의 60~80%, 예시). | 종목별 대용가 비율을 매일 갱신. 관리종목 등은 대용 불인정 처리. |
| 위탁증거금 | Margin Deposit | 주문을 체결하기 위해 증권사에 사전 납부해야 하는 보증금. 종목·투자자 등급에 따라 비율이 다르다. | 주문 전 가용 증거금 계산. 체결 후 증거금 확정 및 결제일 현금 차감 처리. |
| 미수금 | Unsettled Balance (Cash Deficit) | 매수 체결 후 결제일에 결제 대금이 부족한 상태. 다음 영업일 강제 매도 대상이 된다. | 결제일 전 계좌 잔액 모니터링. 미수 발생 즉시 고객 알림, 익일 반대매매 자동 실행. |
| 미수동결계좌 | Frozen Account (due to unsettled debit) | 미수금이 반복 발생하거나 결제 불이행 시 일정 기간 현금 결제만 허용되는 계좌. | 동결 계좌 플래그 관리. 동결 기간 중 신용·대주 주문 차단, 당일 결제(현금 100%) 강제 적용. |

## 4. 주문 유형·속성

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|------------|
| 지정가주문 | Limit Order | 투자자가 직접 가격을 지정하여 제출하는 주문. 지정가 이상(매도) 또는 이하(매수)에서만 체결됨 | 주문 수신 시 가격 유효성 검증(호가단위 준수) 필수. 미체결 시 호가창에 잔류 |
| 시장가주문 | Market Order | 가격을 지정하지 않고 즉시 현재 시장 최선가에 체결되도록 하는 주문 | 단가 필드를 0 또는 null로 처리. 가격 급변 시 예상치 못한 가격에 체결될 수 있으므로 슬리피지 모니터링 필요 |
| 조건부지정가주문 | Conditional Limit Order | 장 중에는 지정가로 동작하다가 장 마감 시 미체결 잔량이 있으면 시장가로 전환되는 주문 | 장 종료 직전(15:20) 미체결 주문을 자동으로 시장가로 전환하는 배치 로직 구현 필요 |
| 최유리지정가주문 | Best Limit Order | 주문 접수 시점의 상대방 최우선호가를 지정가로 설정하는 주문 | 접수 시 실시간 호가 스냅샷을 읽어 단가를 동적으로 결정. 호가 공백 시 처리 로직 필요 |
| 최우선지정가주문 | First Priority Limit Order | 주문 접수 시점의 자신 측 최우선호가에 지정하는 주문. 호가창 상단에 즉시 올라감 | 매수의 경우 현재 매수 최우선호가, 매도의 경우 현재 매도 최우선호가로 단가 설정 |
| 목표가주문 | Target Price Order | 특정 목표 가격 도달 시 자동으로 주문을 제출하는 주문 유형 | 실시간 체결가 모니터링 후 조건 충족 시 신규 주문 생성. 트리거 가격과 실제 주문가격 분리 관리 |
| 경쟁대량매매 | Block Trade / Competitive Bulk Trade | KRX의 대량매매 제도 중 하나로, 기관 간 대량 주식을 별도 시장에서 경쟁 방식으로 매매 | 일반 주문과 별도 채널(시장 구분 코드) 사용. 최소 수량 조건 검증 필수 |
| 시간외대량매매 | After-Hours Block Trade | 정규 장 이외의 시간(장 전·후)에 대량 매매를 협의하여 체결하는 거래 | 거래 가능 시간대 별도 관리(08:00~08:30, 15:40~16:00 등). 가격은 당일 종가 기준 ±범위 제한 |
| 바스켓주문 | Basket Order | 다수 종목을 한 번에 묶어 일괄 제출하는 주문 방식. 인덱스 펀드 리밸런싱 등에 활용 | 종목별 개별 주문으로 분리 후 순차 또는 병렬 전송. 트랜잭션 묶음 처리 및 부분 실패 롤백 전략 필요 |
| 예약주문 | Reserved/Scheduled Order | 장 개시 전 미리 주문을 등록해 두고 장 시작 시 자동 제출되는 주문 | DB에 예약 상태로 저장 후 장 시작 시 스케줄러가 일괄 전송. 중복 전송 방지 로직 필수 |
| 스톱로스주문 | Stop-Loss Order | 보유 포지션이 일정 손실 수준에 도달하면 자동으로 매도(청산)하는 주문 | 실시간 평가손익 또는 체결가 모니터링. 트리거 발동 후 시장가 또는 지정가 주문 자동 생성 |
| 스톱리밋주문 | Stop-Limit Order | 스톱 가격 도달 시 지정가 주문을 자동 제출하는 주문. 체결 보장 없음 | 트리거 가격(stop price)과 주문 단가(limit price) 두 가격 필드 별도 관리 |
| 자동주문 | Automated/Algorithmic Order | 사전 정의된 조건(가격, 시간, 수량 등)에 따라 시스템이 자동으로 생성·제출하는 주문 | 무한 루프 방지를 위한 최대 주문 횟수 제한 및 킬 스위치 구현 필수 |
| IOC | Immediate Or Cancel | 접수 즉시 체결 가능한 수량만 체결하고 미체결 잔량은 즉시 취소하는 주문 조건 | 주문 전송 후 미체결 잔량에 대한 취소 요청을 즉시 자동 발송. 별도 취소 플래그 필드 관리 |
| FOK | Fill Or Kill | 주문 수량 전량이 즉시 체결되지 않으면 전량 취소하는 주문 조건 | 매칭엔진에서 전량 체결 가능 여부 사전 확인 후 주문 수락/거부 처리. 부분 체결 후 취소 시나리오 방지 |
| GTC | Good Till Cancelled | 명시적으로 취소하기 전까지 유효한 주문 조건 (국내는 당일 유효가 원칙이나 일부 상품에서 지원) | KRX 기준 정규 주문은 당일 장 종료 시 자동 소멸. GTC 구현 시 익일 재전송 또는 주문 이월 로직 필요 |
| 일부충족조건 | Partial Fill Condition | 주문 수량의 일부만 체결되어도 허용하는 조건. 일반적인 지정가 주문의 기본 동작 | 체결 수량과 미체결 잔량을 별도 필드로 추적. 복수 부분 체결 시 누적 합산 처리 |
| 전량충족조건 | All-Or-None / AON | 주문 수량 전량이 체결되지 않으면 체결하지 않는 조건 | FOK와 달리 즉시 취소 강제는 아닐 수 있음. 호가창 잔류 가능 여부에 따라 구현 방식 상이 |
| 매수주문 | Buy Order | 증권을 취득하기 위해 제출하는 주문 | 매수/매도 구분 코드(1/2 또는 B/S)를 모든 주문 메시지에 포함. 매수 가능 금액 사전 검증 |
| 매도주문 | Sell Order | 보유 증권을 처분하기 위해 제출하는 주문 | 매도 가능 수량(보유 수량 - 매도 주문 중 수량) 실시간 검증. 과매도 방지 로직 |
| 신규주문 | Open Order / New Order | 새로운 포지션을 여는 주문. 선물·옵션에서는 포지션 개시를 의미 | 파생상품의 경우 신규/청산 구분 코드 필수. 증거금 계산 시 신규와 청산의 증거금 요율 상이 |
| 청산주문 | Close Order / Liquidation Order | 기존 포지션을 종료하는 주문. 선물·옵션에서 반대 방향 매매로 포지션 해소 | 보유 포지션 수량을 초과하는 청산 주문 차단. 청산 시 증거금 반환 처리 순서 주의 |
| 정정주문 | Order Modification / Amend Order | 기제출 주문의 가격 또는 수량을 변경하는 요청 | 원주문번호와 정정 내용을 함께 전송. 이미 체결된 수량은 정정 불가. 수량 감소만 허용(증가 불가)하는 규칙 적용 |
| 취소주문 | Order Cancellation | 미체결 주문을 철회하는 요청 | 원주문번호 기반 취소 요청. 이미 체결된 주문 또는 처리 중 주문 취소 시 오류 처리. 취소 확인 메시지 비동기 수신 처리 |
| 원주문 | Original Order | 정정 또는 취소의 대상이 되는 최초 주문 | 주문 체인 관리: 원주문번호와 파생 주문(정정/취소) 연결 관계 DB 저장. 체결 내역 조회 시 원주문 기준 집계 |
| 주문번호 | Order Number / Order ID | 주문 건별 고유 식별자. KRX 및 증권사 내부 주문번호 체계가 별도로 존재 | KRX 주문번호(거래소 채번)와 내부 주문번호(사내 채번) 매핑 테이블 관리. 채번 시 중복 방지를 위해 시퀀스 또는 분산 ID 사용 |
| 주문수량 | Order Quantity | 주문 건에 명시된 거래 수량(주, 계약 수 등) | 종목별 매매 단위(1주, 1계약 등) 및 최소 주문 수량 검증. 정수형 처리(소수점 허용 여부 상품별 상이) |
| 주문단가 | Order Price | 주문에 지정된 체결 희망 가격 | 호가 단위(틱 사이즈) 검증 필수. 가격 범위 검증(상하한가 내 여부). 시장가 주문은 0 또는 특수 값으로 처리 |
| 미체결잔량 | Unfilled Quantity / Open Quantity | 주문 수량 중 아직 체결되지 않은 잔여 수량 | 주문 수량 - 누적 체결 수량으로 실시간 계산. 정정·부분 체결 발생마다 갱신 |
| 주문가능수량 | Available Order Quantity | 투자자가 현재 추가로 주문할 수 있는 최대 수량 | 보유 수량 - 매도 주문 중 수량(매도 시). 매수는 주문 가능 금액 ÷ 주문 단가로 산출. 실시간 갱신 필요 |
| 주문가능금액 | Available Order Amount | 투자자가 매수에 사용할 수 있는 최대 금액. 예수금·신용 한도 등 반영 | 예수금 + 신용 한도 - 미결제 매수 주문 금액 합계. 수수료 예상액 포함하여 보수적 계산 권장 |
| 호가효력 | Order Validity / Time In Force | 주문이 호가창에서 유효하게 유지되는 조건과 기간 | Day(당일), GTC, IOC, FOK 등 TIF 코드로 관리. 장 종료 시 Day 주문 자동 소멸 배치 처리 |
| 자기주식주문 | Treasury Stock Order | 상장법인이 자사 주식을 매수하는 주문. 별도 공시 의무 및 매매 제한 존재 | 자사주 매매 가능 수량·금액·기간 한도 관리. 공시 연동 및 KRX 신고 처리 로직 포함 필요 |
| 공매도주문 | Short Selling Order | 보유하지 않은 주식을 빌려서 매도하는 주문 | 차입 가능 수량 사전 확인. 업틱룰 적용 여부 실시간 체크. 공매도 금지 종목 필터링. 잔고 마이너스 허용 여부 설정 |
| 차입공매도 | Covered Short Selling | 주식을 대차 계약을 통해 빌린 후 매도하는 공매도. 국내에서 합법적 공매도 방식 | 대차 잔고와 공매도 주문 수량 연동 관리. 반환 일정 추적. 대차료 정산 처리 |
| 업틱룰 | Uptick Rule | 공매도 호가는 직전 체결가보다 낮은 가격으로 제출할 수 없다는 규칙 | 공매도 주문 접수 시 직전 체결가와 비교하여 호가 적정성 실시간 검증. 위반 시 주문 거부 |
| 단일계좌 다중주문 | Multiple Orders per Account | 하나의 계좌에서 동일 종목에 복수의 주문을 동시에 보유하는 것 | 계좌별 주문 수량 합산 시 누락 없이 집계. 동일 방향 주문 합산 및 반대 방향 주문 상계 처리 |
| 분할주문 | Order Splitting / Child Order | 대량 주문을 시장 충격을 최소화하기 위해 소량으로 분할하여 순차 제출하는 방식 | 부모 주문(parent order)과 자식 주문(child order) 계층 구조 관리. 분할 비율, 시간 간격 파라미터화 |
| TWAP주문 | Time-Weighted Average Price Order | 지정 시간 동안 균등하게 분할하여 주문을 제출하는 알고리즘 주문 방식 | 총 수량 ÷ 슬롯 수로 균등 분할. 스케줄러가 정해진 시간 간격으로 자식 주문 생성·전송 |
| VWAP주문 | Volume-Weighted Average Price Order | 시장의 거래량 비중에 따라 수량을 배분하여 주문하는 알고리즘 방식 | 과거 또는 예측 거래량 패턴 데이터 필요. 시간대별 목표 수량 동적 계산. 실시간 실적 대비 조정 로직 |
| 알고리즘주문 | Algorithmic Order / Algo Order | 사전 정의된 알고리즘에 따라 자동으로 주문을 생성·제출·관리하는 방식 | 알고리즘 파라미터 버전 관리. 긴급 정지(kill switch) 기능 필수. 주문 생성 속도 제한(rate limiting) |
| 스마트오더라우팅 | Smart Order Routing / SOR | 여러 거래소 또는 ATS를 실시간 비교하여 최적 호가로 주문을 자동 라우팅하는 시스템 | 복수 시장의 실시간 호가 수집 및 비교 엔진 필요. 분산 체결 시 각 시장 체결 결과 통합 관리 |
| 조건부주문 | Conditional Order | 특정 조건(가격·수량·시간·지표 등) 충족 시 자동으로 발동되는 주문 | 조건 모니터링 엔진과 주문 엔진 분리 설계. 조건 충족 시 중복 발동 방지 플래그 관리 |
| 연계주문 | Linked Order / OCO | OCO(One Cancels the Other) 등 두 주문 중 하나가 체결되면 나머지를 자동 취소하는 주문 쌍 | 주문 간 연결 관계 DB 관리. 한 쪽 체결 이벤트 수신 즉시 나머지 취소 요청 자동 발송 |
| 역지정가주문 | Stop Order | 지정 가격(stop price) 도달 시 시장가 주문으로 전환되어 체결되는 주문 | 실시간 체결가 모니터링. 트리거 발동 후 시장가 주문 자동 생성. 갭 발생 시 목표 가격 미도달 위험 고지 |
| 시간지정주문 | Time-Specific Order | 특정 시간에만 유효하거나 특정 시간에 제출되는 주문 | 주문 유효 시작·종료 시각 필드 관리. 스케줄러 정밀도(초 단위) 및 시간 동기화(NTP) 중요 |
| 프로그램매매주문 | Program Trading Order | 사전 설정된 프로그램 기준에 따라 다수 종목을 동시에 매매하는 주문 방식. 차익거래·리밸런싱 등 | KRX 프로그램매매 신고 요건 충족. 바스켓 주문과 연동. 차익/비차익 거래 구분 코드 관리 |
| 장전시간외주문 | Pre-Market Order | 정규장(09:00) 개시 전 시간외 단일가 매매(08:30~08:40)에 제출하는 주문 | 시간대별 허용 주문 유형 제한. 장전 시간외는 전일 종가로만 매매 가능하도록 가격 고정 로직 |
| 장후시간외주문 | After-Market Order | 정규장 종료 후 시간외 단일가(15:30~16:00) 또는 시간외 종가(15:30~15:40)에 제출하는 주문 | 종가 기준 매매 가격 고정. 매매 가능 종목 및 수량 제한 확인 |
| 대량매매주문 | Bulk/Block Order | 일정 수량 이상의 대량 주문으로 KRX 별도 대량매매 시장에서 처리 | KRX 대량매매 최소 수량 기준(종목별 상이) 충족 여부 사전 검증. 별도 시장 코드로 주문 전송 |
| 주문유효기간 | Order Validity Period / TIF | 주문이 효력을 유지하는 기간 설정. Day, GTD, GTC, IOC, FOK 등 | TIF 코드를 주문 메시지 헤더에 포함. 만료 시점에 자동 취소 처리 배치 또는 이벤트 구현 |
| 복합주문 | Combo Order | 두 개 이상의 주문을 하나의 단위로 묶어 처리하는 주문 유형. 스프레드·컨버전 등 | 구성 주문 간 전량 체결 원자성 보장 필요. 일부만 체결 시 처리 정책 명확화 |
| 주문재전송 | Order Re-submission | 통신 장애·오류로 미전송된 주문을 다시 전송하는 처리 | 멱등성 보장을 위한 주문 고유 식별자(클라이언트 주문 ID) 활용. 중복 체결 방지 필수 |
| 주문상태 | Order Status | 주문의 현재 처리 상태 (접수, 전송, 체결, 부분체결, 취소, 거부 등) | 상태 전이 모델(FSM) 명확히 정의. 상태 변경 이력 타임스탬프 포함 저장. 비정상 상태 알림 처리 |
| 주문거부 | Order Rejection | 증권사 또는 KRX에서 주문을 거부하는 상황. 사유코드와 함께 반환 | 거부 사유 코드 체계적 관리. 재시도 가능 여부 분류. 고객 통보 메시지 자동 생성 |
| 주문접수 | Order Receipt / Acknowledgement | 주문이 거래소 또는 증권사 시스템에 정상 수신되었음을 확인하는 단계 | 접수 타임스탬프 기록. ACK 수신 전 타임아웃 처리 및 재전송 로직 구현 |
| 위탁주문 | Brokered Order / Client Order | 고객이 증권사를 통해 위탁하여 제출하는 주문 | 고객 계좌 식별자, 위탁 수수료율, 세금 계산 연동. MTS/HTS/ARS 채널별 구분 코드 관리 |
| 자기매매주문 | Proprietary Order | 증권사가 자기 자본으로 직접 매매하는 주문. 고객 주문과 명확히 구분 | 계좌 유형(위탁/자기) 구분 필드 관리. 내부 한도 관리 및 리스크 통제 별도 적용 |

## 5. 호가(Quote)

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|------------|
| 매수호가 | Bid | 매수자가 제시하는 매수 희망 가격. 호가창의 하단(낮은 가격) 영역 | 매수호가는 가격 내림차순 정렬 유지. 동일 가격 다수 주문은 시간 우선으로 내부 관리 |
| 매도호가 | Ask / Offer | 매도자가 제시하는 매도 희망 가격. 호가창의 상단(높은 가격) 영역 | 매도호가는 가격 오름차순 정렬 유지. 매도 최우선 호가가 매수 최우선 호가보다 항상 높아야 함 |
| 최우선매수호가 | Best Bid | 매수호가 중 가장 높은 가격. 즉시 체결 가능한 최고 매수 가격 | 실시간 갱신. 호가창 변동 시마다 재계산. 시장가 매도 주문 체결 기준가 |
| 최우선매도호가 | Best Ask / Best Offer | 매도호가 중 가장 낮은 가격. 즉시 체결 가능한 최저 매도 가격 | 실시간 갱신. 시장가 매수 주문 체결 기준가. Best Bid와 함께 스프레드 산출에 사용 |
| 호가잔량 | Quote Volume / Order Book Depth | 각 호가 단계별 매수 또는 매도 주문의 총 잔여 수량 | 주문 접수·체결·취소 발생마다 해당 가격 단계의 잔량 실시간 갱신. 음수 방지 검증 |
| 호가단계 | Quote Level / Price Level | 호가창에 표시되는 가격 단계 수. KRX 기준 상하 각 10단계 표시 | 10단계×2(매수·매도) = 20개 가격 레벨 실시간 관리. 호가창 전체를 배열 구조로 메모리 캐싱 |
| 총매수잔량 | Total Bid Volume | 호가창 내 모든 매수호가 단계의 잔량 합계 | 전체 10단계 매수 잔량 합산. 공개된 매수 압력 지표로 활용 |
| 총매도잔량 | Total Ask Volume | 호가창 내 모든 매도호가 단계의 잔량 합계 | 전체 10단계 매도 잔량 합산. 공개된 매도 압력 지표로 활용 |
| 예상체결가 | Expected Trading Price / ETP | 현재 호가 상황에서 단일가 경매가 진행될 경우 예상되는 체결 가격 | 동시호가·단일가 구간에서 계산. 매수·매도 수량이 최대로 체결되는 가격을 시뮬레이션 계산 |
| 예상체결수량 | Expected Trading Volume | 예상체결가에서 체결될 것으로 예상되는 수량 | 예상체결가 산출 알고리즘과 함께 계산. 해당 가격의 매수·매도 잔량 중 소량 값 |
| 스프레드 | Spread / Bid-Ask Spread | 최우선매도호가와 최우선매수호가의 가격 차이. 시장 유동성의 핵심 지표 | Best Ask - Best Bid로 실시간 계산. 스프레드 급증 시 유동성 경고 알림 가능 |
| 매수1~10호가 | Bid Level 1~10 | 최우선매수호가부터 10번째 매수호가까지의 가격과 잔량 | 배열 인덱스 0~9로 관리. 호가 변동 시 해당 레벨만 업데이트하여 성능 최적화 |
| 매도1~10호가 | Ask Level 1~10 | 최우선매도호가부터 10번째 매도호가까지의 가격과 잔량 | 배열 인덱스 0~9로 관리. KRX STEP 메시지 수신 시 변동 레벨만 증분 업데이트 |
| 호가스프레드 | Quote Spread | 호가창에서 특정 두 호가 단계 간의 가격 차이. 유동성 측정에 활용 | 틱 단위로 환산하여 비교. 특정 종목의 호가 스프레드 임계치 초과 시 알림 |
| 시장가호가 | Market Quote | 가격 미지정 상태의 호가. 호가창에서는 별도 구역에 표시 | 시장가 주문의 호가창 표현. 가격 기반 정렬 대상에서 제외하고 최우선 처리 |
| 잔량변동 | Volume Change / Delta | 특정 호가 단계의 잔량이 이전 대비 증감한 양 | 이전 잔량과 현재 잔량 차이를 스트림에 포함. 증분 업데이트(delta) 방식으로 대역폭 절감 |
| 순매수잔량 | Net Bid Volume | 총매수잔량에서 총매도잔량을 뺀 값. 시장의 순 매수 압력 | 총매수잔량 - 총매도잔량. 양수면 매수 우위, 음수면 매도 우위 |
| 순매도잔량 | Net Ask Volume | 총매도잔량에서 총매수잔량을 뺀 값 | 순매수잔량과 반대값. 부호 기반으로 매수/매도 우위 판단 |
| 매수우위 | Bid Dominance | 매수 잔량이 매도 잔량보다 많아 매수세가 강한 상태 | 잔량 비율 또는 차이로 수치화. UI 색상 구분 및 지표 계산에 활용 |
| 매도우위 | Ask Dominance | 매도 잔량이 매수 잔량보다 많아 매도세가 강한 상태 | 매수우위와 동일한 계산 구조. 방향만 반대 |
| 체결강도 | Trading Strength / Volume Ratio | 매수 체결량과 매도 체결량의 비율로 매수·매도 세력 비교 지표 | 일정 기간 매수 체결 수량 ÷ 매도 체결 수량 × 100. 100 초과 시 매수 우세 |
| 거래원 | Broker / Member | 증권사별 매수·매도 주문 내역을 집계한 정보. 외국인·기관·개인 구분에도 활용 | KRX 거래원 코드와 증권사 내부 코드 매핑. 거래원별 순매수 집계 실시간 업데이트 |
| 실시간호가 | Real-time Quote | 시장에서 실시간으로 변동되는 호가 정보. 스트리밍 방식으로 제공 | WebSocket 또는 KRX KOSCOM MDG를 통해 수신. 지연(latency) 모니터링 및 타임스탬프 비교 |
| 스냅샷호가 | Snapshot Quote | 특정 시점의 호가 상태를 일괄 캡처한 데이터 | 세션 초기화 또는 재연결 시 전체 호가창 복원용으로 사용. 증분 업데이트 이전 기준점 |
| 호가창 | Order Book | 매수·매도 호가와 잔량을 가격 단계별로 정렬하여 표시한 구조 | 가격 레벨을 키로 하는 정렬된 맵(TreeMap) 구조 권장. 빈번한 업데이트에 대한 성능 최적화 필수 |
| 호가 단위 | Tick Size / Price Step | 호가를 제출할 수 있는 최소 가격 변동 단위. 주가 구간별로 상이 | KRX 호가 단위 테이블(주가 구간별 5~1000원) DB화. 주문 단가의 호가 단위 배수 여부 검증 |
| 동결호가 | Locked Quote | 매수 최우선호가와 매도 최우선호가가 동일한 비정상 상태 | 발생 시 즉시 감지 및 알림. 매칭 지연 또는 시스템 이상의 신호일 수 있음 |
| 역전호가 | Crossed Quote | 매수 최우선호가가 매도 최우선호가보다 높은 비정상 상태 | 정상적으로는 발생하면 안 됨. 발생 감지 시 즉시 시스템 점검 트리거. 네트워크 지연으로 순서 역전 시 발생 가능 |
| 호가 공백 | Quote Gap | 매수·매도 호가 사이 또는 호가 단계 사이에 주문이 없는 가격 구간 | 최유리·최우선 지정가 주문 처리 시 호가 공백 여부 확인 필요. 유동성 부족 경고 |
| 직전체결가 | Last Trade Price / LTP | 가장 최근에 체결된 거래의 가격 | 업틱룰 검증 기준가. 예상체결가 계산 입력값. 실시간 체결 이벤트마다 갱신 |
| 가중평균호가 | Volume Weighted Quote | 호가 잔량을 가중치로 한 평균 호가 가격 | 각 레벨의 (가격 × 잔량) 합계 ÷ 총잔량으로 계산. 시장 심리 분석용 지표 |
| 호가 접수 시각 | Quote Timestamp | 호가(주문)가 거래소 시스템에 접수된 시각. 나노초 단위까지 기록 | KRX 타임스탬프와 내부 시스템 시각 동기화 필수. 시간 우선 원칙 적용 기준 |
| 비공개호가 | Hidden Order / Iceberg Order | 전체 수량 중 일부만 호가창에 표시하고 나머지는 비공개로 유지하는 주문 | 표시 수량과 전체 수량 분리 관리. 표시 수량 체결 시 다음 표시 수량 자동 보충 로직 |
| 아이스버그주문 | Iceberg Order | 비공개호가와 동일. 큰 주문을 소량씩 나누어 공개하는 방식 | Peak size와 전체 주문 수량 분리 저장. 체결마다 남은 수량에서 다음 피크 자동 생성 |
| 호가 변동률 | Quote Change Rate | 특정 호가의 직전 대비 변동 비율 | (현재 호가 - 직전 호가) ÷ 직전 호가 × 100. 급변 시 이상 거래 모니터링 트리거 |
| 매수매도비율 | Buy-Sell Ratio | 매수 잔량 대비 매도 잔량의 비율 | 총매수잔량 ÷ 총매도잔량. 1.0 초과 시 매수 우위. 실시간 대시보드 지표 |
| 호가제공자 | Market Maker / Liquidity Provider | 지속적으로 매수·매도 양방향 호가를 제출하여 유동성을 공급하는 주체 | LP 호가와 일반 호가 구분 코드 관리. LP 의무 호가 조건 준수 여부 모니터링 |
| 최우선호가 스프레드 | Best Bid-Ask Spread | Best Bid와 Best Ask의 가격 차이. 협의의 스프레드 | (Best Ask - Best Bid) / Best Ask × 100으로 상대 스프레드 계산. 유동성 지표 |
| 호가 리플레시 | Quote Refresh | 호가창 데이터를 전체 재수신하여 갱신하는 작업 | 네트워크 재연결 또는 장 개시 시 스냅샷 수신 후 증분 업데이트로 전환 |
| 잔량소진 | Order Book Exhaustion | 특정 가격 레벨의 잔량이 0이 되어 해당 호가 단계가 사라지는 것 | 잔량이 0 이하로 내려가지 않도록 보호. 레벨 제거 후 다음 레벨이 최우선으로 승격 처리 |

## 6. 체결(Execution)·매칭

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|------------|
| 체결 | Execution / Trade | 매수 주문과 매도 주문이 가격·수량 조건을 충족하여 실제 거래가 이루어진 상태 | 체결 이벤트 발생 시 주문 상태, 잔량, 포지션, 예수금 등 다수 테이블 원자적 업데이트 필요 |
| 전량체결 | Full Fill / Complete Fill | 주문 수량 전부가 체결된 상태 | 체결 수량 = 주문 수량 확인. 주문 상태를 '체결완료'로 변경. 잔량 0 처리 |
| 부분체결 | Partial Fill | 주문 수량의 일부만 체결되고 나머지는 미체결로 남아 있는 상태 | 누적 체결 수량 관리. 주문 상태를 '부분체결'로 유지. 복수의 부분체결 이벤트 처리 |
| 미체결 | Unfilled / Open | 주문이 접수되었으나 아직 체결되지 않은 상태 | 장 종료 시 미체결 주문 자동 취소 배치. 미체결 주문 건수 및 금액 실시간 집계 |
| 체결가 | Execution Price / Trade Price | 실제로 거래가 이루어진 가격 | 체결가 기반 평균 단가, 손익 계산. 체결가는 주문 단가와 다를 수 있음(시장가, 슬리피지) |
| 체결수량 | Execution Quantity / Filled Quantity | 해당 체결 건에서 거래된 수량 | 부분체결 시 각 건별 체결 수량 기록. 누적 체결 수량으로 전량 체결 판단 |
| 체결시각 | Execution Timestamp / Trade Time | 체결이 발생한 정확한 시각. KRX는 나노초·마이크로초 단위 제공 | KRX 체결 시각과 내부 수신 시각 모두 기록. 체결 시각 기준 정산 및 보고서 생성 |
| 체결통보 | Execution Report | 체결 발생 시 시스템이 관련 당사자에게 전달하는 체결 확인 메시지 | FIX 프로토콜 기준 Execution Report(MsgType=8) 처리. 고객 통보 채널(앱 푸시, SMS, 이메일) 연동 |
| 체결번호 | Execution ID / Trade ID | 체결 건별 고유 식별자. KRX 및 내부 채번 체계 분리 | KRX 체결번호와 내부 채번 매핑 보관. 중복 체결 통보 방지를 위한 멱등성 처리 |
| 매칭엔진 | Matching Engine | 매수·매도 주문을 가격·시간 우선 원칙에 따라 자동으로 매칭하여 체결시키는 핵심 시스템 | 단일 스레드 직렬 처리로 순서 보장 일반적. 수십만 TPS 처리를 위한 저지연 설계(메모리 내 주문장) |
| 가격시간우선원칙 | Price-Time Priority | 더 유리한 가격의 주문을 먼저 체결하고, 동일 가격이면 먼저 접수된 주문을 우선 체결하는 규칙 | 주문장을 가격 우선 정렬 후 시간 순 큐로 관리. 타임스탬프 정밀도가 우선순위에 직접 영향 |
| 자전거래 | Crossing Trade / Cross Trade | 동일한 주체가 동일 종목을 동시에 매수와 매도하여 인위적으로 체결을 일으키는 거래 | 동일 계좌 또는 관련 계좌 간 자전 여부 실시간 감지. 적발 시 주문 차단 또는 경보 발송 |
| 통정매매 | Wash Trade / Matched Order | 사전에 합의하여 인위적으로 거래량이나 시세를 조작하는 불법 행위 | 이상 거래 탐지 시스템(FDS) 연동. 특정 패턴(동일 시간대, 동일 수량, 반복 매매) 탐지 알고리즘 |
| 단일가체결 | Single Price Auction / Call Auction | 일정 시간 동안 호가를 모아 하나의 가격으로 일괄 체결하는 방식. 동시호가 구간에서 사용 | 접수된 모든 주문을 수집 후 체결 극대화 가격 계산. 연속 체결 방식과 완전히 다른 로직 |
| 동시호가체결 | Simultaneous Quotes Matching | 장 개시 전 또는 종료 전 일정 시간 동안 접수된 주문을 동시에 체결하는 제도 | 장 전 동시호가(08:30~09:00), 장 중 동시호가, 종가 동시호가(15:20~15:30) 각 구간별 로직 분리 |
| 시가결정 | Opening Price Determination | 장 개시 시 단일가 경매를 통해 그날의 첫 번째 체결가를 결정하는 과정 | 동시호가 결과로 시가 확정. 시가 = 매수·매도 수량 체결 최대화 단일가 |
| 종가결정 | Closing Price Determination | 장 마감 시 단일가 경매를 통해 그날의 마지막 체결가를 결정하는 과정 | 15:20~15:30 동시호가 → 15:30에 종가 확정. 종가는 차일 기준가·시간외 매매 기준으로 활용 |
| 거래체결내역 | Trade History / Execution History | 일정 기간 동안 발생한 체결 거래의 목록과 상세 내용 | 체결 건별 저장(종목, 가격, 수량, 시각, 계좌). 조회 성능을 위해 날짜·계좌·종목 복합 인덱스 |
| 약정 | Contract / Trade Confirmation | 매매 거래가 성립되어 쌍방 간 계약이 확정된 상태. '체결'과 혼용 | 약정 기준 수수료, 세금 계산. 약정 금액 = 체결 단가 × 체결 수량 |
| 약정금액 | Contract Amount / Trade Amount | 체결된 거래의 총 금액. 체결가 × 체결 수량 | 수수료 계산 기준. 당일 약정 금액 누계로 수수료 등급 산정 |
| 반대매매체결 | Forced Liquidation Execution | 증거금 부족 등으로 증권사가 강제로 고객 보유 포지션을 청산하는 체결 | 담보 비율 하락 모니터링 → 임계치 도달 시 자동 반대매매 주문 생성. 고객 사전 통지 프로세스 포함 |
| LP호가체결 | LP Quote Execution | 유동성공급자(LP)가 제출한 호가에 의한 체결. ETF·신주인수권증서 등에서 활용 | LP 체결은 일반 체결과 동일 처리 가능. LP 의무 이행 현황 별도 모니터링 |
| 호가창소진 | Order Book Sweep | 시장가 또는 대량 주문이 한 번에 여러 호가 단계의 잔량을 소진하며 체결되는 것 | 다단계 체결 처리: 각 가격 레벨별 체결 수량·금액을 개별 체결 레코드로 분리 저장 |
| 체결률 | Fill Rate | 주문 수량 대비 실제 체결된 수량의 비율. 주문 집행 품질 지표 | 체결 수량 ÷ 주문 수량 × 100. 알고리즘 주문 성과 측정 및 SOR 최적화에 활용 |
| 슬리피지 | Slippage | 주문 단가와 실제 체결가의 차이. 시장 충격 및 유동성 부족으로 발생 | 예상 체결가와 실제 체결가 차이를 건별 기록. 알고리즘 성과 분석 및 비용 계산 |
| 시장충격비용 | Market Impact Cost | 대량 주문 집행 시 호가창을 이동시켜 발생하는 추가 비용 | 주문 전후 VWAP 비교로 측정. 알고리즘 주문 분할의 핵심 최소화 대상 |
| 체결 우선순위 | Execution Priority | 동일 가격의 주문들 사이에서 체결 순서를 결정하는 규칙 (시간, 수량, 고객 구분 등) | KRX 기준 가격→시간 우선. 동시호가 중 수량 우선 규칙 적용 여부 확인 |
| 연속체결 | Continuous Auction / Continuous Trading | 장 중 매수·매도 주문이 접수될 때마다 즉시 매칭을 시도하는 방식 | 09:00~15:20 정규 장 중 적용. 주문 접수 즉시 매칭엔진 호출 |
| 체결 확인서 | Trade Confirmation | 체결 완료 후 고객에게 발송하는 거래 확인 문서 | 법적 의무 문서. 체결일, 종목, 수량, 가격, 수수료, 세금 포함. 이메일·우편 자동 발송 |
| 체결 잔고 반영 | Position Update on Fill | 체결 발생 시 보유 잔고(포지션)를 즉시 업데이트하는 처리 | 체결 이벤트 수신 → 잔고 테이블 원자적 업데이트 → 평균 단가 재계산. 동시성 이슈(낙관적 잠금 또는 큐 직렬화) 처리 |
| 동시 체결 | Concurrent Execution | 동일 시각에 복수 주문이 체결되는 상황. 동시호가 체결 시 빈번히 발생 | 배치 체결 처리 시 DB 트랜잭션 묶음 최적화. 데드락 방지를 위한 처리 순서 고정 |
| 체결 평균 단가 | Average Execution Price / VWAP | 복수 체결 건의 수량 가중 평균 체결가 | Σ(체결가 × 수량) ÷ 총 체결 수량. 평균 매입 단가 산출 및 손익 계산 기준 |
| 체결 지연 | Execution Latency | 주문 제출부터 체결 확인 수신까지 걸리는 시간 | 주문 전송 타임스탬프와 체결 수신 타임스탬프 차이를 측정·기록. SLA 임계값 초과 시 알림 |
| 역체결 | Trade Bust / Erroneous Trade | 착오 또는 시스템 오류로 발생한 체결을 취소하는 것. KRX 착오거래 취소 제도 | 착오거래 신청 시한 내(체결 후 30분) KRX 신청. 취소 처리 시 관련 잔고·예수금·체결 내역 전체 롤백 |
| 체결 통계 | Trade Statistics | 일정 기간 체결 건수, 거래량, 거래대금 등을 집계한 통계 | 실시간 집계 vs 배치 집계 분리. 고빈도 업데이트에는 in-memory 집계 후 주기적 DB 플러시 |
| 시간외 체결 | Off-Hours Execution | 정규 장 외 시간대(장전·장후 시간외)에 이루어진 체결 | 시간외 체결가는 당일 종가로 고정. 별도 세션 코드 및 체결 유형 코드로 정규 장 체결과 구분 |
| 대량 체결 | Block Execution | 대량매매 시장에서 협의된 대량 거래의 체결 | 일반 체결과 별도 처리 경로. KRX 대량매매 체결 확인 메시지 별도 수신 및 파싱 |
| 매칭 알고리즘 | Matching Algorithm | 매수·매도 주문을 매칭하는 로직. 가격시간우선, 수량우선, 비례배분 등 다양한 방식 | KRX는 가격시간우선 기본. 동시호가 구간에서는 비례배분 방식 적용 여부 확인 필요 |
| 정정 체결 | Amendment Fill | 주문 정정 후 정정된 조건으로 이루어지는 체결 | 정정 주문번호 기반으로 원주문과 연결. 정정 후 체결 수량은 정정 주문의 잔량 내에서만 허용 |
| 취소 체결 | Cancelled Fill | 이미 체결된 주문을 취소하는 역체결(Bust). 일반적으로는 발생하면 안 됨 | KRX 착오거래 취소 확인 메시지 수신 시 체결 삭제 및 관련 데이터 전체 원상복구 |
| 체결 이벤트 | Trade Event | 체결 발생을 알리는 시스템 내부 또는 외부 이벤트 | 이벤트 기반 아키텍처에서 체결 이벤트를 메시지 큐(Kafka 등)에 발행. 소비자별 독립 처리 |
| 체결 우선 배분 | Pro-Rata Allocation | 동일 가격 주문을 수량 비율에 따라 분배 체결하는 방식. 일부 파생상품 시장에서 사용 | 수량 비율 계산 시 반올림 처리로 총합 불일치 발생 가능. 잔여 수량 처리 규칙 명확화 필요 |
| 거래 내역 원장 | Trade Ledger | 모든 체결 거래를 기록하는 불변(immutable) 원장 데이터 | 삭제·수정 불가 원칙. 역체결은 역분개(compensating entry)로 처리. 감사 추적 목적 영구 보관 |
## 7. 계좌·예수금·증거금

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|---|---|---|---|
| 위탁계좌 | Brokerage Account | 고객이 증권사에 매매를 위탁하는 기본 계좌 | 계좌유형코드(ACCT_TYPE_CD)로 구분; 위탁/신용/종합 등 복수 유형 관리 |
| 종합계좌 | Integrated Account | 위탁·CMA·신용 등 여러 기능을 하나의 계좌번호 아래 통합한 계좌 | 서브계좌(Sub-account) 구조; 잔고 집계 시 서브별 합산 로직 필요 |
| CMA | Cash Management Account | 고객 예수금을 MMF·RP 등에 자동 운용해 이자를 지급하는 계좌 | 운용유형(MMF/RP/종금) 코드 관리; 일별 이자 계산 배치 |
| ISA | Individual Savings Account / 개인종합자산관리계좌 | 비과세·분리과세 혜택을 부여하는 절세 계좌 (일반형/서민형/농어민형) | 납입한도 누적 관리(BigDecimal); 계좌 만기·해지 시 세금 처리 분기 필요 |
| 연금저축계좌 | Pension Savings Account | 세액공제 혜택을 받는 장기 연금 상품 계좌 | 연간 납입한도(예: 600만 원) 초과 여부 검증; 예시이며 사내/법령 기준 확인 |
| IRP | Individual Retirement Pension / 개인형퇴직연금 | 퇴직금을 적립·운용하는 개인형 퇴직연금 계좌 | 퇴직금 입금 이벤트 처리; 중도인출 가능 사유 코드 관리 |
| 펀드계좌 | Fund Account | 펀드 매수·환매·분배금 수령에 사용되는 계좌 | 좌수(BigDecimal, 소수점 3~4자리) 및 기준가 정밀도 관리 |
| 신탁계좌 | Trust Account | 고객 재산을 수탁해 운용하는 계좌 | 신탁원장 별도 관리; 신탁보수 자동 계산 배치 |
| 해외주식계좌 | Overseas Stock Account | 해외 거래소 주식 매매를 위한 계좌 | 통화 코드(USD, JPY 등) 멀티 커런시; 환전 잔고 연계 |
| 계좌번호 | Account Number | 계좌를 식별하는 고유 번호 | 체크디지트 검증 로직 내재화; 마스킹(앞 n자리 노출) 처리 |
| 예수금 | Deposit Balance / Cash Balance | 계좌에 예치된 현금 총액 | BigDecimal 저장; D+0/D+1/D+2 일자별 분리 관리 |
| 출금가능금액 | Withdrawable Amount | 즉시 출금 가능한 금액 (예수금 - 증거금 - 미결제금 등) | 실시간 차감 로직; 음수 방지를 위한 검증 필수 |
| 주문가능금액 | Orderable Amount | 신규 주문 시 사용할 수 있는 금액 (현금 + 대용금 활용) | 증거금률 적용 후 산출; BigDecimal 반올림(RoundingMode.FLOOR) 권장 |
| 증거금 | Margin | 주문 시 담보로 확보하는 금액 | 종목별·시장별 증거금률 테이블 참조; 실시간 재계산 |
| 위탁증거금 | Brokerage Margin | 위탁매매 시 요구하는 증거금 | 증거금률 × 주문금액; 예시이며 사내/법령 기준 확인 |
| 위탁증거금률 | Brokerage Margin Ratio | 위탁증거금을 산정하는 비율 (종목별 상이, 예: 40%, 100%) | 종목 마스터(ITEM_MARGIN_RATIO)에서 조회; 예시이며 사내/법령 기준 확인 |
| 현금증거금 | Cash Margin | 증거금 중 현금으로 납입한 부분 | 대용금과 합산하여 총증거금 구성 |
| 대용증거금 | Substitute Margin | 보유 주식·채권 등을 현금 대신 증거금으로 인정하는 것 | 대용가율 × 평가금액; 종목별 대용가율 테이블 관리 |
| 대용금 | Substitute Securities Value | 대용증거금의 현금 환산 금액 | BigDecimal; 장중 가격 변동 시 재산출 필요 |
| 증거금률(종목별) | Margin Ratio per Security | 종목별로 차등 적용되는 증거금 비율 (예: 일반 40%, 관리종목 100%) | ITEM_MRGN_RATIO 테이블; 변경 이력 관리; 예시이며 사내/법령 기준 확인 |
| 미수금 | Uncollected Amount / Debit | 결제일 기준 입금이 부족하여 발생한 미수채권 | 미수 발생일 기록; 연체이자 계산(일별 배치); BigDecimal |
| 미수동결계좌 | Frozen Account (Margin Call Default) | 미수 미납으로 인해 신용·미수 거래가 정지된 계좌 | FREEZE_YN 플래그; 동결 해제 조건(완납) 자동 처리 |
| 결제대금 | Settlement Amount | 매매 체결 후 T+2(국내 주식 기준) 결제 시 납입/수령하는 금액 | 체결금액 + 수수료 + 세금 합산; 예시이며 사내/법령 기준 확인 |
| D+1 예수금 | D+1 Deposit | 익영업일 결제 예정 포함 예수금 | 날짜별 슬롯(D0/D1/D2) 구조로 관리; 영업일 캘린더 참조 |
| D+2 예수금 | D+2 Deposit | 익익영업일 결제 예정 포함 예수금 | 결제일 도래 시 D0으로 전환하는 배치 |
| 예탁금이용료 | Interest on Customer Deposits | 예수금에 대해 증권사가 고객에게 지급하는 이자 | 일별 이자 = 예수금 × 이율 / 365; BigDecimal; 이율은 사내 기준 확인 |
| 예수금이체 | Cash Transfer | 계좌 간 예수금을 이체하는 거래 | 원자적 트랜잭션(출금 계좌 차감 + 입금 계좌 증액 동시); 이중 출금 방지 Lock |
| 가상계좌 | Virtual Account | 고객별 입금 식별을 위해 발급하는 일회성 또는 전용 계좌번호 | 은행 가상계좌 연동 API; 입금 통보 수신 후 예수금 반영 배치 |
| 출금신청 | Withdrawal Request | 고객이 예수금 출금을 신청하는 행위 | 출금가능금액 실시간 검증; 신청 후 이체 전까지 금액 예약(hold) 처리 |
| 입금 | Deposit / Credit | 고객 계좌에 현금이 입금되는 거래 | 실시간 반영 또는 배치 반영 여부 구분; 입금원 구분 코드 |
| 자동이체 | Auto-transfer / Standing Order | 사전 약정에 따라 자동으로 이체되는 거래 | 이체 스케줄 테이블; 실패 시 재시도 정책 및 알림 |
| 증거금부족 | Margin Deficit / Margin Shortage | 주문 시 증거금이 요구 수준에 미달하는 상태 | 주문 수신 시점에 선검증(pre-validation); 부족액 계산 후 오류 반환 |
| 추가증거금 | Additional Margin / Margin Call | 평가손실 등으로 증거금이 유지증거금 이하로 내려갈 때 추가 납부 요구 | 마진콜 발생 이벤트 큐잉; 납부 기한 관리; SMS/Push 알림 연동 |
| 마진콜 | Margin Call | 추가증거금 납부 요구와 동일 개념(선물·옵션에서 주로 사용) | 미납 시 강제청산(반대매매) 트리거 조건 관리 |
| 반대매매 | Forced Liquidation | 미수·신용 미납 시 증권사가 보유 종목을 강제 처분하는 행위 | 대상 잔고 조회 → 시장가 주문 자동 생성; 감사 로그 필수 |
| 신용계좌 | Margin Loan Account | 증권사 자금을 빌려 매매하는 신용거래 계좌 | 신용한도·담보비율 실시간 관리; 반대매매 임계값 모니터링 |
| 신용한도 | Credit Limit | 신용거래 가능한 최대 차입 금액 | 담보비율 기반 산출; 실시간 갱신 필요 |
| 담보비율 | Collateral Ratio | 신용융자 잔액 대비 담보 자산의 비율 | (담보평가금액 / 신용융자금액) × 100; BigDecimal; 유지비율 미달 시 알림 |
| 유지증거금 | Maintenance Margin | 포지션을 유지하기 위해 최소한으로 보유해야 하는 증거금 | 초기증거금과 별도 비율 관리; 예시이며 사내/법령 기준 확인 |
| 초기증거금 | Initial Margin | 신규 포지션 진입 시 요구되는 증거금 | 유지증거금보다 높게 설정; 예시이며 사내/법령 기준 확인 |
| 계좌개설 | Account Opening | 신규 계좌를 생성하는 프로세스 | KYC/AML 검증 연동; 비대면 계좌개설 API 플로우 |
| 계좌해지 | Account Closure | 계좌를 폐쇄하는 프로세스 | 잔고·미결제·미수 전량 정리 후 해지; 세금 정산 선행 |
| 계좌정지 | Account Suspension | 거래를 일시 정지하는 상태 | ACCT_STATUS_CD; 정지 사유 코드 및 해제 조건 관리 |
| 계좌이관 | Account Transfer | 타 금융기관으로 계좌 및 잔고를 이전하는 것 | 잔고 이전 프로토콜(ACATS 등); 세금 이관 처리 주의 |

## 8. 잔고·손익·평가

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|---|---|---|---|
| 보유잔고 | Holdings / Position | 계좌에서 현재 보유 중인 종목과 수량 | 잔고 원장 테이블(HOLDINGS); 체결·입출고 발생 시 실시간 업데이트 |
| 잔고수량 | Balance Quantity / Held Quantity | 보유 중인 종목의 총 수량 | 정수형 또는 BigDecimal(소수점 허용 종목: ETF 분할, 해외주식 등) |
| 매입단가 / 평균단가 (평단가) | Average Purchase Price | 보유 수량에 대한 평균 취득 단가 | 총매입금액 / 잔고수량; BigDecimal; 추가매수 시 가중평균 재계산 |
| 매입금액 | Purchase Amount / Cost Basis | 보유 종목의 총 취득 원가 | 체결금액 + 수수료 포함 여부 정책 결정; 세금 신고 기준 |
| 평가금액 | Evaluated Amount / Market Value | 현재 시장가 기준으로 산출한 보유 종목의 가치 | 현재가 × 잔고수량; 장중 실시간 갱신; BigDecimal |
| 평가손익 (미실현손익) | Unrealized Gain/Loss / Paper P&L | 보유 중이나 아직 매도하지 않은 상태의 손익 | 평가금액 - 매입금액; 음수 허용; BigDecimal |
| 평가수익률 | Return Rate / Yield | 평가손익을 매입금액 대비 비율로 표현 | (평가손익 / 매입금액) × 100; 0 나누기 예외 처리 필수 |
| 실현손익 | Realized Gain/Loss | 매도 체결 완료 후 확정된 손익 | (매도금액 - 수수료 - 세금) - (평균단가 × 매도수량); 세금 신고 기준 |
| 손익통산 | Profit/Loss Netting | 여러 종목·거래의 손익을 합산하여 세금 과세표준을 줄이는 것 | 기간별·종목별 집계; 금투세 도입 시 통산 범위 확대; 예시이며 법령 기준 확인 |
| 총자산 | Total Assets | 예수금 + 평가금액 + 기타 자산의 합계 | 멀티 계좌 합산 시 통화 환산 포함; BigDecimal |
| 순자산 | Net Assets | 총자산 - 부채(신용융자·미수금 등) | 음수 가능; 실시간 대차대조 개념 |
| 추정자산 | Estimated Assets | 장중 현재가 기준 실시간 추정 총자산 | 마감 후 확정자산과 구분; 캐시 갱신 주기 설정 |
| 출금가능 | Withdrawable Balance | 즉시 현금 출금 가능한 금액 | 예수금 - 증거금 - 미수금 - 출금예약금; BigDecimal; 음수 방지 |
| 예탁자산 | Deposited Assets | 계좌에 예탁된 전체 자산(현금 + 유가증권 평가액) | 담보비율 산정 기준; 신용거래 한도 산출에 활용 |
| 결제예정수량 | Pending Settlement Quantity | 체결은 됐으나 아직 결제(T+2)가 완료되지 않은 수량 | D+1/D+2 슬롯 구분; 결제일 도래 시 확정 잔고로 전환 배치 |
| 미결제약정 | Open Interest / Unsettled Contract | 선물·옵션에서 아직 청산되지 않은 계약 수 | 파생상품 원장 별도 관리; 만기일 강제 청산 트리거 |
| 보유비중 | Portfolio Weight | 특정 종목의 평가금액이 총 포트폴리오에서 차지하는 비율 | (종목 평가금액 / 총평가금액) × 100; 실시간 갱신 |
| 매도가능수량 | Sellable Quantity | 매도 주문을 낼 수 있는 최대 수량 | 잔고수량 - 매도 주문 중 수량 - 대기수량 - 권리수량; 정수 |
| 대기수량 | Pending Quantity / In-order Quantity | 주문이 접수되었으나 아직 체결되지 않아 잔고에서 묶인 수량 | 주문 생성 시 증가, 체결/취소 시 감소; 원자적 업데이트 |
| 권리수량 | Rights Quantity / Locked Quantity | 주식배당·유상증자 등 권리 행사를 위해 잠금된 수량 | 권리일 기준 설정·해제 배치; 매도 차단 로직 연동 |
| 입고 | Receipt / Stock-in | 외부에서 유가증권이 계좌로 들어오는 것 | 한국예탁결제원 연동; 입고 확인 후 잔고 반영 |
| 출고 | Delivery / Stock-out | 계좌에서 유가증권이 외부로 나가는 것 | 잔고 차감 후 예탁원 이전 처리; 출고가능 여부 사전 검증 |
| 대체 | Transfer / Book Transfer | 동일 증권사 내 계좌 간 유가증권을 이전하는 것 | 원장 내 출고·입고 동시 처리; 세금 이관 처리 정책 확인 |
| 잔고대사 | Position Reconciliation | 사내 잔고 원장과 예탁결제원·청산기관 잔고를 대조·검증하는 작업 | 일배치 대사; 차이 발생 시 알림 및 조정 프로세스 |
| 평가기준가 | Valuation Price | 잔고 평가에 사용하는 기준 가격 (종가, 최유리호가 등) | 장중/장후 기준 분리; 가격 공급원(시세 서버) 명시 |
| 수익률 | Rate of Return | 일정 기간 투자 수익의 비율 | 단순수익률 vs. 시간가중수익률(TWR); 계산 방식 통일 필요 |
| 누적수익률 | Cumulative Return | 투자 시작 이후 누적된 수익률 | 복리 계산; 기준일 명확화 |
| 일별손익 | Daily P&L | 당일 평가손익 변동분 | 전일 평가금액과 당일 평가금액 차이 + 당일 실현손익 |
| 기간손익 | Period P&L | 특정 기간의 손익 합산 | 기간 내 실현손익 + 기말 평가손익 - 기초 평가손익 |
| 투자원금 | Invested Principal | 투자에 사용된 원래 금액 | 입금 이력 기반 산출; 이체 입금 vs. 운용 수익 구분 |
| 배당금수령 | Dividend Receipt | 주식 배당금이 계좌에 입금되는 것 | 배당 기준일·지급일 관리; 원천징수 후 실수령액 반영 |
| 분배금 | Distribution | 펀드·ETF 등에서 발생한 이익을 투자자에게 분배하는 금액 | 좌수 기준 안분; 재투자 옵션 분기 처리 |
| 대여잔고 | Securities Lending Balance | 대차거래로 빌려준 주식의 잔량 | 대여 수수료 수취 계산; 대여 기간·상환 관리 |
| 차입잔고 | Securities Borrowing Balance | 대차거래로 빌린 주식의 잔량 | 공매도 연동; 차입 수수료 비용 처리 |
| 공매도잔고 | Short Selling Balance | 차입 후 매도한 미상환 잔량 | 상환 의무 추적; 공매도 보고 의무(법령) 연동; 예시이며 법령 기준 확인 |
| 손절 | Stop-loss | 손실이 일정 수준에 달했을 때 매도하는 행위 | 자동손절 주문 조건 엔진; 임계 손실률 파라미터 관리 |
| 익절 | Take-profit | 목표 이익에 도달했을 때 매도하는 행위 | 목표 수익률 조건 주문 엔진 |

## 9. 세금·수수료·비용

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|---|---|---|---|
| 증권거래세 | Securities Transaction Tax (STT) | 주식 매도 시 매도금액에 부과되는 세금 | 매도체결금액 × 세율; BigDecimal; RoundingMode.FLOOR 또는 정책 기준 반올림; 예시이며 사내/법령 기준 확인 |
| 농어촌특별세 | Special Rural Development Tax | 증권거래세에 부가되는 세금 (코스피 거래세의 일부로 납부) | 거래세 × 농특세율; 코스피·코스닥 적용 여부 상이; 예시이며 법령 기준 확인 |
| 양도소득세 | Capital Gains Tax (CGT) | 주식 양도로 발생한 차익에 부과되는 소득세 | 대주주 요건·해외주식·비상장주식 판단 로직; 연간 손익통산; 예시이며 법령 기준 확인 |
| 배당소득세 | Dividend Income Tax | 배당금에 부과되는 소득세 | 원천징수 15.4%(소득세 14% + 지방소득세 1.4%); BigDecimal; 예시이며 사내/법령 기준 확인 |
| 금융투자소득세 | Financial Investment Income Tax / 금투세 | 주식·채권·펀드 등 금융투자 상품의 손익을 통산하여 과세하는 세제(향후 도입 예정·유예 논의 중) | 법 시행 여부 및 일정 수시 모니터링; 손익통산 범위·세율 구현 준비; 예시이며 법령 기준 확인 |
| 원천징수 | Withholding Tax | 소득 지급 시 지급자가 세금을 미리 공제하고 납부하는 방식 | 배당·이자 지급 시 자동 차감; 원천징수세액 신고 배치; 세금코드 관리 |
| 거래수수료 | Transaction Fee | 매매 체결 시 발생하는 총 수수료 (위탁수수료 + 유관기관제비용) | 체결금액 × 수수료율; BigDecimal; 최소수수료 정책 반영 |
| 위탁수수료 | Brokerage Commission | 고객이 증권사에 지급하는 매매 중개 수수료 | 수수료 체계(정액/정률/온라인 할인 등) 코드화; 예시이며 사내 기준 확인 |
| 유관기관제비용 | Related Institution Fee | 거래소·예탁원 등에 납부하는 제비용 | 거래소수수료 + 예탁결제수수료 합산; 종목·시장별 단가 테이블 |
| 거래소수수료 | Exchange Fee | 한국거래소(KRX)에 납부하는 수수료 | 매매 체결 건당 산출; 정산 배치에서 KRX 청구 데이터와 대사 |
| 예탁결제수수료 | KSD Settlement Fee | 한국예탁결제원(KSD) 결제·보관 서비스에 대한 수수료 | 결제금액 기준 산출; 예시이며 사내/법령 기준 확인 |
| 매매수수료 | Trading Commission | 위탁수수료와 동의어로 사용되기도 하며, 체결 단위로 부과되는 수수료 | 매수·매도 각각 발생; 수수료 면제 이벤트 처리 로직 |
| 환전수수료 | Foreign Exchange Fee | 외화 환전 시 부과되는 수수료 | 환율 스프레드 또는 정액 수수료; 우대환율 쿠폰 적용 분기 |
| 환율 | Exchange Rate | 통화 간 교환 비율 | 실시간 환율 시세 연동; 매매기준율·매입율·매도율 구분; BigDecimal 소수점 4자리 이상 |
| 슬리피지비용 | Slippage Cost | 주문 가격과 실제 체결 가격의 차이로 발생하는 암묵적 비용 | 체결분석 리포트용 계산; 직접 과세 대상 아님 |
| 세금우대 | Tax Preference / Tax Benefit | 일정 요건 충족 시 세금을 감면받는 혜택 | 계좌 유형별 세금우대 코드 관리; 한도 초과 시 일반 과세로 전환 |
| 비과세 | Tax-exempt | 과세 대상에서 제외되어 세금이 발생하지 않는 것 | ISA·연금 계좌 비과세 한도 관리; 예시이며 법령 기준 확인 |
| 분리과세 | Separate Taxation | 금융소득을 종합소득에 합산하지 않고 별도로 과세하는 방식 | ISA 만기 해지 시 분리과세 처리; 예시이며 법령 기준 확인 |
| 종합과세 | Global Income Taxation / Comprehensive Taxation | 모든 소득을 합산하여 누진세율로 과세하는 방식 | 금융소득 2,000만 원 초과 시 종합과세 전환; 예시이며 법령 기준 확인 |
| 손익통산 (세금) | Loss Carryforward / Gain-Loss Netting (Tax) | 양도손실을 양도이익과 상계하여 과세표준을 줄이는 것 | 연간 집계 배치; 종목·상품 유형별 통산 가능 범위 법령 확인 |
| 이연 | Tax Deferral | 과세 시점을 미래로 연기하는 것 (연금 계좌 내 운용 수익 등) | 연금 수령 시점에 과세 이벤트 트리거; 이연 기간 관리 |
| 세금계산서 | Tax Invoice | 부가가치세가 적용되는 거래에서 발행하는 세금 증빙 (법인 고객 등) | 전자세금계산서 발행 API 연동; 공급가액·세액 BigDecimal |
| 거래내역서 | Transaction Statement | 일정 기간 거래 내역을 정리한 문서 | 조회 기간·종목·거래유형 필터; 대용량 데이터 페이징·엑셀 다운로드 |
| 원천징수영수증 | Withholding Tax Receipt / Certificate | 원천징수한 세금 내역을 증명하는 문서 | 연말 배치 생성; 고객 조회·출력·국세청 전송 연동 |
| 지방소득세 | Local Income Tax | 소득세의 10%에 해당하는 지방세 | 소득세 계산 후 10% 추가; 원천징수 동시 처리; 예시이며 법령 기준 확인 |
| 이자소득세 | Interest Income Tax | 예금·채권 이자에 부과되는 소득세 | 이자 지급 시 원천징수; 비과세 상품 여부 사전 판단 |
| 금융소득 | Financial Income | 이자소득 + 배당소득의 합계 | 연간 2,000만 원 초과 시 종합과세; 누적 집계 실시간 관리; 예시이며 법령 기준 확인 |
| 과세표준 | Taxable Income / Tax Base | 세금을 계산하는 기준이 되는 금액 (소득 - 공제) | 손익통산 후 최종 산출; BigDecimal; 음수 시 0으로 처리 |
| 공제 | Deduction | 과세표준 산정 시 차감하는 금액 (기본공제·필요경비 등) | 공제 유형 코드 관리; 자동 적용 vs. 신청 분기 |
| 세율 | Tax Rate | 과세표준에 적용하는 비율 | 누진세율 구간 테이블 관리; BigDecimal; 예시이며 법령 기준 확인 |
| 원천징수의무자 | Withholding Agent | 세금을 원천징수하여 납부할 의무가 있는 자 (증권사) | 증권사가 의무자; 신고·납부 기한 배치 스케줄링 |
| 적격외국금융기관 | Qualified Foreign Financial Institution / QFFI | 외국인 투자자에 적용되는 세금 면제·감면 자격 기관 | 외국인 투자자 구분 코드; 조세조약 기준 세율 적용; 예시이며 법령 기준 확인 |
| 조세조약 | Tax Treaty | 이중과세 방지를 위해 국가 간 체결하는 협약 | 국가별 조약 세율 테이블; 외국인 배당세율 분기 처리 |
| 해외주식 양도소득세 | Overseas Stock Capital Gains Tax | 해외 주식 매도 차익에 부과되는 양도소득세 | 연간 250만 원 기본공제 후 22% 세율(예시; 법령 기준 확인); 환율 적용 원화 환산 필수; BigDecimal |
| 외국납부세액공제 | Foreign Tax Credit | 해외에서 납부한 세금을 국내 세금에서 공제하는 것 | 국가별 납부세액 집계; 공제 한도 계산; 예시이며 법령 기준 확인 |
| 수수료 할인 | Fee Discount | 특정 조건(VIP 등급, 이벤트 등)에서 수수료를 감면하는 것 | 할인율 코드 및 적용 우선순위 관리; 기간 유효성 검증 |
| 최소수수료 | Minimum Commission | 수수료가 일정 금액 이하일 경우 적용하는 하한선 | MAX(산출수수료, 최소수수료) 로직; 예시이며 사내 기준 확인 |
| 수수료율 | Commission Rate | 거래금액 대비 수수료 비율 | 시장·계좌유형·채널별 차등 테이블; 변경 이력 관리 |
| 후취수수료 | Back-end Load | 펀드 환매 시 부과되는 수수료 | 보유 기간에 따른 체감 수수료율 테이블; 환매일 기준 산출 |
| 선취수수료 | Front-end Load | 펀드 매수 시 선불로 부과되는 수수료 | 매수금액에서 수수료 차감 후 순투자금 계산 |
| 신탁보수 | Trust Management Fee | 신탁 운용에 대해 수탁사에 지급하는 보수 | 일별 보수 = 신탁재산 × 보수율 / 365; BigDecimal |
| 운용보수 | Management Fee | 펀드 운용에 대해 운용사에 지급하는 보수 | 기준가 계산 시 반영; 총보수(TER) 구성 요소 |
| 판매보수 | Sales Fee | 펀드 판매에 대해 판매사에 지급하는 보수 | 총보수(TER) 구성 요소; 기준가 일별 차감 |
| 총보수비율 | Total Expense Ratio / TER | 펀드 운용·판매·신탁 보수를 합산한 연간 비용 비율 | 기준가 산출 시 일할 차감; 소수점 4자리 이상 BigDecimal |
| 세금환급 | Tax Refund | 과납한 세금을 환급받는 것 | 연말정산·수정신고 후 환급 이벤트; 예수금 반영 배치 |
| 가산세 | Additional Tax / Penalty Tax | 세금 신고·납부 지연 시 부과되는 가산 세금 | 미납 일수 × 가산세율; 예시이며 법령 기준 확인 |
| 세금신고 | Tax Filing | 세금 관련 신고서를 세무 당국에 제출하는 행위 | 국세청 전자신고 API 연동; 신고 기한 스케줄 관리 |
| 원천징수이행상황신고서 | Withholding Tax Return | 원천징수한 세금을 신고하는 서류 | 월별 신고 배치; 합계 금액 검증 후 국세청 전송 |
| 지급명세서 | Payment Record | 소득 지급 내역을 기재한 법정 서류 | 연간 배치 생성; 고객·종목·금액별 명세 집계 |
## 10. 청산·결제·정산

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 청산 | Clearing | 매매체결 후 매수·매도 당사자 간 채권·채무를 확정하고 상계하는 과정. KRX가 중앙청산소(CCP) 역할 수행 | 체결 후 KRX로부터 청산 데이터 수신 시점과 결제 데이터 생성 시점 구분 필요 |
| 결제 | Settlement | 청산된 채권·채무를 실제 증권·대금의 이동으로 이행하는 과정 | T+2 기준 결제일 자동 산출 로직, 휴장일 제외 처리 필수 |
| T+2 결제 | T+2 Settlement | 매매체결일(T)로부터 2 영업일째에 결제 이행. 국내 주식 표준 | 결제일 = T + 2 영업일. 달력 기반 영업일 테이블 필요 |
| T+0 결제 | Same-day Settlement | 당일 결제. 일부 채권·단기금융상품에 적용 | 결제일 = 체결일. 시스템 내 결제사이클 구분 코드 관리 |
| T+1 결제 | T+1 Settlement | 체결 익일 결제. ETF·국채 일부 적용 | 결제일 산출 시 사이클 파라미터화 권장 |
| 결제일 | Settlement Date | 실제 증권·대금이 오가는 날짜 | DB에 settle_date 별도 컬럼으로 저장, 체결일과 혼용 금지 |
| 결제대금 | Settlement Amount | 결제 시 이동해야 할 금액(체결금액 ± 제비용) | 수수료·세금 차감 후 순금액 계산 필요 |
| 결제이행 | Settlement Fulfillment | 결제일에 증권·대금을 정상적으로 납부하는 것 | 이행 완료 상태 플래그(settled) 관리, 타임스탬프 기록 |
| 결제불이행 | Settlement Failure / Default | 결제일에 증권 또는 대금을 납부하지 못한 상태 | 불이행 감지 시 알림·제재 프로세스 트리거 필요 |
| 차감결제 | Netting | 동일 결제일 내 동일 종목의 매수·매도를 상계하여 차액만 결제 | 단일 종목별 net position 집계 쿼리, 방향(매수/매도) 부호 처리 |
| 다자간차감 | Multilateral Netting | 여러 참가자 간 채권·채무를 일괄 상계. KRX CCP 방식 | KRX 순결제 내역 파일 파싱 후 사내 포지션에 반영 |
| DVP | Delivery Versus Payment | 증권 인도와 대금 지급을 동시에 연계하여 결제리스크 제거 | KSD SAFE+ 시스템과 연계. 증권 인도 확인 후 대금 지급 순서 보장 |
| 예탁 | Custody / Deposit | 투자자가 보유 증권을 예탁기관(KSD)에 맡기는 것 | 예탁잔고와 실물잔고 이중 관리, KSD 잔고 대사 주기적 수행 |
| 예탁자 | Depositor | KSD에 계좌를 개설하고 증권을 예탁한 기관(주로 증권사) | 예탁자 코드 = KSD 부여 기관 식별자. API 호출 시 헤더 삽입 |
| 실질주주 | Beneficial Owner | 명의와 무관하게 실질적으로 주식을 소유한 투자자 | 명의개서 전까지 KSD 예탁 원장에서 실질주주 관리 |
| 명의개서 | Transfer of Ownership / Register | 주주명부상 소유자를 변경하는 절차 | 배당기준일 직전 명의개서 정지 일정 관리, 이벤트 트리거 설계 |
| 대사 | Reconciliation | 사내 장부와 KSD/KRX 공식 잔고·내역을 비교·검증하는 작업 | 일별 자동 대사 배치 구현. 불일치 건 별도 큐 적재 후 수기 처리 |
| 정산 | Clearing & Settlement (통합) / 내부정산 | 거래 결과를 최종 확정하여 계좌에 반영하는 내부 절차 | EOD(장 마감 후) 정산 배치 실행 시간 SLA 관리 |
| 일일정산 | Daily Mark-to-Market Settlement | 파생상품 미결제약정을 매일 시가로 평가하여 손익을 정산 | 선물·옵션 계좌의 일일 변동증거금 계산 및 입출금 처리 |
| 마감처리 | End-of-Day Processing / EOD | 영업일 종료 시 모든 거래·잔고를 확정하는 배치 처리 | 배치 실행 순서(체결→청산→정산→잔고갱신) 의존성 그래프 관리 |
| 결제유동성 | Settlement Liquidity | 결제일에 대금을 원활히 지급할 수 있는 자금 여력 | 결제 예정 대금 집계 → 가용 현금 비교 → 부족 시 조달 프로세스 연동 |
| 결제리스크 | Settlement Risk | 결제 불이행으로 발생할 수 있는 손실 위험 | 거래상대방별 결제 익스포저 실시간 계산 필요 |
| 증권결제 | Securities Settlement | 결제일에 증권을 인도하는 행위 | KSD SAFE+를 통한 계좌 간 대체 처리, 결과 수신 후 잔고 반영 |
| 대금결제 | Cash Settlement / Payment Settlement | 결제일에 대금을 지급하는 행위 | 한국은행 BOK-Wire+ 또는 은행계좌 이체 연계 |
| 미결제 | Open Position / Unsettled | ① 파생: 만기 미도달 잔존 계약 ② 현물: 결제 미완료 거래 | 컨텍스트에 따라 의미 구분. 파생은 open_interest, 현물은 unsettled_trade |
| 결제내역 | Settlement Detail | 결제 완료된 거래별 상세 기록 | 규제 보고용 보관 기간(5년 이상) 준수, 조회 인덱스 설계 |
| 영업일 | Business Day | 거래소·은행이 정상 영업하는 날(토·일·공휴일 제외) | 공휴일 테이블 연도별 사전 적재, 결제일 산출 함수에서 참조 |
| 휴장일 캘린더 | Holiday Calendar | 거래소 휴장일 목록. KRX가 연간 공표 | KRX 공시 기반 연간 캘린더 갱신 배치 필요, 결제일·만기일 계산에 적용 |
| 중앙청산소 | CCP (Central Counterparty) | 매수·매도 당사자 사이에 끼어들어 결제를 보증하는 기관. 국내 = KRX | KRX CCP 보증 범위 내 거래는 상대방 신용리스크 대신 CCP 리스크 관리 |
| 결제보증 | Settlement Guarantee | CCP가 회원의 결제불이행 시 대신 결제를 이행하는 제도 | 결제보증기금 분담금 관리, 불이행 시 KRX 통보 수신 처리 |
| 결제보증기금 | Settlement Guarantee Fund | 결제불이행 대비 CCP가 적립하는 공동 기금 | 분담금 납부·환급 이벤트 회계 처리 자동화 |
| 증거금 | Margin | 결제이행을 보증하기 위해 예치하는 담보 자산 | 개시증거금·유지증거금 구분 관리, 실시간 평가 필요 |
| 개시증거금 | Initial Margin | 포지션 개설 시 최초로 납부하는 증거금 | 종목별 증거금률 × 계약금액으로 계산 |
| 유지증거금 | Maintenance Margin | 포지션 유지에 필요한 최소 증거금 수준 | 평가손익 반영 후 유지증거금 미달 시 마진콜 발동 |
| 변동증거금 | Variation Margin | 일일정산으로 발생한 손익을 다음날 주고받는 현금 | 양수면 수취, 음수면 납부. 방향 부호 일관성 주의 |
| 순결제 | Net Settlement | 차감 후 순포지션에 대해서만 증권·대금을 결제하는 방식 | KRX 차감 결과 파일 수신 후 순결제 내역 생성 |
| 총액결제 | Gross Settlement | 차감 없이 개별 거래마다 전액 결제하는 방식 | 채권 OTC 거래 등에 적용. 건별 결제 추적 필요 |
| 결제계좌 | Settlement Account | 결제 대금이 입출금되는 지정 계좌 | 계좌 매핑 테이블 관리, 오입금 방지 검증 로직 |
| 예탁잔고 | Custody Balance | KSD에 예탁된 증권의 수량 | KSD 잔고 조회 API 호출 결과와 사내 원장 일치 여부 일일 검증 |
| 대체 | Transfer (Book-entry) | 예탁계좌 간 증권을 실물 이동 없이 장부상으로 이전 | KSD 계좌 대체 지시 전문 생성·송신, 완료 응답 수신 후 잔고 반영 |
| 반환 | Return / Redelivery | 빌린 증권을 대여자에게 되돌려주는 것(대차 맥락) | 반환 수량·일자 관리, 대차잔고 감소 처리 |
| 실시간총액결제 | RTGS (Real-Time Gross Settlement) | 건별로 즉시 결제하는 방식. BOK-Wire+ | 대금 결제 시 BOK-Wire+ 전문 연동 |
| 결제주기 | Settlement Cycle | 체결에서 결제까지 걸리는 영업일 수(T+N) | 상품유형별 사이클 코드 테이블 관리 |
| 내부결제 | Internal Settlement | 동일 증권사 내 매수·매도 고객 간 결제를 내부에서 처리 | 내부결제 식별 후 KRX 제출 대상에서 제외하는 로직 필요 |
| 결제지시 | Settlement Instruction | KSD에 증권 인도·대금 지급을 요청하는 전문 | KSD SAFE+ 전문 규격 준수, 필드 길이·코드 검증 |
| 결제확인 | Settlement Confirmation | 결제 완료 후 KSD로부터 수신하는 확인 통보 | 확인 수신 시점에 내부 상태를 settled로 업데이트 |
| 장내거래 | On-Exchange Trade | KRX 시장에서 체결된 거래. CCP 청산 적용 | KRX 체결 데이터 수신 채널(FIX/파일) 구분 |
| 장외거래 | OTC Trade | 거래소 밖에서 상대방과 직접 체결한 거래 | CCP 미적용. 상대방 신용리스크 직접 관리 필요 |
| 청산참가자 | Clearing Member | KRX CCP에 직접 참가하여 청산 의무를 지는 기관 | 참가자 코드 관리, 청산 의무 이행 모니터링 |
| 결제참가자 | Settlement Participant | KSD에 직접 참가하여 결제 의무를 지는 기관 | KSD 참가자 코드와 내부 기관 코드 매핑 테이블 필요 |

## 11. 신용·대주·대차·공매도

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 신용거래 | Margin Trading | 증권사로부터 자금(융자) 또는 주식(대주)을 빌려 매매하는 거래 | 신용 계좌와 일반 계좌 분리, 신용 잔고 별도 원장 관리 |
| 신용융자 | Margin Loan | 증권사가 고객에게 주식 매수 자금을 빌려주는 것 | 융자 원금·이자·담보 연계 관리, 만기 일정 알림 |
| 신용대주 | Margin Short (Stock Lending for Short) | 증권사가 고객에게 공매도 목적으로 주식을 빌려주는 것 | 대주 수량·종목·상환 기한 관리, 대주 가능 잔고 실시간 조회 |
| 미수거래 | Unsettled Credit Purchase | 결제대금 부족 상태에서 매수하고 결제일까지 자금을 마련하는 거래 | 미수 발생 시 결제일(T+2) 전 자금 입금 여부 모니터링, 미입금 시 반대매매 |
| 반대매매 | Forced Sale / Liquidation | 담보비율 부족 또는 미수 미결제 시 증권사가 강제로 매도하는 것 | 반대매매 대상 선정 배치, 시장가 주문 자동 생성, 실행 결과 고객 통보 |
| 담보 | Collateral | 신용거래 시 차입금 상환을 보증하기 위해 제공하는 자산(현금·주식 등) | 담보 자산별 평가율 적용, 실시간 평가액 산출 |
| 담보비율 | Collateral Ratio | 담보평가액 ÷ 융자금액 × 100(%). 증권사별 기준 상이 | 실시간 담보비율 계산 및 임계값 모니터링 배치 |
| 담보유지비율 | Maintenance Margin Ratio | 신용거래 유지에 필요한 최소 담보비율(통상 140%) | 비율 하회 시 마진콜 발송 이벤트 트리거 |
| 최소담보유지비율 | Minimum Maintenance Margin Ratio | 법적·내규상 허용되는 가장 낮은 담보유지비율 | 규제 기준(금융투자업규정) 반영, 변경 시 파라미터 갱신 |
| 마진콜 | Margin Call | 담보비율이 유지비율 이하로 떨어졌을 때 추가 담보 납입을 요구하는 통보 | 발송 채널(SMS/이메일/앱 푸시) 멀티채널 구현, 발송 이력 저장 |
| 추가담보 | Additional Collateral | 마진콜에 응하여 추가로 납입하는 담보 자산 | 납입 확인 후 담보비율 재계산, 마진콜 해소 상태 업데이트 |
| 만기 | Maturity / Expiry | 신용융자·대주의 상환 기한. 통상 최장 6~12개월(증권사별 상이) | 만기일 D-N 알림 배치, 만기 미상환 시 자동 반대매매 트리거 |
| 신용이자 | Credit Interest | 신용융자에 대해 고객이 부담하는 이자 | 일별 이자 계산(원금 × 일이자율), 누적 이자 원장 관리 |
| 융자이자율 | Loan Interest Rate | 신용융자에 적용되는 연이자율(구간별·기간별 차등 가능) | 이자율 구간 테이블 관리, 기간 초과 시 자동 구간 변경 |
| 대주 | Stock Lending (for Short Selling) | 증권사가 고객에게 공매도용 주식을 대여하는 것 | 대주 가능 종목·수량 풀 관리, 대주 잔고 실시간 차감 |
| 대주상환 | Stock Return (Short Covering) | 빌린 주식을 매수하여 증권사에 반환하는 것 | 상환 시 대주 잔고 복원, 상환 이자 정산 처리 |
| 대차거래 | Securities Borrowing and Lending (SBL) | 기관·법인 간 유가증권을 대여·차입하는 거래 | KSD 대차 시스템 연계, 대차 계약 원장 별도 관리 |
| 대차잔고 | SBL Balance / Short Interest | 현재 차입되어 있는 주식의 총 수량 | 종목별 대차잔고 집계, KRX/KSD 공시 데이터와 대사 |
| 차입 | Borrowing | 주식을 빌리는 행위(대차의 차입자 입장) | 차입 계약 체결일·만기일·수량·종목·수수료 관리 |
| 공매도 | Short Selling | 보유하지 않은 주식을 빌려서 매도하는 것 | 공매도 주문 시 차입 확인 선행, 무차입 공매도 방지 검증 |
| 차입공매도 | Covered Short Selling | 주식을 빌린 후 매도하는 합법적 공매도 | 차입 수량 ≥ 공매도 수량 검증 로직 필수 |
| 무차입공매도 | Naked Short Selling | 차입 없이 보유하지 않은 주식을 매도하는 것(불법) | 주문 접수 시 차입 확인 강제화, 위반 시 주문 거부 및 로그 기록 |
| 업틱룰 | Uptick Rule | 공매도 호가를 직전 체결가 이상으로만 제출하도록 제한하는 규칙 | 공매도 주문 호가 검증: 제출 호가 ≥ 직전 체결가 |
| 공매도 과열종목 | Short Selling Overheated Stock | KRX가 공매도 비중·증가율 기준으로 지정하는 종목. 지정 시 당일 공매도 금지 | KRX 과열종목 지정 공시 수신 후 해당 종목 공매도 주문 차단 |
| 대주풀 | Lending Pool | 증권사가 대주용으로 보유·확보한 주식의 집합 | 풀 수량 실시간 관리, 대주 신청 시 풀 차감 및 잔량 조회 API 제공 |
| 담보부족 | Collateral Deficiency | 담보평가액이 필요 담보 수준에 미달하는 상태 | 부족액 산출 후 마진콜 발송, 미납 시 반대매매 스케줄 등록 |
| 담보징구 | Collateral Call | 담보부족 해소를 위해 추가 담보 납입을 요청하는 행위 | 징구 통보 채널 및 응답 기한 관리 |
| 로스컷 | Loss Cut | 손실이 일정 수준 초과 시 포지션을 강제 청산하는 리스크 관리 행위 | 로스컷 기준(손실률/금액) 파라미터화, 실시간 P&L 모니터링 연동 |
| 반대매매 예정 | Scheduled Forced Sale | 마진콜 미응 등으로 다음 영업일 반대매매가 예정된 상태 | 반대매매 예정 플래그 설정, 고객 사전 통보 필수 |
| 신용한도 | Credit Limit | 고객에게 허용된 신용융자·대주의 최대 금액 또는 수량 | 고객별 신용한도 테이블 관리, 주문 시 잔여 한도 실시간 검증 |
| 신용공여한도 | Credit Extension Limit | 증권사 전체 또는 특정 고객군에 대한 신용공여 총량 규제 | 법적 한도(자기자본 대비 비율) 준수 모니터링, 초과 시 신규 신용 거부 |
| 담보주식 | Pledged Stock | 신용거래 담보로 제공된 주식 | 담보 주식 처분 제한 플래그, 담보 평가가격 실시간 갱신 |
| 담보현금 | Cash Collateral | 신용거래 담보로 제공된 현금 | 담보 현금 별도 계좌 관리, 출금 제한 처리 |
| 증거금율 | Margin Rate | 신용융자 매수 시 고객이 자기 자금으로 납부해야 하는 비율(통상 40~50%) | 종목·등급별 증거금율 테이블 관리, 주문 가능 금액 산출에 적용 |
| 반대매매 기준가 | Forced Sale Reference Price | 반대매매 시 적용하는 기준 가격(통상 하한가 또는 시장가) | 반대매매 주문 유형(시장가/하한가 지정) 설정 파라미터 관리 |
| 융자만기연장 | Loan Extension | 만기 도래 신용융자의 상환 기한을 연장하는 것 | 연장 가능 여부·횟수 검증, 연장 후 만기일 갱신 및 이자율 재적용 |
| SBL 수수료 | SBL Fee | 대차거래에서 차입자가 대여자에게 지급하는 수수료(연율) | 일별 수수료 계산(차입금액 × 연율 / 365), 정산 처리 |
| 공매도잔고 보고 | Short Position Reporting | 일정 비율 이상 공매도 잔고 보유 시 KRX에 보고 의무 | 종목별 공매도잔고 비율 산출, 0.5% 초과 시 T+1 보고 자동화 |
| 상환 | Repayment | 신용융자 원금을 증권사에 갚는 것 | 상환 시 담보 해제, 이자 정산, 잔여 융자 원장 업데이트 |
| 자동상환 | Auto Repayment | 만기 또는 반대매매 실행 후 융자금을 매도 대금으로 자동 차감하는 것 | 매도 대금 수령 이벤트에 자동 상환 로직 연결 |

## 12. 기업행위(Corporate Action)·권리

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 기업행위 | Corporate Action (CA) | 발행 회사가 주식 수·자본 구조·주주 권리 등을 변경하는 일체의 행위 | CA 유형별 처리 파이프라인 설계, ISIN 변경 이벤트 연동 |
| 배당 | Dividend | 회사가 이익의 일부를 주주에게 분배하는 것 | 배당기준일·지급일·주당 배당금 관리 테이블 |
| 현금배당 | Cash Dividend | 현금으로 지급하는 배당 | 배당금 = 보유 수량 × 주당 배당금. 세금(배당소득세 15.4%) 원천징수 처리 |
| 주식배당 | Stock Dividend | 현금 대신 신주를 교부하는 배당 | 신주 수량 = 보유 수량 × 배당비율. 단수주 처리 정책 필요 |
| 배당락 | Ex-Dividend | 배당기준일 이후 배당권리가 소멸되어 주가가 배당만큼 하락하는 현상 | 배당락일 전일까지 보유자만 배당 대상. 락 처리 날짜 관리 |
| 배당기준일 | Record Date (Dividend) | 배당을 받을 주주를 확정하는 날(T+2 결제 고려 → 실질 매수는 기준일 D-2까지) | 기준일 장마감 후 주주 명부 스냅샷 생성 |
| 배당지급일 | Dividend Payment Date | 배당금이 실제로 주주 계좌에 입금되는 날 | 지급일 배치에서 개인별 배당금 계산 후 입금 처리 |
| 권리락 | Ex-Rights | 유·무상증자 등 신주인수권리가 소멸되는 기준일 이후 상태 | 권리락 조정가 산출 공식 적용(유상: 권리락가 = (시가 × 구주 + 발행가 × 신주)/(구주+신주)) |
| 권리기준일 | Record Date (Rights) | 신주인수권·무상주 배정 기준이 되는 날 | 기준일 보유 수량 기준 배정. 기준일 스냅샷 잔고와 연결 |
| 무상증자 | Bonus Issue / Stock Split (KR식) | 이익잉여금·자본잉여금을 재원으로 기존 주주에게 무상으로 신주 배정 | 배정비율 × 보유 수량 = 신주 수량. 단수주 처리(절사·매도) 정책 |
| 유상증자 | Rights Offering / Capital Increase | 신주를 발행하여 주주 또는 외부 투자자에게 유상으로 청약시키는 것 | 신주인수권 청약 기간·발행가·청약 비율 관리, 청약 주문 처리 |
| 신주인수권 | Rights (Subscription Right) | 유상증자 시 기존 주주가 우선적으로 신주를 인수할 수 있는 권리 | 권리 수량 배정, 청약·실권(미청약) 처리, 권리락일 계산 |
| 신주인수권증서 | Rights Certificate | 신주인수권을 유가증권화한 증서. 시장에서 매매 가능 | ISIN 별도 부여, 단기 매매 가능 기간 관리 |
| 주식분할 | Stock Split / 액면분할 | 주식의 액면가를 낮춰 주식 수를 늘리는 것(예: 1주→5주) | 분할비율 적용 후 수량 증가·단가 조정. ISIN 변경 여부 확인 |
| 주식병합 | Reverse Stock Split | 여러 주식을 합쳐 주식 수를 줄이는 것(예: 5주→1주) | 병합비율 적용, 단수주 처리, 권리락일 관리 |
| 감자 | Capital Reduction | 자본금을 줄이는 것. 유상감자(주주에게 환급)와 무상감자(결손 보전)로 구분 | 감자비율 적용 후 수량 감소 처리, 유상감자 시 환급금 입금 |
| 유상감자 | Paid-in Capital Reduction | 주주에게 대가를 지급하고 주식을 소각하는 감자 | 소각 대금 계산 및 지급 처리 |
| 무상감자 | Gratuitous Capital Reduction | 대가 없이 결손금 보전 목적으로 주식을 소각하는 감자 | 수량만 감소. 주주 손실 발생 이벤트 안내 |
| 합병 | Merger | 두 회사 이상이 하나로 합쳐지는 것 | 합병비율에 따른 존속법인 주식 교부, 소멸법인 ISIN 말소 처리 |
| 분할 | Corporate Split | 하나의 회사를 둘 이상으로 나누는 것(인적분할/물적분할) | 분할 신설법인 ISIN 생성, 지분 비율에 따른 신주 배정 |
| 액면변경 | Par Value Change | 주식 액면가를 변경하는 것(분할·병합과 연동) | 액면가 마스터 데이터 업데이트, 종목 기본 정보 갱신 |
| 주식교환 | Stock Swap | 완전 자회사 전환 등을 위해 자회사 주주의 주식을 모회사 주식으로 교환 | 교환비율 적용, 구주 ISIN 말소·신주 배정 처리 |
| 공개매수 | TOB (Tender Offer / Take-Over Bid) | 불특정 다수 주주로부터 일정 가격에 주식을 공개적으로 매수하는 것 | 공개매수 청약 주문 유형 별도 처리, 청약 기간·가격·수량 관리 |
| 주주명부폐쇄 | Close of Shareholders Register | 기준일 전후 일정 기간 명의개서를 정지하는 것 | 폐쇄 기간 중 명의개서 요청 차단 로직 |
| 기준일 | Record Date | 주주 권리(배당·의결권 등)를 확정하는 날 | 기준일 전일까지 결제 완료된 보유자만 권리 부여. T+2 고려 매수 마감일 안내 |
| 명의개서 | Transfer Registration | 주주명부상 명의를 새 소유자로 변경 등록하는 것 | KSD 명의개서 대행 시스템 연동, 기준일 전 완료 여부 추적 |
| 전환사채 | CB (Convertible Bond) | 일정 조건에 따라 주식으로 전환할 수 있는 사채 | 전환 청구 이벤트 수신 후 사채 소각·신주 발행 처리 |
| 신주인수권부사채 | BW (Bond with Warrant) | 신주인수권이 부여된 사채. 권리 행사 후에도 사채는 존속 | 워런트 행사 시 신주 발행 처리, 사채 원금은 별도 존속 |
| 교환사채 | EB (Exchangeable Bond) | 발행사가 보유한 타사 주식으로 교환할 수 있는 사채 | 교환 청구 시 보유 타사 주식 인도 처리 |
| 스톡옵션 | Stock Option | 임직원에게 부여하는 자사주 매수 선택권 | 행사 조건(기간·가격) 관리, 행사 시 신주 발행 또는 자사주 교부 처리 |
| 자사주매입 | Share Buyback / Treasury Stock Purchase | 회사가 자기 주식을 시장에서 매입하는 것 | 매입 수량·금액·기간 공시 정보 수신, 자사주 계정 별도 관리 |
| 자사주소각 | Share Cancellation | 매입한 자사주를 소각하여 발행 주식 수를 줄이는 것 | 소각 처리 후 발행주식총수 감소, 주당 가치 변동 안내 |
| 상장폐지 | Delisting | 주식이 거래소 시장에서 상장이 취소되는 것 | 상장폐지 공시 수신 후 해당 종목 거래 차단, 정리매매 기간 설정 |
| 정리매매 | Delisting Trading Period | 상장폐지 확정 후 투자자 보호를 위해 7 영업일간 허용되는 매매 | 정리매매 기간·가격제한폭(±100%) 적용, 기간 종료 후 종목 비활성화 |
| ISIN | ISIN (International Securities Identification Number) | 유가증권을 국제적으로 식별하는 12자리 코드(한국: KR로 시작) | CA 후 ISIN 변경 시 구 ISIN → 신 ISIN 매핑 테이블 관리, 잔고 이관 처리 |
| 종목코드 | Issue Code / Stock Code | KRX가 부여하는 6자리 숫자 종목 식별 코드 | ISIN과 종목코드 양방향 매핑 테이블, CA 후 코드 변경 이력 관리 |
| 권리내역 | Rights Details / Entitlement Detail | 기업행위로 발생하는 주주 권리의 구체적 내용(배당금·신주 수량 등) | CA 이벤트별 권리 내역 원장 관리, 고객별 예상 수령 내역 조회 API |
| 단수주 | Fractional Share | CA 처리 후 발생하는 1주 미만의 잔여 주식 | 단수주 처리 정책(절사·매도·현금 환급) 사전 설정, 처리 결과 안내 |
| 배당소득세 | Dividend Withholding Tax | 배당금에 부과되는 소득세(국내 15.4% = 소득세 14% + 지방세 1.4%) | 거주자·비거주자·법인 구분 세율 적용, 원천징수 후 순배당금 입금 |
| 배당재투자 | DRIP (Dividend Reinvestment Plan) | 현금배당금을 자동으로 해당 주식 매수에 재투자하는 프로그램 | 배당 입금 이벤트 트리거 → 시장가 매수 주문 자동 생성 |
| 신주배정기준일 | New Share Allotment Record Date | 무상증자·주식배당 등 신주를 배정받을 주주를 확정하는 날 | 기준일 잔고 스냅샷 기반 배정 수량 계산 |
| 청약일 | Subscription Date | 유상증자 청약을 접수하는 날(보통 2일간) | 청약 기간 내 주문 접수, 기간 외 거부 로직 |
| 청약증거금 | Subscription Deposit | 유상증자 청약 시 납입하는 예치금(청약금액의 100% 선납) | 청약증거금 임시 계정 관리, 배정 확정 후 차액 환불 처리 |
| 실권주 | Unsubscribed Shares | 유상증자에서 주주가 청약하지 않은 신주 | 실권주 일반 공모 또는 주관사 인수 처리 |
| 조정주가 | Adjusted Price | CA(분할·배당·증자) 반영 후 비교 가능하도록 수정한 과거 주가 | 수정 주가 계수 계산 및 차트 데이터 소급 적용 |
| 주식매수청구권 | Appraisal Right | 합병·분할 등 중요 의사결정에 반대하는 주주가 회사에 주식 매수를 청구할 수 있는 권리 | 청구 기간·가격 관리, 청구 접수 후 대금 지급 처리 |
| 우선주 | Preferred Stock | 배당 또는 잔여재산 분배에서 보통주보다 우선권을 갖는 주식 | 우선주 ISIN 별도 관리, 배당 계산 시 우선 배당률 적용 |
| 전환우선주 | Convertible Preferred Stock | 일정 조건 충족 시 보통주로 전환 가능한 우선주 | 전환 조건·비율 관리, 전환 청구 이벤트 처리 |
| 신주상장일 | New Share Listing Date | CA로 발행된 신주가 거래소에 상장되어 매매 가능해지는 날 | 상장일에 맞춰 종목 마스터 활성화, 신주 ISIN 등록 |
| 의결권 | Voting Right | 주주총회에서 의사결정에 참여할 수 있는 권리 | 주주총회 기준일 보유 수량 기반 의결권 수 산출 |
| 주주총회 | General Meeting of Shareholders (AGM/EGM) | 주주가 회사 중요 사항을 의결하는 최고 의사결정 기관 | 총회 소집 공시 수신, 전자투표 시스템 연동(e-Voting) |
| 공시 | Disclosure | 기업이 투자자에게 중요 정보를 공개하는 것. CA 공시는 DART·KRX KIND 통해 발표 | DART/KIND API 구독, CA 관련 공시 자동 파싱 및 이벤트 생성 |
| 이익잉여금 | Retained Earnings | 당기순이익 중 배당 후 사내에 유보된 금액. 무상증자 재원 | 무상증자 시 이익잉여금 감소·자본금 증가 회계 처리 확인 |
| 자본잉여금 | Capital Surplus | 액면 초과 발행 등으로 발생한 잉여금. 무상증자·감자 재원 | 재원 구분에 따라 무상주 과세 여부 상이(자본잉여금 → 비과세) |
| 완전자회사 | Wholly-Owned Subsidiary | 주식교환·주식이전으로 특정 회사가 100% 자회사가 되는 것 | 주식교환 완료 후 구주 ISIN 상장폐지, 모회사 주식 배정 처리 |
| 현물출자 | In-Kind Contribution | 현금 대신 자산(주식·부동산 등)을 납입하여 신주를 취득하는 것 | 현물출자 대상 자산 평가 및 신주 수량 계산 |
| 분기배당 | Quarterly Dividend | 연간 배당을 4회로 나눠 지급하는 것 | 분기별 배당기준일·지급일 4회 관리, 연간 배당 합산 정보 제공 |
| 중간배당 | Interim Dividend | 사업연도 중간에 지급하는 배당(보통 반기) | 중간배당 기준일 별도 관리, 결산배당과 합산 시 이중 계상 방지 |
## 13. 파생상품

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 선물 | Futures | 기초자산을 미래 특정 시점에 미리 정한 가격으로 매매하기로 약정한 계약 | 종목코드 체계 별도(예: 101W9000), 만기월 코드 매핑 필요 |
| 옵션 | Options | 기초자산을 특정 가격에 사거나 팔 수 있는 권리를 매매하는 계약 | 콜/풋 구분 플래그, 행사가격·만기 조합으로 종목 식별 |
| 콜옵션 | Call Option | 기초자산을 행사가격에 살 수 있는 권리 | 권리구분코드 C |
| 풋옵션 | Put Option | 기초자산을 행사가격에 팔 수 있는 권리 | 권리구분코드 P |
| 행사가격 | Strike Price / Exercise Price | 옵션 계약에서 매매하기로 정한 기초자산의 가격 | DECIMAL(10,2) 저장, 옵션 종목코드에 포함(KRX 5자리) |
| 만기 | Expiration / Maturity | 파생상품 계약이 종료되는 날짜 | 최종거래일·만기결제일 별도 관리, calendar 테이블 참조 |
| 최종거래일 | Last Trading Day (LTD) | 해당 파생상품을 마지막으로 거래할 수 있는 날 | 만기일 ≠ 최종거래일인 경우 존재, 공휴일 처리 주의 |
| 결제월 | Contract Month / Delivery Month | 파생상품 계약의 만기가 속하는 월 | 종목코드 내 월 코드(F=1월…Z=12월) 디코딩 로직 |
| 미결제약정 | Open Interest (OI) | 아직 청산되지 않은 미결제 계약 수 | 매수 OI + 매도 OI를 단방향 합산, 일 마감 후 집계 |
| 일일정산 | Mark-to-Market (MTM) | 파생상품 포지션을 매일 종가(정산가)로 평가해 손익을 일별 정산하는 제도 | 일일정산금 = (당일정산가 - 전일정산가) × 승수 × 계약수, 증거금 계좌 자동 입출 |
| 정산가 | Settlement Price | 일일 또는 최종 정산에 사용하는 기준 가격(거래소 공표) | 장 마감 후 거래소 수신, 별도 settlement_price 테이블 |
| 개시증거금 | Initial Margin | 선물·옵션 포지션 개설 시 최초로 예치해야 하는 증거금 | SPAN 또는 KRX 기준 계산, 계좌 잔고 실시간 비교 |
| 유지증거금 | Maintenance Margin | 포지션 유지에 필요한 최소 증거금 수준 | 유지증거금 미달 시 반대매매 또는 마진콜 발동 트리거 |
| 추가증거금 | Margin Call / Variation Margin | 증거금 부족 시 추가로 납부해야 하는 금액 | 부족액 알림 + 납부 기한(T+1 정오 등) 스케줄러 구현 |
| 레버리지 | Leverage | 소액의 증거금으로 대규모 계약을 통제할 수 있는 효과 | 계약금액 = 지수×승수×계약수, 레버리지 비율 노출 |
| 베이시스 | Basis | 현물 가격과 선물 가격의 차이 (현물가 - 선물가) | 실시간 현물·선물 가격 차 계산, 차익거래 신호로 활용 |
| 콘탱고 | Contango | 선물가격이 현물가격보다 높은 상태 (정상 시장) | basis < 0 판별 로직 |
| 백워데이션 | Backwardation | 선물가격이 현물가격보다 낮은 상태 | basis > 0 판별 로직, 배당·보관비용 모델 반영 |
| 롤오버 | Rollover | 만기가 도래하는 선물 포지션을 다음 결제월 선물로 교체하는 거래 | 만기 N영업일 전 자동 롤오버 알림, 스프레드 비용 계산 |
| 청산 | Liquidation / Close-out | 보유 포지션을 반대매매로 소멸시키는 것 | 매수 포지션 → 매도 주문 생성, 포지션 수량 0으로 갱신 |
| 그릭스 | Greeks | 옵션 가격의 민감도 지표 집합 | 실시간 계산 또는 거래소 수신, DECIMAL(18,8) 정밀도 |
| 델타 | Delta (Δ) | 기초자산 가격 1단위 변동 시 옵션 가격 변동량 | 콜: 0~1, 풋: -1~0; 포지션 델타 합산으로 헤지 비율 산출 |
| 감마 | Gamma (Γ) | 기초자산 가격 변동에 따른 델타의 변화율 | 델타 헤지 재조정 빈도 판단 기준 |
| 세타 | Theta (Θ) | 시간 경과에 따른 옵션 가격 감소분 (시간가치 소멸) | 일 단위 음수값, 만기 근접 시 급격히 증가 |
| 베가 | Vega (ν) | 내재변동성 1% 변동 시 옵션 가격 변동량 | IV 변동 시나리오 분석에 활용 |
| 로 | Rho (ρ) | 금리 1% 변동 시 옵션 가격 변동량 | 금리 민감도; 단기옵션에서 상대적으로 중요도 낮음 |
| 내재변동성 | Implied Volatility (IV) | 현재 옵션 시장가격에서 역산한 변동성 | Black-Scholes 역산(뉴턴-랩슨 반복법), VKOSPI 지수와 연계 |
| 역사적변동성 | Historical Volatility (HV) | 과거 기초자산 수익률의 표준편차 기반 변동성 | 일간 로그수익률 표준편차 × √252 |
| 내가격 | In-The-Money (ITM) | 즉시 행사 시 이익이 발생하는 상태 | 콜: 현물 > 행사가, 풋: 현물 < 행사가 |
| 등가격 | At-The-Money (ATM) | 현물가격과 행사가격이 거의 같은 상태 | 현물 ≈ 행사가 (±일정 범위) |
| 외가격 | Out-of-The-Money (OTM) | 즉시 행사 시 손실이 발생하는 상태 | 콜: 현물 < 행사가, 풋: 현물 > 행사가 |
| 프리미엄 | Premium | 옵션 매수자가 매도자에게 지급하는 옵션 가격 | 체결 시 매수자 즉시 출금, 매도자 입금; 증거금과 별도 |
| 행사/배정 | Exercise / Assignment | 옵션 권리 행사(매수자) 및 의무 이행(매도자) | 자동행사 기준(ITM 0.01pt 이상), 배정 알고리즘 구현 |
| 자동행사 | Auto Exercise | 만기 시 ITM 옵션을 자동으로 행사하는 제도 | KRX 기준: 최종결제가 대비 ITM 여부 자동 판단 |
| 최종결제 | Final Settlement | 만기 시 현금으로 차액 정산하는 방식 (코스피200 등) | 최종결제가 = 만기일 기초자산 특별호가 산출 가격 |
| 코스피200 선물 | KOSPI200 Futures | 코스피200 지수를 기초자산으로 하는 선물 | 승수 250,000원, 호가단위 0.05pt, 종목코드 101xxxxx |
| 코스피200 옵션 | KOSPI200 Options | 코스피200 지수를 기초자산으로 하는 옵션 | 승수 100,000원, 주간/야간 구분, 위클리 병행 |
| 미니선물 | Mini Futures | 코스피200 미니선물 (승수 소형화) | 승수 50,000원, 유동성 낮아 슬리피지 고려 |
| 주식선물 | Single Stock Futures | 개별 주식을 기초자산으로 하는 선물 | 기초주식 종목코드 + 결제월 조합, 배당 조정 로직 필요 |
| 위클리옵션 | Weekly Options | 만기가 매주 돌아오는 단기 옵션 | 만기 구분 코드(W) 추가, 만기 달력 주 단위 관리 |
| ELW | Equity Linked Warrant (ELW) | 특정 주식·지수를 기초자산으로 발행된 주식워런트증권 | 주식 시장에서 거래, 전환비율·패리티 계산 필요 |
| 전환비율 | Conversion Ratio | ELW 1증권당 기초자산 수량 | ELW 이론가 = (내재가치 / 전환비율) |
| 양매수 | Long Straddle | 동일 행사가의 콜·풋 동시 매수 전략 | 두 레그 동시 주문, 포지션 합산 그릭스 계산 |
| 양매도 | Short Straddle | 동일 행사가의 콜·풋 동시 매도 전략 | 증거금 = 콜증거금 + 풋증거금 (KRX SPAN 방식) |
| 스프레드거래 | Spread Trading | 두 개 이상의 선물·옵션 포지션을 동시에 보유하는 전략 | 캘린더/수직/대각선 스프레드 종류별 증거금 우대 |
| 헤지 | Hedge | 보유 포지션의 리스크를 반대 포지션으로 상쇄하는 거래 | 헤지 비율 = 포트폴리오 베타 × 포트폴리오가치 / (선물가×승수) |
| 차익거래 | Arbitrage | 현물·선물 간 가격 차이를 이용한 무위험 수익 추구 | 프로그램매매 알고리즘, 베이시스 모니터링 실시간 필요 |
| 프로그램매매 | Program Trading | 사전 설정 알고리즘에 따라 자동 실행되는 대량 바스켓 거래 | 바스켓 구성 종목별 수량 사전 계산, 동시 다종목 주문 |
| VKOSPI | VKOSPI | 코스피200 옵션 기반 변동성 지수 (한국판 VIX) | 거래소 산출값 수신, 시장 공포 지표로 활용 |
| 승수 | Multiplier | 지수 1포인트당 계약 금액 단위 | 계약금액 = 가격 × 승수, 상품별 테이블 관리 |
| 호가단위 | Tick Size | 주문 가격의 최소 변동 단위 | 파생 상품별 호가단위 다름, 가격 유효성 검증에 사용 |
| 틱가치 | Tick Value | 호가단위 1틱 변동 시 손익 금액 | 틱가치 = 호가단위 × 승수 |
| SPAN 증거금 | SPAN Margin | 포트폴리오 기반 표준화 증거금 산출 방식 | CME SPAN 또는 KRX 자체 방식, 시나리오 16개 계산 |
| 포지션 한도 | Position Limit | 특정 파생상품에서 보유 가능한 최대 계약 수 | 투자자별·계좌별 한도 체크, 초과 주문 거부 로직 |
| 결제불이행 | Default | 증거금 부족 시 의무 이행 불능 상태 | 강제청산(반대매매) 자동 트리거, 이벤트 로그 필수 |
| 반대매매 | Forced Liquidation | 증거금 부족 계좌의 포지션을 강제로 청산하는 거래 | 우선순위(손실 큰 포지션부터) 알고리즘, 시장가 주문 |
| 차익매수/차익매도 | Buy/Sell Program | 현선 베이시스를 이용한 차익거래의 현물 바스켓 매수·매도 | 프로그램매매 공시 기준 초과 시 거래소 보고 의무 |

## 14. 채권(Bond)

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 채권 | Bond | 발행자가 일정 기간 후 원금과 이자를 지급하기로 약속한 채무증권 | 종목코드(ISIN 12자리), 발행정보·이자정보 별도 테이블 |
| 국채 | Government Bond | 국가(정부)가 발행한 채권 | 신용위험 최저, 기준금리 벤치마크, KTB·국고채 구분 |
| 국고채 | Korea Treasury Bond (KTB) | 기획재정부가 발행하는 대표적인 국가채무증권 | 3/5/10/20/30/50년 만기 구분, 물가연동국채(KTBi) 별도 |
| 통화안정채권 | Monetary Stabilization Bond (MSB) | 한국은행이 통화량 조절을 위해 발행하는 채권 | 통안채, 91일~2년 만기, 할인채 방식 |
| 지방채 | Municipal Bond | 지방자치단체가 발행한 채권 | 발행기관 코드 관리, 상대적으로 유동성 낮음 |
| 특수채 | Special Bond | 특별법에 근거한 공공기관이 발행한 채권 | 산업금융채권, 주택금융채권 등 발행기관별 코드 |
| 회사채 | Corporate Bond | 민간 기업이 자금 조달을 위해 발행한 채권 | 신용등급 필수, 공모/사모 구분 플래그 |
| 표면금리 | Coupon Rate | 채권 액면가 대비 연간 이자 지급 비율 | DECIMAL(8,4) 저장, 이자계산: 액면 × 쿠폰 × 일수/365 |
| 쿠폰 | Coupon | 채권 이자 지급금액 또는 이자율 | 지급 스케줄 테이블, 지급일 달력 처리(공휴일 이월) |
| 액면가 | Face Value / Par Value | 채권 만기 시 상환되는 원금 금액 | 통상 10,000원(장외) 또는 1,000원 단위 |
| 만기수익률 | Yield to Maturity (YTM) | 현재 가격으로 채권을 매입해 만기까지 보유 시 실현되는 연수익률 | IRR 방식 반복 계산(뉴턴-랩슨), 호가 기준 실시간 산출 |
| 듀레이션 | Duration | 채권의 실질적인 평균 회수 기간; 금리 민감도 지표 | 맥콜리 듀레이션 = Σ(t × PV(CF_t)) / 채권가격 |
| 수정듀레이션 | Modified Duration | 금리 1% 변동 시 채권 가격 변동률 | 수정듀레이션 = 맥콜리듀레이션 / (1 + YTM/m) |
| 컨벡시티 | Convexity | 듀레이션의 금리 변동에 따른 비선형 보정 지표 | 2차 근사항: ½ × Convexity × (ΔY)² |
| 신용등급 | Credit Rating | 채무 상환 능력에 대한 신용평가사의 평가 등급 | AAA~D, NR 포함; 등급변동 이력 관리, 발행사·이슈별 구분 |
| 스프레드 | Spread | 국채 대비 회사채 등의 금리 차이 (신용·유동성 보상) | 스프레드 = 채권 YTM - 동일만기 국채 YTM |
| 발행시장 | Primary Market | 채권이 최초로 발행·판매되는 시장 | 청약·배정·납입 프로세스, 발행정보 수신(금융결제원) |
| 유통시장 | Secondary Market | 이미 발행된 채권이 거래되는 시장 | 장내(거래소)/장외(딜러) 구분, 체결 방식 상이 |
| 할인채 | Discount Bond | 이자를 만기에 한 번에 지급하며 액면가 이하로 발행되는 채권 | 가격 = 액면 / (1 + r×d/365), 이표 없음 |
| 이표채 | Coupon Bond | 정기적으로 이자를 지급하는 일반적인 채권 | 이자지급주기(3/6/12개월), 스케줄 생성 로직 |
| 복리채 | Compound Interest Bond | 이자를 재투자하여 만기에 원리금을 일괄 지급하는 채권 | 복리계산: 액면 × (1+r/m)^(n×m) |
| 경과이자 | Accrued Interest | 마지막 이자 지급일부터 결제일까지 발생한 이자 | 경과이자 = 액면 × 쿠폰 × 경과일수/이자기산일수 |
| 더티프라이스 | Dirty Price / Full Price | 경과이자를 포함한 실제 결제 가격 | 실제 지급금액, 더티 = 클린 + 경과이자 |
| 클린프라이스 | Clean Price / Flat Price | 경과이자를 제외한 채권 가격 (호가에 표시) | 시장 호가 기준, 단가 표시에 사용 |
| 채권단가 | Bond Price (per unit) | 액면 10,000원 기준 채권 1단위 가격 | 소수점 이하 5자리 이상 정밀도 필요 |
| 매매수익률 | Transaction Yield | 실제 거래에 적용되는 수익률 | YTM 기반, 거래소/장외 적용 방식 상이 |
| RP | Repurchase Agreement (RP) | 채권을 일정 기간 후 환매 조건으로 매도하는 단기 자금 거래 | 매도RP(자금조달)·매수RP(운용) 구분, 만기·이율 관리 |
| 장내채권 | Exchange-traded Bond | 거래소를 통해 매매되는 채권 | 주문 방식 주식과 유사, 소액 단위 거래 가능 |
| 장외채권 | OTC Bond | 금융기관 간 또는 딜러 통해 장외에서 거래되는 채권 | FRN, 협의 가격, K-Bond·Bondweb 등 시스템 연동 |
| 신종자본증권 | Hybrid Capital Securities | 자본과 부채의 성격을 동시에 가진 증권 (영구채 포함) | 상각형/비상각형, 조건부자본증권(CoCo) 포함 |
| 전환사채 | Convertible Bond (CB) | 보유자가 일정 조건에서 발행 회사 주식으로 전환할 수 있는 사채 | 전환가격·전환비율 관리, 전환 이벤트 처리 로직 |
| 신주인수권부사채 | Bond with Warrant (BW) | 신주 인수권이 부여된 사채 | 행사가·행사기간 관리, 워런트 분리 여부 플래그 |
| 교환사채 | Exchangeable Bond (EB) | 발행사가 보유한 타 회사 주식으로 교환 가능한 사채 | 교환 대상 종목코드 연결, 교환비율 관리 |
| 조건부자본증권 | Contingent Convertible Bond (CoCo) | 특정 손실 이벤트 발생 시 주식으로 전환되거나 원금이 상각되는 채권 | 트리거 이벤트(BIS비율 등) 모니터링 연동 |
| 채권평가 | Bond Valuation | 채권 평가 기관이 산출한 채권의 시가 | KIS채권평가·한국채권평가 등 외부 데이터 수신 |
| 채권평가사 | Bond Pricing Agency | 채권 시가를 평가하는 전문 기관 | KIS, KBP, NICE, FnGuide 4개사 평균값 활용 |
| 물가연동채 | Inflation-Linked Bond (ILB) | 원금 및 이자가 물가(CPI)에 연동되어 변하는 채권 | CPI 지수 수신, 조정원금 계산 로직 별도 |
| 변동금리채 | Floating Rate Note (FRN) | 기준금리(CD·SOFR 등)에 연동하여 이자가 변하는 채권 | 금리 리셋 주기·스프레드 관리, 기준금리 자동 적용 |
| 후순위채 | Subordinated Bond | 일반 채권보다 변제 우선순위가 낮은 채권 | 신용등급 산정 시 후순위 반영, 상각 조건 모니터링 |
| 이자지급주기 | Payment Frequency | 쿠폰 이자를 지급하는 빈도 | 월/3개월/6개월/1년 코드값, 스케줄 자동 생성 |
| 발행일 | Issue Date | 채권이 최초 발행된 날짜 | 경과이자 기산일 기준, ISIN 등록일과 구분 |
| 기산일 | Interest Accrual Date | 이자 계산 시작 기준일 | 발행일 또는 전 이자지급일 |
| 일수계산방식 | Day Count Convention | 이자 계산 시 적용하는 일수 산정 방식 | Actual/365, Actual/360, 30/360 등 채권별 상이 |
| 수익률커브 | Yield Curve | 만기별 채권 수익률을 연결한 곡선 | 국고채 3/5/10/20/30년물 기준, 보간법(Nelson-Siegel) |
| 역수익률커브 | Inverted Yield Curve | 단기 금리가 장기 금리보다 높은 비정상적 수익률 곡선 | 경기침체 선행지표, 모니터링 알림 |
| 채권지수 | Bond Index | 채권 시장 전체 또는 특정 세그먼트 성과를 나타내는 지수 | KRX 채권지수, 외부 데이터 수신 및 수익률 비교 |

## 15. 펀드·ETF·구조화상품

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 펀드 | Fund | 다수 투자자의 자금을 모아 전문 운용사가 운용하는 집합투자기구 | 펀드코드(금투협 표준코드), 클래스별 기준가 별도 관리 |

| 보수 | Management Fee | 펀드 운용·판매·수탁에 대한 대가로 순자산에서 차감하는 비용 | 운용/판매/수탁/일반사무 보수 합산, 일할 계산 |
| TER | Total Expense Ratio | 펀드의 총 비용 비율 | 연간 총비용 / 평균 순자산, 투자설명서 공시값 |
| 클래스 | Share Class | 동일 펀드 내 보수·환매조건이 다른 투자 단위 | A/B/C/Ce/E/S 클래스 코드, 클래스별 기준가 독립 계산 |

| ETN | Exchange Traded Note | 증권사가 발행하는 지수 연계 만기부 파생결합증권 | 발행사 신용위험 존재, 만기·조기상환 조건 관리 |
| 추적오차 | Tracking Error | ETF 수익률과 기초지수 수익률 간 차이 | TE = std(펀드수익률 - 지수수익률), 일별 모니터링 |
| 괴리율 | Premium/Discount Rate | ETF 시장가격과 NAV 간 차이율 | 괴리율 = (시장가 - NAV) / NAV × 100 |
| LP | Liquidity Provider | ETF·ELW의 유동성을 공급하는 증권사 | 호가 스프레드·수량 의무 준수 모니터링, LP 주문 구분 |
| 설정단위 | Creation Unit (CU) | ETF 설정·환매가 가능한 최소 좌수 단위 | 통상 50,000~100,000좌, 기관 전용 |
| 실시간NAV | iNAV (Indicative NAV) | ETF 편입 자산 기준으로 실시간 산출하는 추정 NAV | 15초~1분 주기 산출, 편입 종목 실시간 가격 합산 |

| 계정계 | Core Banking/Brokerage System | 계좌 개설·유지·잔고·이체 등 핵심 금융 업무를 처리하는 시스템 | 분리 배포, 변경 최소화, 고가용성(HA) 필수 |
| 정보계 | Information System / MIS | 계정계 데이터를 복제하여 조회·분석·리포팅에 활용하는 시스템 | CDC(Change Data Capture)로 실시간 복제, 계정계와 DB 분리 |
| 채널계 | Channel System | MTS·HTS·WTS 등 고객 접점 채널을 처리하는 레이어 | 세션 관리, 부하분산, 프로토콜 변환(WebSocket ↔ 전문) |
| 대외계 | External System | 거래소·청산소·은행 등 외부 기관 연계 시스템 | 전문 규격 준수, 회선 이중화, 장애시 큐 보관 |
| FEP | Front-End Processor | 대외 기관과의 통신 전처리 장비/소프트웨어. 프로토콜 변환·분배 담당 | TCP 소켓 풀 관리, 재접속 자동화, 순번(SEQ) 관리 |
| 주문중계 | Order Routing | 고객 주문을 채널계→주문처리→거래소로 전달하는 중계 레이어 | 주문 유효성 검증, 거부 코드 표준화, 메시지 추적 ID |
| 매매체결시스템 | Trade Matching System / Order Matching Engine | 매수·매도 주문을 가격·시간 우선으로 매칭하는 엔진 | 가격-시간 우선 알고리즘, 락-프리 큐, 나노초 타임스탬프 |
| 시세분배시스템 | Market Data Distribution System | 거래소에서 수신한 시세를 내부 시스템 및 고객 단말에 분배 | 멀티캐스트 UDP, 패킷 손실 복구 로직, 지연 모니터링 |
| 체결엔진 | Matching Engine | 매매체결시스템 내 핵심 매칭 로직 컴포넌트 | 오더북(Order Book) 자료구조, 가격-수량 잔량 관리 |
| FIX 프로토콜 | Financial Information eXchange | 증권 거래 메시지 교환을 위한 국제 표준 프로토콜(태그=값 형식) | FIX 4.2/4.4/5.0 버전 구분, 세션 레이어(Logon/Heartbeat/Logout) 구현 |
| 거래소접속 | Exchange Connectivity | 증권사가 거래소(KRX 등)와 주문·시세를 주고받기 위한 전용 회선·프로토콜 | KRX EXTURE+ API 규격, 이중화 접속, 접수 확인(ACK) 처리 |
| MCA | Multicast Address / Market Connection Agent | 거래소 시세를 멀티캐스트로 수신하는 주소 또는 에이전트 | IGMP 그룹 가입, NIC 멀티캐스트 필터 설정 |
| 멀티캐스트 | Multicast | 하나의 패킷을 다수 수신자에게 동시 전송하는 네트워크 방식 | UDP 기반, 패킷 순서 보장 없음 → 시퀀스 번호 기반 손실 감지 |
| 시세피드 | Market Data Feed | 거래소로부터 실시간 호가·체결 데이터를 수신하는 데이터 스트림 | 지연 측정(feed latency), 스냅샷+증분(delta) 방식 구분 |
| 마켓데이터 | Market Data | 종목별 현재가·호가·거래량·지수 등 시장 정보 전반 | 캐싱 레이어(Redis 등), TTL 관리, 구독 모델 구현 |
| 배치 | Batch | 대량 데이터를 일괄 처리하는 프로그램 또는 작업 방식 | 멱등성 보장, 재처리 체크포인트, 실행 이력 로깅 |
| 온라인배치 | Online Batch | 시스템 가동 중(영업 시간 내) 실행되는 배치 | 온라인 트랜잭션과 충돌 방지, 잠금 범위 최소화 |
| 일중배치 | Intraday Batch | 장중 주기적으로 실행되는 배치(수수료 계산, 담보 평가 등) | 정해진 시각에 자동 기동, 실패 시 알림 및 수동 재기동 절차 |
| 마감배치 | End-of-Day Batch / EOD Batch | 장 마감 후 정산·잔고 확정·손익 계산 등을 처리하는 배치 | 순서 의존성 DAG 관리, 단계별 체크포인트, 롤백 절차 |
| 정산배치 | Settlement Batch | 거래 결제(T+2 등) 및 대금·증권 이체를 처리하는 배치 | DVP 원칙 적용, KSD 연계 전문 생성, 미결제 예외 처리 |
| EAI/ESB | Enterprise Application Integration / Enterprise Service Bus | 이기종 시스템 간 메시지·데이터 연계 미들웨어 | 어댑터 패턴, 메시지 변환, 라우팅 규칙 관리 |
| MCI | Multi-Channel Integration | 다채널 고객 요청을 단일 창구로 통합·중계하는 미들웨어 | 채널별 인증 토큰 통합, 요청 추적 ID 부여 |
| 메시지큐 | Message Queue / MQ | 비동기 메시지 전달을 위한 큐 미들웨어(IBM MQ, RabbitMQ, Kafka 등) | 메시지 영속성, 소비자 그룹, 데드레터 큐(DLQ) 설계 |
| 전문(電文) | Financial Message / Telegram | 금융 시스템 간 통신에 사용하는 정형화된 데이터 구조 | 고정길이/가변길이 구분, 헤더(거래코드·길이·일련번호) 필수 |
| 거래코드 | Transaction Code / Tran Code | 각 업무 처리를 식별하는 고유 코드 | 코드 체계 문서화, 신규 코드 채번 정책 수립 |
| 표준전문 | Standard Message Format | 금융결제원·KRX 등 기관이 정의한 공통 전문 규격 | 버전 관리, 기관별 규격 차이 추상화 레이어 필요 |
| 처리계 | Processing System | 업무 로직을 실행하는 서버·프로세스 군 | 무상태(stateless) 설계로 수평 확장, 처리 결과 로그 보존 |
| 채번 | Sequence Number Generation / ID Generation | 주문번호·접수번호 등 유일 식별자를 순차 발급하는 메커니즘 | DB 시퀀스 또는 분산 ID(Snowflake 등), 중복 방지 보장 |
| TP모니터 | Transaction Processing Monitor | 대량 트랜잭션을 안정적으로 처리하는 미들웨어(Tuxedo, WebLogic Tuxedo 등) | 서비스 등록·라우팅, 커넥션 풀, 트랜잭션 XA 지원 |
| 메인프레임 | Mainframe | IBM Z 시리즈 등 대형 호스트 컴퓨터. 구형 증권사 계정계에 사용 | COBOL/PL1 레거시 연계, EBCDIC ↔ ASCII 변환 |
| 차세대시스템 | Next-Generation System | 노후 레거시 시스템을 최신 기술 스택으로 전면 재구축하는 프로젝트 | 데이터 마이그레이션 정합성, 이중화 운영(병행) 기간 관리 |
| 가상계좌 | Virtual Account | 실계좌와 매핑되는 임시 계좌번호. 입금 자동 식별에 사용 | 채번·매핑 테이블 관리, 재사용 정책, 은행 펌뱅킹 연동 |
| 인증서 | Certificate / PKI Certificate | 공개키 기반 전자 인증서(공동인증서, 금융인증서 등) | X.509 파싱, 유효기간 검증, OCSP/CRL 폐기 확인 |
| OTP | One-Time Password | 일회용 비밀번호. 이체·로그인 2차 인증에 사용 | TOTP(RFC 6238) 또는 하드웨어 OTP, 시간 동기화 오차 허용 |
| 펌뱅킹 | Firm Banking | 증권사가 은행과 전용 회선으로 직접 연결해 자금 이체를 처리하는 방식 | 이체 결과 비동기 수신, 오류 코드 매핑, 이중 이체 방지 |
| 인터넷뱅킹 연동 | Internet Banking Integration | 고객 은행 계좌와 증권 계좌 간 실시간 입출금 연동 | 오픈뱅킹 API(금융결제원), 토큰 기반 인증, 한도 관리 |
| 소켓통신 | Socket Communication | TCP/UDP 소켓 기반 저지연 금융 전문 통신 | 접속 풀(connection pool), Keep-Alive, 재접속 로직 |
| 세션관리 | Session Management | 고객 로그인 상태와 거래 컨텍스트를 유지·관리하는 메커니즘 | 세션 만료·갱신, 동시 로그인 제어, 세션 고정 공격 방어 |
| 업무분리 | Separation of Duties / SoD | 한 사람이 주문·승인·실행을 모두 처리하지 못하도록 권한 분리 | RBAC 설계, 결재 워크플로우 로직 |
| 시스템이중화 | High Availability / HA Clustering | 장애 시 자동 전환을 위한 Active-Active 또는 Active-Standby 구성 | 페일오버 시간 측정, 데이터 동기화 지연 허용 기준 |
| 재해복구 | Disaster Recovery / DR | 주 센터 장애 시 백업 센터에서 서비스를 복구하는 체계 | RTO/RPO 정의, DR 훈련 주기, 데이터 복제 방식(동기/비동기) |
| 거래량제한 | Throttle / Rate Limiting | 단위 시간당 처리 가능한 주문·거래 건수를 제한하는 기능 | Token Bucket 알고리즘, 초과 시 거부 vs 큐잉 정책 |
| 계좌계층 | Account Hierarchy | 법인 마스터 계좌 → 하위 계좌 등 계층형 계좌 구조 | 트리 구조 DB 설계, 상위 잔고 집계 쿼리 최적화 |
| 수수료체계 | Commission Structure | 거래 유형·채널·고객 등급별 수수료율 적용 체계 | 복잡한 조건 분기, 수수료 계산 로직 별도 서비스 분리 |
| 공시연동 | Disclosure Integration | DART 등 공시 시스템과 연동하여 공시 데이터를 수신·표시 | OpenAPI 폴링 또는 웹훅, 공시 유형 코드 매핑 |
| 약정 | Contract / Commitment | 매매 체결 후 결제 전 상태의 거래 약속 | 약정잔고 vs 실잔고 이중 관리, 결제일 도래 시 자동 전환 |

## 18. 리스크·컴플라이언스·규제

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|------------|
| 리스크관리 | Risk Management | 투자 손실·유동성·운영 위험을 식별·측정·통제하는 전반적 체계 | 리스크 지표 실시간 계산 엔진, 임계치 알림 파이프라인 |
| 사전위험관리 | Pre-Trade Risk Management | 주문 제출 전 한도·자격·포지션을 검증하는 통제 | 주문 처리 경로의 동기 검증, 지연 최소화(< 1ms 목표) |
| 사후관리 | Post-Trade Management | 체결 후 포지션·손익·마진 현황을 모니터링·관리 | 배치 또는 실시간 포지션 재계산, 이상 탐지 룰 엔진 |
| 주문한도 | Order Limit | 단일 주문의 수량·금액 상한 | 고객 등급·계좌 유형별 다단계 한도 테이블, 초과 시 즉시 거부 |
| 포지션한도 | Position Limit | 특정 종목 또는 전체 포트폴리오의 보유 가능 최대 수량·금액 | 실시간 포지션 집계, 한도 초과 주문 자동 차단 |
| 손실한도 | Loss Limit / Stop-Loss Limit | 허용 가능한 최대 손실액. 초과 시 추가 매매 중단 | 미실현손익 포함 계산, 임계치 비율(80%·90%·100%) 단계별 경보 |
| fat-finger 방지 | Fat-Finger Prevention | 입력 오류로 인한 이상 대량·고가 주문을 사전 차단하는 기능 | 직전가 대비 가격·수량 편차율 임계치, 2차 확인 UX |
| 이상거래탐지 | Fraud Detection System / FDS | 비정상적 거래 패턴을 실시간으로 탐지·차단하는 시스템 | 룰 기반 + ML 모델 결합, 탐지 결과 심사 워크플로우 |
| 시세조종 | Market Manipulation | 인위적으로 주가를 움직이기 위한 허수주문·통정매매 등 불공정 행위 | 반복 주문·취소 패턴, 급격한 가격 변동 모니터링 |
| 미공개정보이용 | Insider Trading | 내부 정보를 이용한 불법 매매 | 임직원 계좌 특이 거래 모니터링, 공시일 전후 매매 패턴 분석 |
| 부정거래 | Fraudulent Trading | 허위 사실 유포·위계를 이용한 거래 조작 행위 | 거래 패턴 이상 탐지, 외부 뉴스 연계 분석 |
| 자전거래 | Wash Trade | 동일 주체가 매수·매도를 동시에 하여 거래량을 인위적으로 늘리는 행위 | 동일 계좌·연계 계좌 간 체결 감지 로직 |
| 통정매매 | Matched Order / Pre-Arranged Trade | 사전 합의 하에 특정 상대방과 매매하여 가격을 조작하는 행위 | 매수·매도 계좌 IP·단말 연관 분석 |
| 불공정거래 | Unfair Trading | 시세조종·내부자거래·부정거래 등 자본시장법 위반 거래 총칭 | 모니터링 대상 이벤트 목록 관리, 금감원 보고 연계 |
| AML | Anti-Money Laundering | 자금세탁방지. 불법 자금의 금융 시스템 유입·세탁을 차단하는 체계 | STR·CTR 자동 생성, 고위험 국가·거래 필터링 |
| KYC | Know Your Customer | 고객 신원 확인 절차. 계좌 개설 시 신분증 확인·위험도 평가 | 신분증 OCR, 생체인증, 위험등급 산출 모델 |
| CDD | Customer Due Diligence | 고객 실사. 거래 목적·자금 출처 등을 확인하는 기본 절차 | 계좌 개설 시 설문 + 주기적 재확인 워크플로우 |
| EDD | Enhanced Due Diligence | 강화 고객 실사. 고위험 고객(PEP 등)에 대한 심층 검증 | PEP 리스트 연동, 추가 서류 요청 자동화 |
| 의심거래보고 | Suspicious Transaction Report / STR | 자금세탁 의심 거래를 금융정보분석원(KoFIU)에 보고하는 제도 | 보고서 자동 생성, 암호화 전송, 보고 이력 보존 |
| 고액현금거래보고 | Currency Transaction Report / CTR | 1거래일 2천만 원 이상 현금 거래를 KoFIU에 보고하는 의무 | 일별 집계, 임계치 초과 시 자동 보고서 생성 |
| 자본시장법 | Financial Investment Services and Capital Markets Act | 금융투자상품 거래·업무를 규율하는 한국 기본법 | 조항별 의무(설명의무·적합성 등) 기능 구현 목록 관리 |
| 전자금융거래법 | Electronic Financial Transactions Act | 전자 금융 서비스의 안전성·이용자 보호를 규정하는 법률 | 접근매체 관리, 부정 거래 배상 책임 조항 준수 |
| 개인정보보호법 | Personal Information Protection Act / PIPA | 개인정보 수집·처리·보관에 관한 규제 법률 | 수집 동의 관리, 파기 스케줄러, 처리 로그 보존 |
| 신용정보법 | Credit Information Use and Protection Act | 신용정보의 수집·활용·보호를 규정하는 법률 | 신용정보 제공 동의, 제3자 공유 이력 관리 |
| 내부통제 | Internal Control | 법규 준수·오류 방지를 위한 내부 정책·절차·시스템 체계 | 통제 항목 체크리스트 자동화, 이슈 추적 시스템 |
| 준법감시 | Compliance Monitoring | 법규·내규 준수 여부를 모니터링하고 위반을 예방하는 활동 | 모니터링 룰셋 관리, 위반 이벤트 에스컬레이션 |
| 감사추적 | Audit Trail | 시스템 내 모든 행위(조회·수정·삭제)를 기록하여 추적 가능하게 하는 로그 | 불변 로그 저장소(append-only), 타임스탬프 위변조 방지 |
| 적합성원칙 | Suitability Principle | 투자자 성향에 맞는 금융상품만 권유해야 하는 의무 | 상품 위험등급 vs 투자자 등급 매칭 로직, 부적합 상품 차단 |
| 적정성원칙 | Appropriateness Principle | 비권유 상품도 투자자에게 적정한지 확인해야 하는 의무 | 파생·레버리지 상품 구매 전 적정성 팝업·동의 플로우 |
| 설명의무 | Duty to Explain | 금융상품 위험·특성을 고객에게 충분히 설명해야 하는 법적 의무 | 설명 동의 이력 저장(시각·방법·내용 포함) |
| 투자자성향 | Investor Risk Profile | 고객의 투자 목적·기간·위험 감수 수준을 분류한 등급(안정형~공격형) | 설문 점수 기반 등급 산출, 주기적 재설문 알림 |
| 전문투자자 | Professional Investor | 법인·고액자산가 등 일정 요건을 충족한 고객. 규제 적용 완화 | 자격 확인 서류 검증, 만료일 관리, 자동 전환 로직 |
| 일반투자자 | Retail Investor | 전문투자자가 아닌 일반 개인 고객. 강화된 보호 규제 적용 | 파생상품 기본예탁금, 레버리지 한도 제한 로직 |
| 예금자보호 | Deposit Insurance | 금융기관 파산 시 예금(증권은 위탁재산 분리)을 보호하는 제도 | 위탁재산 분리 보관(신탁), 고객 안내 문구 표시 |
| 매매내역보존 | Trade Record Retention | 주문·체결·취소 내역을 법정 기간(5~10년) 보존하는 의무 | 아카이빙 파이프라인, 불변 스토리지, 검색 인덱스 유지 |
| 녹취 | Call Recording | 투자 권유 상담 전화를 의무 녹음하는 제도 | 녹취 파일 암호화 저장, 고객 요청 시 제공 API |
| 자기자본 규제 | Capital Adequacy Regulation | 금융투자업자가 유지해야 할 최소 자기자본 비율 규제 | 영업용순자본비율(NCR) 일별 계산, 규제 임계치 경보 |
| NCR | Net Capital Ratio | 영업용순자본비율. 금융투자업자 재무 건전성 지표 | 자산·부채 일별 평가, 금감원 보고 자동화 |
| 레버리지 한도 | Leverage Limit | 투자자가 신용·대출로 활용할 수 있는 최대 배율 제한 | 계좌별 현재 레버리지 실시간 계산, 초과 주문 차단 |
| 공매도 규제 | Short-Selling Regulation | 무차입 공매도 금지 등 공매도 관련 법규 준수 체계 | 차입 확인 전 공매도 주문 차단, 잔고 실시간 검증 |
| 대주(貸株) | Securities Lending | 주식을 빌려주는 서비스. 공매도·헤지에 활용 | 대주 한도·기간·금리 관리, 반환 스케줄러 |
| 신용공여 | Credit Extension | 투자자에게 주식 구매 자금을 대출하는 서비스(신용융자 등) | 담보 비율 실시간 계산, 반대매매 자동 실행 로직 |
| 반대매매 | Forced Liquidation | 신용·대출 담보 부족 시 증권사가 강제로 보유 주식을 매도하는 것 | 담보율 임계치 모니터링, 반대매매 대상 선정 알고리즘 |
| 추가증거금 | Margin Call | 파생상품 증거금 부족 시 추가 납부를 요구하는 통지 | 장중 실시간 증거금 계산, SMS/Push 알림, 납부 기한 관리 |
| 규제보고 | Regulatory Reporting | 감독기관(금감원·KRX 등)에 정기 또는 수시로 보고하는 의무 | 보고서 양식 버전 관리, 자동 생성·전송, 접수 확인 처리 |
| 제재 필터링 | Sanctions Screening | OFAC·UN 제재 리스트 대조 고객·거래 필터링 | 리스트 일 1회 이상 업데이트, 이름 퍼지 매칭 알고리즘 |
| PEP | Politically Exposed Person | 정치적 주요 인물. AML 고위험군으로 EDD 대상 | PEP 데이터베이스 연동, 가족·측근 연계 확인 |

| 기술적분석 | Technical Analysis | 과거 가격·거래량 데이터의 패턴으로 미래 가격을 예측하는 분석 방법 | OHLCV 시계열 저장, 지표 계산 라이브러리(ta-lib 등) 연동 |

| 지지 | Support | 가격 하락 시 수요가 증가하여 반등이 예상되는 가격대 | 과거 저점 클러스터링으로 지지선 자동 탐지 |

| 알파 | Alpha | 시장 수익률(벤치마크) 초과 수익률. 운용 능력 지표 | CAPM 기반 계산, 벤치마크 지수 선택 파라미터화 |
| 외국인 수급 | Foreign Investor Flow | 외국인 투자자의 순매수·순매도 동향 | KRX 투자자별 매매 데이터 파싱, 누적 집계 |
| 기관 수급 | Institutional Investor Flow | 연기금·자산운용사·보험사 등 기관의 매매 동향 | 기관 세부 유형(투신·연기금 등) 구분 집계 |
| 개인 수급 | Retail Investor Flow | 개인 투자자의 매매 동향 | 전체 거래량 - 외국인 - 기관 으로 산출 |
| 프로그램매매 | Program Trading | 사전 설정 알고리즘에 따라 자동 실행되는 대량 매매 | 차익거래·비차익거래 구분, KRX 실시간 공표 데이터 연동 |
| 공매도잔고 | Short Interest | 공매도 후 아직 상환되지 않은 주식 수량 | 일별 KRX 공시 데이터 수신, 잔고율(%) 계산 |
| 신용잔고 | Credit Balance / Margin Loan Balance | 신용융자로 매수 후 아직 상환하지 않은 금액 또는 주식 수 | 증권사 자체 집계, 전체 시장 신용잔고는 금투협 데이터 |
| 대차잔고 | Securities Lending Balance | 대차거래(기관 간 주식 대여)의 미상환 잔량 | KSD 대차거래 데이터 연동 |
| 52주 신고가/신저가 | 52-Week High/Low | 최근 52주 중 최고가·최저가 | 롤링 252거래일 최대·최솟값 계산, 돌파 이벤트 알림 |
| 괴리율 | Premium/Discount Rate | ETF·ELW 등의 시장가격과 이론가(NAV) 간 차이 비율 | (시장가-NAV)/NAV × 100, 실시간 NAV 계산 필요 |
| 수익률 곡선 | Yield Curve | 만기별 채권 수익률을 연결한 곡선. 경기 예측에 활용 | 국고채 만기별 금리 데이터 수집, 역전 감지 알림 |
| 듀레이션 | Duration | 채권 현금흐름의 가중 평균 만기. 금리 민감도 척도 | 맥컬레이·수정 듀레이션 계산, 금리 시나리오 분석 |
| 컨벡시티 | Convexity | 금리 변화에 대한 채권 가격 반응의 곡률. 듀레이션 보완 지표 | 2차 테일러 근사로 가격 변화 예측 정확도 향상 |
| 델타 | Delta (δ) | 기초자산 가격 변화에 따른 옵션 가격 변화율 | Black-Scholes 편미분, 실시간 그릭스 계산 엔진 |
| 감마 | Gamma (Γ) | 델타의 변화율. 옵션 가격 곡률 | 델타 헤징 빈도 결정에 활용 |
| 세타 | Theta (Θ) | 시간 경과에 따른 옵션 가치 감소(시간가치 소멸) | 일별 세타 계산, 만기 접근 시 급격한 증가 모니터링 |
| 베가 | Vega (ν) | 내재변동성 변화에 따른 옵션 가격 변화율 | 변동성 서피스(Volatility Surface) 구축 |
| 로 | Rho (ρ) | 금리 변화에 따른 옵션 가격 변화율 | 금리 인상기 풋옵션 영향 분석 |
