# 55. 시세·체결시스템 심화 용어집

증권사 백엔드 개발자가 시세(마켓데이터) 및 매매체결 시스템을 구현·운영할 때 반드시 숙지해야 할 핵심 용어를 도메인별로 망라한 심화 용어집이다. 시스템 설계·개발 관점의 포인트를 함께 정리하여 실무 적용성을 높였다.

---

## 시세 데이터 종류

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 현재가 | Current Price | 직전 체결이 이루어진 가격. 가장 기본적인 시세값 | 체결 이벤트 발생 시마다 갱신; 초기값은 기준가 또는 전일 종가 |
| 체결가 | Execution Price / Trade Price | 매수·매도 주문이 실제로 매칭된 개별 가격 | 체결 틱 단위로 저장; 현재가와 혼동하지 않도록 필드 분리 |
| 시가 | Open Price | 당일 장 개시 후 첫 체결가 | 동시호가 종료 시 확정; 장전·장후 시세와 별도 관리 |
| 고가 | High Price | 당일 체결가 중 최고값 | 실시간 업데이트; 체결 발생마다 max() 비교 갱신 |
| 저가 | Low Price | 당일 체결가 중 최저값 | 실시간 업데이트; 체결 발생마다 min() 비교 갱신 |
| 종가 | Close Price | 당일 장 마감 기준 최종 체결가 또는 단일가 | 장마감 후 확정; 다음날 기준가 산출에 사용 |
| 예상체결가 | Expected Execution Price | 동시호가 구간에서 시장 참여자 주문을 기반으로 산출한 예측 체결가 | 호가 변동 시마다 재계산; 실제 체결가와 다를 수 있음을 UI에 명시 |
| 호가 | Quote / Bid-Ask | 매수(bid)·매도(ask) 의사를 나타낸 가격·수량 | 매수10호가·매도10호가 배열 구조; 호가 단위(tick size) 검증 필수 |
| 호가잔량 | Quote Residual / Order Queue | 특정 호가 가격대에 쌓인 미체결 주문 수량 합계 | orderbook depth 구조; 취소·정정 시 즉시 감산 처리 |
| 거래량 | Volume | 당일 체결된 주식 수량 합계 | 체결 발생마다 누적 합산; 부분체결도 포함 |
| 거래대금 | Turnover / Trade Value | 당일 체결된 금액 합계(수량 × 체결가) | 거래량과 별도 필드; 대형주·소형주 비교 지표로 활용 |
| 누적거래량 | Cumulative Volume | 개장 이후 현재까지 체결 수량의 누적합 | 장중 스냅샷 요청 시 해당 시점 누적값 반환 |
| 전일대비 | Change / Price Change | 현재가 - 전일 종가 | 부호(+/-)와 절대값 모두 저장; 기준가 변경(권리락 등) 시 재산출 |
| 등락률 | Change Rate / % Change | (현재가 - 전일 종가) / 전일 종가 × 100 | 소수점 자릿수 정책 통일; 전일 종가 0일 경우 예외 처리 |
| 기준가 | Base Price / Reference Price | 당일 가격 제한폭 산출의 기준이 되는 가격(통상 전일 종가) | 권리락·배당락 등 이벤트 발생 시 별도 산출 로직 적용 |
| 상한가 | Upper Limit Price | 기준가 대비 허용된 최고 매도 호가 (±30% 등 거래소 정책) | 등록 주문의 가격 범위 서버사이드 검증에 사용 |
| 하한가 | Lower Limit Price | 기준가 대비 허용된 최저 매도 호가 | 상한가와 동일 로직; 기준가 변동 시 재계산 후 배포 |
| 체결강도 | Trade Strength / Buy-Sell Ratio | 체결 수량 중 매수 주도 비율 = 매수체결량 / (매수체결량 + 매도체결량) × 100 | 어떤 쪽이 '주도'인지 판별 알고리즘(호가 기준) 설계 필요 |
| 거래원 | Broker / Member Firm | 체결에 참여한 회원사(증권사) 정보 | 거래소 전문 피드에서 제공; 일반 시세에서는 익명 처리 |
| 투자자별매매동향 | Investor Trend by Type | 개인·기관·외국인 등 투자자 유형별 순매수/매도 집계 | 체결 건별 투자자 구분 태그 필요; 배치 집계 또는 실시간 스트림 처리 |
| 프로그램매매 | Program Trading | 알고리즘·차익거래 목적의 대량 자동화 주문 | 거래소 코드로 구분; 시세 피드에 별도 필드로 제공 |
| 시간외단일가시세 | After-Hours Single Price | 장 종료 후 단일가 방식으로 형성되는 시간외 매매 시세 | 정규 시세와 구분된 별도 채널·필드로 배포 |
| VI발동정보 | Volatility Interruption (VI) | 주가 급변 시 투자 냉각을 위해 2분간 단일가 매매로 전환하는 제도 정보 | 발동·해제 이벤트 실시간 수신; 주문 수락 정책 즉시 변경 |
| 장운영상태 | Market Session Status | 개장 전(Pre-Open), 동시호가, 접속매매, 장 마감 등 거래소 운영 상태 | 상태 변화 이벤트 수신 시 주문 수락 여부·시세 처리 로직 분기 |

## 시세 배포·기술

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 마켓데이터피드 | Market Data Feed | 거래소·데이터벤더로부터 실시간 시세 데이터를 수신하는 스트림 | 수신 프로토콜(UDP 멀티캐스트, TCP) 선택; 핸들러 단일 스레드 권장 |
| 시세분배 | Market Data Distribution | 수신한 원본 시세를 내부 시스템·클라이언트에 전달하는 구조 | 허브-스포크 또는 Pub/Sub 패턴; 팬아웃 부하 설계 필수 |
| 멀티캐스트 | Multicast | 하나의 패킷을 다수 수신자에게 동시 전달하는 UDP 기반 네트워크 방식 | IGMP 그룹 관리; 패킷 손실 탐지 및 재전송 메커니즘 필수 |
| 유니캐스트 | Unicast | 송신자와 수신자 1:1 연결 방식 | 소규모 구독자 또는 재전송 채널에 활용; 연결 수 증가 시 서버 부하 주의 |
| 실시간시세 | Real-Time Quote | 체결·호가 발생 즉시 전달되는 지연 없는 시세 | 지연 측정(latency monitoring) 필수; SLA 수십 밀리초 이내 |
| 지연시세 | Delayed Quote | 실시간 대비 일정 시간(통상 15~20분) 지연 배포되는 시세 | 무료 채널 제공 시 사용; 지연 버퍼 큐 구현 |
| 스냅샷 | Snapshot | 특정 시점의 시세 전체 상태를 한 번에 전달하는 데이터 패킷 | 신규 구독자 연결 시 초기 상태 전송; 이후 델타 업데이트 전환 |
| 증분(델타)업데이트 | Incremental Update / Delta | 직전 상태 대비 변경된 필드만 전송하는 방식 | 전송량 절감; 클라이언트 상태 병합 로직(merge) 구현 필요 |
| 컨플레이션 | Conflation | 전송 지연 시 복수의 이벤트를 최신 상태 하나로 합쳐 전달하는 기법 | HFT 환경에서 불리; 일반 리테일 채널의 부하 절감에 활용 |
| 스로틀링 | Throttling | 초당 최대 메시지 수를 제한해 수신자 과부하를 방지하는 기법 | 종목별·채널별 임계값 설정; 중요 이벤트(VI 등)는 제외 처리 |
| 시세서버 | Quote Server / Market Data Server | 시세 데이터를 수신·가공·분배하는 중앙 서버 | 고가용성(HA) 구성; Active-Standby 또는 Active-Active 이중화 |
| 시세료 | Market Data Fee | 거래소·데이터벤더가 부과하는 시세 데이터 이용 요금 | 사용자 수·용도별 라이선스 관리; 감사 로그 유지 의무 |
| 정규화 | Normalization | 복수 거래소·종목의 이질적 포맷을 통일된 내부 스키마로 변환하는 과정 | 필드 매핑 테이블 관리; 거래소 포맷 변경 시 어댑터 수정 |
| 시퀀스번호 | Sequence Number | 피드 메시지에 부여된 순차 번호로 순서·유실 여부 확인에 사용 | 수신 시 연속성 검사; 갭 발생 시 즉시 재전송 요청 |
| 갭/유실 | Gap / Packet Loss | 시퀀스번호 불연속으로 메시지가 누락된 상태 | 갭 감지 로직 구현; 허용 시간 내 재전송 없으면 스냅샷 재요청 |
| 재전송 | Retransmission | 유실된 메시지를 별도 채널(TCP)로 다시 수신하는 절차 | 재전송 채널 별도 연결 유지; 유실 허용 범위(Gap Threshold) 정책 수립 |
| WebSocket 푸시 | WebSocket Push | 서버가 클라이언트 연결을 유지하며 시세 변경을 능동적으로 전송하는 방식 | 연결 수 제한·하트비트·재연결 로직 구현; 프레임 크기 최적화 |
| 구독 | Subscription | 특정 종목·데이터 유형의 시세 수신을 등록하는 행위 | 구독 목록 서버 메모리 관리; 구독 해제 시 리소스 즉시 반환 |
| FIX 피드 | FIX Market Data | FIX 프로토콜 기반 시세 배포 방식 (Market Data Request/Snapshot/Incremental Refresh) | MsgType=W(스냅샷), X(증분); MDReqID로 구독 세션 추적 |
| UDP 소켓 | UDP Socket | 비연결형 데이터그램 소켓; 멀티캐스트 시세 수신에 주로 사용 | SO_RCVBUF 크기 튜닝; 커널 버퍼 오버플로 모니터링 필수 |

## 차트·집계 데이터

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| OHLCV | Open-High-Low-Close-Volume | 특정 시간 단위의 시가·고가·저가·종가·거래량 집합 | 기본 차트 데이터 단위; 시간대별 집계 파이프라인 설계 |
| 봉(캔들) | Candlestick | OHLCV를 시각화한 차트 요소; 시가·종가·고가·저가를 한 단위로 표현 | 양봉(종가>시가)·음봉(종가<시가) 구분; 색상 정책 프론트와 협의 |
| 분봉 | Minute Bar / Minute Candle | 1분·3분·5분·10분·30분 단위 OHLCV 집계 | 틱 데이터로부터 실시간 집계; 장 종료 후 확정값과 대조 검증 |
| 일봉 | Daily Bar / Daily Candle | 1거래일 단위 OHLCV | 거래소 제공 EOD 데이터와 자체 집계값 대조; 수정주가 반영 여부 구분 |
| 주봉 | Weekly Bar | 1주(月~金) 단위 OHLCV | 주 경계 처리(공휴일 포함); 마지막 거래일 종가를 주 종가로 설정 |
| 월봉 | Monthly Bar | 1개월 단위 OHLCV | 월말 거래일 종가를 월 종가로 설정; 윤년·말일 처리 주의 |
| 틱 데이터 | Tick Data | 체결 1건마다 기록된 원시 데이터(시각·가격·수량) | 가장 세밀한 단위; 저장 용량 대용량; 압축·파티셔닝 전략 필수 |
| 체결 데이터 | Trade Data | 체결 발생 시 생성되는 데이터(체결가·수량·체결번호·시각 등) | 틱 데이터의 부분집합; 거래 재현(replay)에 사용 |
| 거래량가중평균 | VWAP (Volume Weighted Average Price) | 체결가 × 체결량의 합 / 총 체결량 | 알고리즘 주문 기준가; 장중 실시간 VWAP와 일별 VWAP 구분 |
| 시간가중평균 | TWAP (Time Weighted Average Price) | 일정 시간 동안 가격의 단순 시간 가중 평균 | 샘플링 간격 정책 결정; VWAP와 함께 집행 알고리즘 분석에 활용 |
| 보정주가(수정주가) | Adjusted Price | 유·무상증자·배당·분할 등 권리 이벤트를 반영하여 소급 조정한 주가 | 조정계수 관리 테이블 필요; 과거 OHLCV 일괄 재계산 배치 |
| 권리락반영 | Ex-Rights Adjustment | 유·무상증자 기준일 이후 주가에 권리락을 반영하는 처리 | 기준가 재산출; 차트 연속성 유지를 위해 과거 데이터도 소급 조정 |
| 차트집계 | Chart Aggregation | 틱·분봉 등 세밀한 데이터를 상위 시간대로 합산하는 과정 | 스트림 집계(Flink, Kafka Streams 등) 활용; 윈도우 경계 처리 주의 |
| 시계열DB | Time-Series DB (TSDB) | 시간 축으로 최적화된 데이터베이스 (InfluxDB, TimescaleDB, kdb+ 등) | 삽입 성능·압축률·쿼리 패턴 검토; 시퀀스 보장 정책 수립 |
| 다운샘플링 | Downsampling | 고해상도 데이터를 저해상도로 집계하여 용량 절감하는 기법 | 보존 정책(Retention Policy) 설정; 집계 함수(OHLCV) 정의 |
| 연속주가 | Continuous Price Series | 상장폐지·합병 등 이벤트 전후의 주가를 하나의 시계열로 이어 붙인 데이터 | 이벤트 매핑 테이블; 복합 종목코드 처리 |
| 가격 단위(호가 단위) | Tick Size | 가격 변동의 최소 단위 (예: 1원, 5원, 10원 등 가격대별 차등 적용) | 가격대별 tick size 테이블 관리; 주문 가격 유효성 검증에 사용 |

## 체결·매매시스템

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 매칭엔진 | Matching Engine | 매수·매도 주문을 가격-시간 우선 원칙으로 매칭하여 체결을 생성하는 핵심 컴포넌트 | 단일 스레드 이벤트루프 설계 권장; 컨텍스트 스위칭 최소화 |
| 주문북 | Order Book | 미체결 매수·매도 주문을 가격별로 정렬·관리하는 자료구조 | 가격 레벨별 큐(FIFO); Red-Black Tree 또는 sorted map 사용 |
| 가격-시간 우선 | Price-Time Priority | 더 유리한 가격, 동일 가격 내에서는 먼저 접수된 주문을 우선 체결하는 원칙 | 주문 접수 타임스탬프 나노초 단위 보존; 서버 시계 동기화(PTP/GPS) |
| 체결 생성 | Trade Generation | 매칭 조건이 충족될 때 체결 레코드를 생성하는 절차 | 체결번호(거래소 채번) 중복 방지; 원자적 트랜잭션 처리 |
| 부분체결 | Partial Fill | 주문 수량 중 일부만 체결된 상태 | 미체결 잔량 주문북 유지; 복수 체결 통보 이벤트 발행 |
| 체결통보 | Execution Report | 체결 발생 시 주문자에게 전송하는 확인 메시지 | FIX MsgType=8; 체결가·수량·주문상태 포함; 순서 보장 전송 |
| FEP | Front-End Processor | 증권사와 거래소 간 주문·체결 메시지를 중계하는 시스템 | 거래소 프로토콜 변환; 세션 이중화·HA 구성 필수 |
| 주문중계 | Order Routing | 투자자 주문을 적절한 거래소·시장으로 전달하는 과정 | 스마트오더라우팅(SOR) 포함; 라우팅 규칙 테이블 관리 |
| FIX | Financial Information eXchange | 금융 거래 메시지 교환 표준 프로토콜 | FIX 4.2/4.4/5.0 버전 혼용; 태그 번호 기반 파싱; 세션 관리(Logon/Heartbeat) |
| 거래소 세션 | Exchange Session | 거래소와의 FIX/독점 프로토콜 연결 단위 | 세션 재연결 자동화; SeqNum 동기화; 갭 복구 절차 |
| 주문순번 | Order Sequence Number / ClOrdID | 증권사가 주문에 부여하는 고유 식별 번호 | 중복 방지 채번 서비스; 정정·취소 시 원주문 ClOrdID 참조 |
| 체결순번 | Trade Sequence Number | 체결 건에 부여된 거래소 또는 내부 일련번호 | 정산 대조·감사에 사용; 거래소 채번과 내부 채번 매핑 테이블 |
| 호가응답 | Quote Acknowledgement | 호가 등록 요청에 대한 거래소의 수락/거부 응답 | 응답 지연 타임아웃 처리; 거부 시 사유 코드 로깅 |
| 주문응답 | Order Acknowledgement | 주문 접수에 대한 거래소의 확인 응답 | 접수 확인 전 상태 = Pending New; 확인 후 = New |
| 거부 | Reject | 거래소 또는 내부 리스크 시스템이 주문을 받아들이지 않는 처리 | 거부 사유 코드(FIX Tag 58, 103) 저장; 고객 통보 및 로깅 |
| 정정/취소 응답 | Replace/Cancel Acknowledgement | 주문 정정·취소 요청에 대한 거래소 확인 또는 거부 응답 | Pending Replace/Cancel 상태 관리; 타임아웃 시 재시도 또는 에러 처리 |
| 시퀀싱 | Sequencing | 메시지 처리 순서를 결정론적으로 보장하는 메커니즘 | 단일 시퀀서 또는 분산 시퀀서(Aeron, LMAX Disruptor 등) 활용 |
| 저지연 | Low-Latency | 주문 접수부터 체결 통보까지 수십 마이크로초~밀리초 수준의 처리 속도 | GC 최소화(off-heap 메모리); busy-spin; CPU 코어 격리; DPDK 활용 |
| 멱등처리 | Idempotent Processing | 동일 주문·메시지를 중복 수신해도 결과가 같도록 보장하는 처리 | 주문 ID 기반 중복 체크; exactly-once 시맨틱 구현 |
| 리스크 게이트 | Risk Gate / Pre-Trade Risk Check | 주문 전달 전 한도·포지션·신용을 검증하는 사전 리스크 관문 | 인메모리 포지션 캐시; 마이크로초 단위 검증 목표 |
| 잔고검증 | Balance Verification | 주문 수량·금액이 보유 잔고·한도를 초과하지 않는지 확인 | 실시간 잔고 캐시 동기화; 동시 주문 시 race condition 방지 |

## 시세 관련 지표·파생

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 실시간지수 | Real-Time Index | 구성 종목 현재가를 가중 평균하여 실시간 산출되는 지수(KOSPI, KOSDAQ 등) | 체결 발생 시마다 지수 재산출; 시가총액 가중 방식 구현 |
| 예상지수 | Expected Index | 동시호가 중 예상체결가 기반으로 산출되는 장전 지수 | 예상체결가 변동 시마다 재산출; 실제 지수와 구분 표시 |
| VKOSPI | Volatility Index of KOSPI | 코스피 옵션 가격을 활용한 내재 변동성 지수(한국판 VIX) | 옵션 시세 피드 별도 수신; 30일 내재 변동성 가중 산출 |
| 업종지수 | Sector Index | 동일 업종 종목들의 시가총액 가중 평균 지수 | 업종 분류 테이블 관리; 종목 편입·편출 이벤트 처리 |
| 시가총액 | Market Capitalization | 현재가 × 상장주식수 | 상장주식수 변경(유·무상증자 등) 시 즉시 갱신 |
| 등락종목수 | Advance-Decline Count | 전일 대비 상승·하락·보합 종목 수 | 전 종목 스캔; 장중 실시간 집계; 시장 너비(breadth) 지표 |
| 거래량순위 | Volume Ranking | 당일 거래량 기준 종목 순위 | 실시간 힙(heap) 또는 정렬 자료구조; 상위 N개 추출 |
| 호가스프레드 | Bid-Ask Spread | 최우선 매도호가 - 최우선 매수호가 | 유동성 지표; 실시간 호가 갱신 시마다 재산출 |
| 시장심도 | Market Depth | 가격 레벨별 매수·매도 잔량의 분포(10호가 이상 계층 구조) | 깊은 호가 데이터 요구 시 별도 피드 구독; Level 2 데이터 |
| 유동성지표 | Liquidity Indicator | 거래량·스프레드·시장심도를 복합 반영한 유동성 측정값 | 복합 산식 정의; 투자자 참고 지표로 제공 |
| 체결빈도 | Trade Frequency | 단위 시간당 체결 발생 건수 | TPS(체결/초) 모니터링; 급등 시 서킷브레이커 트리거 연동 가능 |
| 외국인 보유율 | Foreign Ownership Ratio | 외국인 투자자가 보유한 주식 수 / 상장주식수 | 실시간 집계 어려움; 장 마감 후 배치 산출 + 실시간 추정 병행 |
| 공매도잔고 | Short Selling Balance | 공매도 후 아직 환매수되지 않은 잔고 수량 | 거래소 공매도 피드 별도 수신; 일별 공시 데이터와 대조 |
| 서킷브레이커 | Circuit Breaker (CB) | 지수 또는 개별 종목 가격이 임계치 초과 하락 시 거래를 일시 중단하는 제도 | CB 발동 이벤트 수신 시 주문 접수 즉시 차단; 해제 후 재개 |
| 사이드카 | Sidecar | 선물 가격이 기준 대비 일정폭 급변 시 프로그램 매매를 5분간 정지시키는 제도 | 사이드카 이벤트 플래그 관리; 프로그램 매매 주문 필터링 로직 |
| 공시정보 | Disclosure / Corporate Action | 기업의 중요 경영 정보 공시 (실적, 유상증자, 배당 등) | 공시 피드 또는 API 연동; 가격 영향 이벤트와 시세 연계 처리 |

## 시스템 아키텍처·운영

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 시세 게이트웨이 | Market Data Gateway | 거래소·외부 피드를 수신해 내부 시스템에 공급하는 진입점 컴포넌트 | 이중화 구성; 피드 전환 시 무중단 핸드오버 |
| 인메모리 캐시 | In-Memory Cache | 현재가·호가·지표 등 최신 시세를 RAM에 저장해 빠른 조회를 지원 | Redis Cluster 또는 Hazelcast; TTL 정책; 메모리 사이징 |
| 메시지 큐 | Message Queue | 시세 이벤트를 비동기로 전달하는 브로커(Kafka, RabbitMQ 등) | 파티션 전략(종목별 파티셔닝); 오프셋 관리; 컨슈머 랙 모니터링 |
| 이벤트 소싱 | Event Sourcing | 상태 변경을 이벤트 스트림으로 저장해 언제든 상태를 재구성하는 패턴 | 체결 이벤트 영속화; 리플레이로 일별 집계 재산출 가능 |
| CQRS | Command Query Responsibility Segregation | 쓰기(주문·체결) 와 읽기(시세 조회) 경로를 분리하는 아키텍처 패턴 | 시세 조회 전용 읽기 모델; 체결 발생 시 읽기 모델 갱신 |
| 시계 동기화 | Clock Synchronization | 서버 간 시각 일치를 보장하는 메커니즘(PTP, GPS, NTP) | PTP(IEEE 1588) 서브마이크로초 정밀도; 타임스탬프 감사 |
| 레이턴시 모니터링 | Latency Monitoring | 주문·시세 처리 각 단계의 소요 시간을 측정·추적 | 퍼센타일(P50/P95/P99) 지표; 히스토그램 기반 알림 |
| 핫패스 | Hot Path | 저지연이 요구되는 핵심 처리 경로(주문 접수 → 거래소 전달) | 객체 풀링, GC-free 코딩; 로깅 최소화; 비동기 분기 처리 |
| 콜드패스 | Cold Path | 감사·리포팅 등 지연이 허용되는 보조 처리 경로 | 핫패스에서 비동기로 분리; 별도 스레드 풀 또는 큐 |
| 피드 핸들러 | Feed Handler | 거래소 원본 피드를 파싱·정규화하는 컴포넌트 | 거래소별 전용 구현; 바이너리 파싱 최적화; 제로카피 활용 |
| 드롭카피 | Drop Copy | 체결 내역을 리스크 관리·백오피스 시스템에 실시간 복사 배포 | FIX Drop Copy 세션; 원본 체결과 동기; 리스크 시스템 SLA |
| 종목 마스터 | Instrument Master | 종목코드·명칭·상장주식수·tick size 등 종목 기준 정보 | 거래 개시 전 최신화; 변경 이벤트 배포; 시세 처리 전 조회 |
| 거래시간 스케줄 | Trading Schedule | 개장·동시호가·접속매매·마감 등 시간대별 장 운영 일정 | 공휴일·임시휴장 반영; 자동화 스케줄러 연동; 오버라이드 기능 |
| 감사 로그 | Audit Log | 주문·체결·시세 처리 이력을 추적 가능하도록 기록한 불변 로그 | append-only 저장소; 타임스탬프·사용자 ID·이벤트 유형 포함 |
| 재해복구 | Disaster Recovery (DR) | 주 시스템 장애 시 백업 사이트에서 서비스를 복구하는 체계 | RPO/RTO 목표 정의; 시세·주문 상태 실시간 복제 |
