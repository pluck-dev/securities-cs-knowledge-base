# 53. 펀드·신탁·랩·구조화상품 심화 용어집

증권사 백엔드 개발자가 자산관리 시스템을 구축·운영할 때 반드시 숙지해야 하는 펀드·신탁·랩·구조화상품 도메인 핵심 용어를 망라하였다. 각 용어의 영문·약어, 개념 정의, 그리고 시스템 구현 시 고려해야 할 개발 포인트를 함께 제시한다.

---

## 펀드 기본

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|---|---|---|---|
| 집합투자기구 | Collective Investment Vehicle / CIV | 다수 투자자로부터 자금을 모아 운용·배분하는 법적 기구. 투자신탁·투자회사·투자합자회사 등이 있음 | 상품 마스터 테이블에 `fund_type` 구분 컬럼 필수. 세제·보수 계산 로직이 기구 유형별로 분기됨 |
| 공모펀드 | Public Offering Fund | 50인 이상 불특정 다수를 대상으로 모집하는 펀드. 자본시장법상 엄격한 공시·규제 적용 | 판매 가능 여부 체크 시 `is_public=true` 플래그 기준. 투자설명서 교부 이력 로그 필수 |
| 사모펀드 | Private Fund / PEF | 49인 이하 적격투자자 대상 모집. 공모 대비 운용·규제 자유도 높음 | 투자자 수 실시간 집계 필요(49인 초과 시 발행 중단). `investor_count` 원자적 업데이트 처리 |
| 수익증권 | Beneficiary Certificate | 투자신탁 수익권을 표창하는 유가증권. 좌수 단위로 발행됨 | 증권 번호 체계(`fund_code` + `class_code` + `series`)가 KSD 등록 기준과 일치해야 함 |
| 모자펀드 | Master-Feeder Fund | 자(子)펀드가 모(母)펀드에 투자하는 구조. 운용 효율화 목적 | 기준가 계산 시 모펀드 NAV 의존성 체인 존재. 순환 참조 방지 로직 및 계산 순서 보장 필요 |
| 재간접펀드 | Fund of Funds / FoF | 다른 펀드에 투자하는 펀드. 분산 효과와 운용사 선별 비용이 중첩됨 | 구성 펀드별 보수 이중 집계 방지. 총보수 계산 시 하위 펀드 보수까지 look-through 합산 |
| 기준가격 | NAV (Net Asset Value) | 펀드 순자산총액을 총좌수로 나눈 1좌당 가격. 설정가 예시: 1,000원 | 소수점 8자리 이상 NUMERIC 타입 저장. 계산 후 절사·반올림 정책 준수, 히스토리 테이블 보관 |
| 좌수 | Unit(s) | 펀드 지분의 최소 단위. 1좌 = 기준가 1원에서 출발, 운용 후 기준가 변동 | `DECIMAL(20,8)` 이상 정밀도. 설정/환매 시 좌수 ↔ 금액 변환 오차 관리 중요 |
| 설정 | Subscription / Creation | 투자자가 매수(청약) 후 좌수를 부여받는 과정 | T+1 또는 T+2 결제일 로직. 설정요청 → 기준가확정 → 좌수배정 상태머신 구현 |
| 환매 | Redemption | 투자자가 보유 좌수를 반환하고 현금을 수령하는 과정 | 환매제한일(환매금지기간), 환매청구 마감시간, 환매지급일을 별도 컬럼으로 관리 |
| 설정좌수 | Units Issued | 특정 시점까지 설정된 누적 총좌수 | `fund_unit_ledger` 테이블에 일별 스냅샷. 모자펀드의 경우 자펀드 합산 로직 필요 |
| 환매청구 | Redemption Request | 투자자가 환매 의사를 표시하는 행위. 청구 시점과 기준가 적용일이 다름 | 청구 테이블에 `request_dt`, `base_price_dt`, `payment_dt` 분리 저장 |
| 환매대금 | Redemption Proceeds | 환매 시 투자자에게 지급되는 금액. 환매좌수 × 기준가 - 환매수수료 | 환매 수수료율이 보유기간 구간별로 다를 수 있음. 슬라이딩 스케일 구현 |
| 환매수수료 | Redemption Fee | 조기 환매 시 부과되는 수수료. 일반적으로 90일 미만 보유 시 예시: 이익금의 70% 부과 | 수수료 구간(`fee_tier`) 테이블로 관리. 보유기간 = 환매청구일 - 설정일(좌수별 FIFO/LIFO 선택) |
| 판매보수 | Distribution Fee | 판매사에 귀속되는 연간 보수. 예시: 연 0.5~1.5% | 일할 계산(365일 기준) 후 기준가에 반영. 판매사 코드별 요율 분리 관리 |
| 운용보수 | Management Fee | 자산운용사에 귀속되는 연간 보수. 예시: 연 0.2~1.0% | 판매보수와 동일한 일할 계산. 클래스별 운용보수율 상이할 수 있음 |
| 수탁보수 | Trustee Fee | 신탁업자(수탁사)에 귀속되는 보수. 예시: 연 0.02~0.05% | 별도 보수 계산 배치 or 기준가 계산 통합 처리 여부 결정 필요 |
| 사무관리보수 | Administration Fee | 사무관리회사에 귀속되는 보수. 예시: 연 0.01~0.03% | 수탁보수와 함께 기준가 차감 항목. 수취 계좌(`admin_account_id`) 매핑 필수 |
| 총보수(TER) | Total Expense Ratio | 판매·운용·수탁·사무관리 등 모든 보수를 합산한 비율 | 공시용 TER = 연환산. 클래스별로 달라지므로 `fund_class_fee` 테이블에 구성항목 분리 저장 |
| 선취수수료 | Front-end Load | 펀드 가입 시 납입액의 일정 비율을 선취. 예시: 납입금액의 1% | 가입 시 즉시 차감. 실제 투자금액 = 납입금 × (1 - 선취율). 좌수 계산 기준 금액에 주의 |
| 후취수수료 | Back-end Load | 환매 시 환매금액에서 차감하는 수수료 | 보유기간에 따라 체감되는 구조가 많음. 환매대금 산출 시 마지막에 차감 |
| 클래스 | Class (A/C/I/E 등) | 동일 펀드 내에서 보수 체계·판매 채널이 다른 수익증권 구분 | `fund_id` + `class_code` 복합키. 클래스간 전환(Switch) 시 설정/환매 처리 순서 유의 |
| 결산 | Settlement / Accounting Period-End | 회계기간 종료 시 손익 확정 및 기준가 재산정 | 결산일 스케줄러가 기준가 계산, 분배금 산정, 이월 처리 순서로 실행되어야 함 |
| 분배금 | Distribution | 결산 시 투자자에게 지급하는 이익금. 재투자 또는 현금 지급 | 분배 유형(`REINVEST` / `CASH`)에 따라 좌수 재설정 or 출금 처리 분기 |
| 좌당기준가 | NAV per Unit | 기준가격의 다른 표현. 통상 소수점 2자리 고시 | 고시 기준가(절사)와 내부 계산 기준가(원본) 별도 저장. 공시 테이블과 계산 테이블 분리 |

## 펀드 운용·평가

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|---|---|---|---|
| 자산운용사 | Asset Management Company / AMC | 펀드를 운용하는 금융기관. 투자 의사결정 주체 | `company_master` 테이블에 AMC 구분 플래그. 운용사별 데이터 피드 연동 설정 |
| 판매사 | Distributor / Selling Agent | 투자자에게 펀드를 판매하는 증권사·은행·보험사 등 | 판매사 코드(`dist_code`)로 보수 정산 분개. 판매사별 수익증권 잔고 조회 API 제공 필요 |
| 수탁사 | Trustee / Custodian | 펀드 자산을 실질적으로 보관·관리하는 금융기관(주로 은행) | 수탁사로부터 자산 잔고·거래 내역 수신. 대사(Reconciliation) 배치 일 1회 이상 실행 |
| 사무관리회사 | Fund Administrator | 기준가 계산·회계처리·공시 업무를 수행하는 전문 기관 | 기준가 계산 결과를 사무관리사 시스템과 교차검증. FTP/API 연동 방식 정의 필요 |
| 일반사무수탁 | General Administrative Trustee | 수익증권 발행·명의개서·좌수관리 등을 담당하는 기관 | KSD(예탁결제원)와 명의개서 데이터 동기화. 좌수 대사 배치 필수 |
| 자산평가 | Asset Valuation | 펀드 보유 자산의 가치를 산정하는 과정 | 주식·채권·파생 등 자산유형별 평가 엔진 분리. 평가 실패 시 전일가 사용 정책 코드화 |
| 시가평가 | Mark-to-Market (MTM) | 시장 가격 기준으로 자산을 평가하는 방식 | 실시간 시세 피드 연동. 장 마감 후 종가 확정 시점에 평가 배치 트리거 |
| 공정가액 | Fair Value | 시장가가 없거나 유동성이 낮은 자산을 합리적 기준으로 평가한 가액 | 공정가액 산정 모델(`fair_value_model`) 및 산정 이력 별도 테이블. 감사 대응용 |
| 미수금 | Accounts Receivable (in fund) | 펀드가 받아야 할 이자·배당 등 미수령 수익 | 기준가 계산 시 자산 항목으로 포함. 수령 확정일에 현금 전환 처리 |
| 미지급금 | Accounts Payable (in fund) | 보수·거래비용 등 아직 지급하지 않은 비용 | 기준가 차감 항목. 일별 발생액 계산 후 미지급 누계 관리 |
| 좌수정정 | Unit Correction | 오류로 인한 좌수 수정 처리 | 정정 사유·승인자·정정 전후 값 감사 로그 필수. 기준가 재계산 연쇄 트리거 여부 결정 |
| 기준가오류 | NAV Error | 잘못 계산된 기준가로 인한 투자자 피해 발생 사건 | 오류 기준가 적용 기간 탐지 쿼리, 영향받은 투자자 목록 추출, 보상금 계산 자동화 필요 |
| 보유자산명세 | Portfolio Holdings | 특정 시점의 펀드 구성 자산 목록 및 수량·평가액 | `portfolio_snapshot` 테이블에 일별 저장. 자산유형별 파티셔닝 권장 |
| 포트폴리오 | Portfolio | 펀드가 보유한 자산의 집합 및 배분 구조 | 자산군·섹터·지역별 집계 뷰 제공. 리스크 지표 계산 기반 데이터 |
| 벤치마크 | Benchmark / BM | 펀드 성과를 비교하는 기준 지수. 예시: KOSPI200, MSCI World | `benchmark_index_code` 연결. 벤치마크 수익률 시계열 저장 및 상대 수익률 계산 |
| 추적오차 | Tracking Error / TE | 펀드 수익률과 벤치마크 수익률 차이의 표준편차 | 일별 수익률 차이(Δr) 시계열로 계산. 예시: 252영업일 롤링 TE = std(Δr_i) × √252 |
| 정보비율 | Information Ratio / IR | 초과수익률을 추적오차로 나눈 위험조정 성과지표 | IR = 연환산 초과수익률 ÷ TE. 성과 평가 리포트 API 응답에 포함 |
| 샤프비율 | Sharpe Ratio | (펀드 수익률 - 무위험수익률) ÷ 표준편차 | 무위험수익률(`risk_free_rate`) 마스터 테이블로 관리. 주기적 업데이트 필요 |
| 운용역 | Portfolio Manager / PM | 실제 투자 의사결정을 수행하는 운용사 담당자 | PM 변경 이력 관리. 운용역 교체 시 투자자 통보 의무 트리거 연동 |

## ETF·ETN

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|---|---|---|---|
| ETF | Exchange Traded Fund | 특정 지수를 추종하며 거래소에 상장된 펀드. 장중 매매 가능 | 일반 주식 주문 시스템과 펀드 시스템 이중 연동. 종목 코드(`isin`) + 펀드 코드 매핑 테이블 필수 |
| ETN | Exchange Traded Note | 발행사(증권사) 신용으로 지수 수익률을 보장하는 상장 채무증권 | 발행사 신용위험 관리. 만기일(`maturity_dt`) 관리 및 만기 상환 자동화 |
| 추적지수 | Underlying Index | ETF·ETN이 복제·추적하는 기준 지수. 예시: KRX300, S&P500 | 지수 제공사 데이터 피드 계약 및 라이선스 관리. 지수 변경 시 PDF 갱신 연동 |
| 실시간NAV | iNAV (Indicative NAV) | 장중 15초~1분 주기로 산출·공시되는 ETF 추정 NAV | 실시간 스트리밍 파이프라인 필요(예: Kafka). 정밀도 요구사항 확인(소수점 2자리 공시) |
| 괴리율 | Premium/Discount Rate | ETF 시장가와 iNAV의 차이 비율. (시장가 - iNAV) / iNAV × 100 | 실시간 괴리율 모니터링 대시보드. 임계값(예시: ±1%) 초과 시 알림 발송 로직 |
| 추적오차 | Tracking Error | ETF 수익률과 추적지수 수익률의 차이. ETF 품질 지표 | 일별 수익률 차이 누적 계산. 공시 주기(월·분기) 맞춰 배치 실행 |
| 유동성공급자 | LP (Liquidity Provider) | ETF 호가 공급을 담당하는 증권사. 스프레드를 일정 수준 이내로 유지 의무 | LP 계좌 구분 플래그. LP 의무 이행 모니터링(최우선 매도호가 - 최우선 매수호가 ≤ 기준) |
| 설정단위 | CU (Creation Unit) | ETF 설정·환매의 최소 단위 블록. 예시: 50,000좌 | CU 미만 소액 설정 불가 로직. CU 수량 마스터 테이블로 관리 |
| 현물설정·환매 | In-Kind Creation/Redemption | ETF 설정·환매 시 현금 대신 구성 종목 주식 바스켓을 교환하는 방식 | PDF 기반 바스켓 구성 자동화. 주식 이전·수령 처리 및 결제 연동 |
| PDF | Portfolio Deposit File | ETF 설정·환매에 필요한 구성 종목 목록과 수량 정보 | 매 영업일 장 시작 전 업데이트. 버전 관리(`pdf_version`) 및 이력 보관 |
| 분배금 | Distribution (ETF) | ETF 보유 종목에서 발생한 배당·이자 등을 투자자에게 분배하는 금액 | 분배락 기준일·지급일 스케줄 관리. 주식 분배(재투자)와 현금 분배 구분 처리 |
| 레버리지 ETF | Leveraged ETF | 추적지수의 일간 수익률 ×2 또는 ×3을 목표로 하는 ETF | 일별 리셋 구조로 장기 보유 시 복리 손실 주의. 레버리지 배율 컬럼 및 위험 고지 연동 |
| 인버스 ETF | Inverse ETF | 추적지수 수익률의 역방향(-1배) 성과를 목표로 하는 ETF | 레버리지·인버스 조합 상품(`leverage_ratio` 음수) 처리. 거래 제한 규정 적용 여부 확인 |
| 액티브 ETF | Active ETF | 지수를 수동 복제하지 않고 운용역 재량으로 운용하는 ETF | PDF 일 1회 이상 공시 의무. 포트폴리오 변경 빈도가 높아 PDF 갱신 주기 단축 필요 |
| 채권 ETF | Bond ETF | 채권 지수를 추적하는 ETF. 이자 분배 주기가 잦음 | 채권 가격 소수점 처리 주의. 월·분기 이자 분배 스케줄 자동화 |
| 합성 ETF | Synthetic ETF | 실물 자산 보유 대신 스왑 계약으로 지수 수익률을 복제하는 ETF | 스왑 상대방(Counterparty) 신용위험 노출액 관리. 담보 자산 평가 연동 |
| 순자산가치 괴리경보 | NAV Deviation Alert | 시장가가 iNAV 대비 일정 수준 이상 벗어날 때 발동하는 경보 | 임계값 설정 테이블(`alert_threshold`). 거래소 공시 연동 API 호출 자동화 |

## 구조화상품

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|---|---|---|---|
| ELS | Equity Linked Securities | 주가(지수) 연계 파생결합증권. 기초자산 성과에 따라 수익이 결정됨 | 상품 구조 파라미터(배리어·쿠폰·만기·평가일 등) JSON 또는 구조화 컬럼으로 저장 |
| DLS | Derivative Linked Securities | 주가 외 금리·환율·원자재 등을 기초자산으로 하는 파생결합증권 | ELS 엔진 재사용 + 기초자산 유형(`underlying_type`) 분기 처리 |
| ELB | Equity Linked Bond | 원금이 보장되는 주가 연계 파생결합사채 | 원금보장형 플래그(`principal_protected=true`). 채권 발행 구조 회계 처리 분리 |
| DLB | Derivative Linked Bond | 원금보장형 DLS. 기초자산이 주가 이외인 파생결합사채 | ELB와 동일한 원금보장 처리 로직 적용. 기초자산 유형만 상이 |
| ELF | Equity Linked Fund | ELS를 편입하거나 ELS 구조로 운용되는 펀드 | 펀드 + ELS 복합 구조. 기준가 계산 시 내재 ELS 평가 연동 |
| ELT | Equity Linked Trust | ELS를 신탁 형태로 운용하는 특정금전신탁 상품 | 신탁 계정과 ELS 상품 이중 매핑. 신탁 원장과 ELS 원장 대사 필요 |
| ELW | Equity Linked Warrant | 주식·지수 등을 기초자산으로 특정 가격에 매수·매도할 수 있는 권리 증서 | 주식 시스템과 연동. 행사가·만기·레버리지 지표(`gearing_ratio`) 실시간 계산 |
| 기초자산 | Underlying Asset | ELS·DLS 등의 수익 구조를 결정하는 참조 자산. 예시: KOSPI200, S&P500, 금 가격 | `underlying_master` 테이블에 코드·유형·데이터 소스 관리. 다중 기초자산 구조 지원 |
| 조기상환 | Early Redemption | 만기 전 특정 조건 충족 시 원금+이자를 지급하고 상품이 종료되는 구조 | 조기상환 평가일 스케줄러. 조건 평가 → 상환 확정 → 고객 통보 → 지급 처리 파이프라인 |
| 자동조기상환 | Auto-Call | 평가일에 기초자산 가격이 기준 이상이면 자동으로 조기상환되는 구조 | 오토콜 배리어(`autocall_barrier`) 퍼센트 저장. 평가일 장 마감 후 조건 일괄 평가 배치 |
| 낙인 | Knock-In / KI | 기초자산이 하락해 일정 수준(낙인 배리어) 이하로 터치하면 손실 구조로 전환되는 조건 | 낙인 이벤트 실시간 감지(장중 Low 가격 기준). KI 발생 시 `ki_triggered_dt` 기록 및 투자자 알림 |
| 낙아웃 | Knock-Out / KO | 기초자산이 상단 배리어를 돌파하면 상품이 조기 소멸되는 조건 (ELW에서 주로 사용) | KO 조건 장중 실시간 체크. KO 발생 즉시 상품 상태(`status=KNOCKED_OUT`) 전환 |
| 배리어 | Barrier | 낙인·낙아웃·조기상환 등의 트리거가 되는 기초자산 가격 수준(비율) | 배리어 유형(`barrier_type`: KI/KO/AUTOCALL)별 테이블 행 분리. 발행가 대비 비율 저장 |
| 쿠폰 | Coupon Rate | 조기상환 또는 만기 시 지급되는 수익률. 예시: 연 8% | 쿠폰 지급 조건과 쿠폰율 분리 저장. 조기상환 차수별 쿠폰 차등 적용 구조 지원 |
| 스텝다운 | Step-Down | 조기상환 평가 차수가 늘어날수록 상환 조건 배리어가 낮아지는 구조. 예시: 95%→90%→85% | 차수(`evaluation_seq`)별 배리어 테이블. 차수 증가 시 자동으로 다음 배리어 조건 조회 |
| 원금보장 | Principal Protected | 만기 시 최소 원금 전액을 돌려주는 구조 | `protection_ratio` 필드(예시: 100% = 전액보장). 만기 상환금 계산 시 MAX(원금×보장비율, 정산금) |
| 원금비보장 | Non-Principal Protected | 기초자산 성과에 따라 원금 손실 가능성이 있는 구조 | 위험 고지 레벨(`risk_grade`) 높게 설정. 적합성 판단 알고리즘에 원금비보장 여부 반영 |
| 상환금 지급 | Redemption Payment | 조기상환 또는 만기 시 투자자에게 지급하는 최종 금액 | 지급 스케줄 생성 → 출금 지시 → 정산 확인 3단계. 지급 실패 시 재처리 큐 관리 |
| 평가일 | Valuation Date / Observation Date | ELS 조기상환 조건 또는 만기 수익을 결정하는 기준 가격 산정일 | 영업일 기준 캘린더 연동. 공휴일·증시 휴장일 자동 조정 로직 필수 |
| 청약 | Subscription (Structured) | 구조화상품에 투자 신청하는 행위. 모집 기간 내에만 가능 | 청약 모집 상태머신(모집중→청약완료→발행→운용중). 청약 한도·잔액 실시간 관리 |
| 발행사 신용위험 | Issuer Credit Risk | 구조화상품 발행 증권사가 부도 시 원금·수익 전액 손실 가능 위험 | 발행사 신용등급(`issuer_credit_grade`) 저장 및 모니터링. 신용등급 하락 시 내부 알림 연동 |
| 만기평가 | Final Valuation | 만기일의 기초자산 가격으로 최종 수익금을 확정하는 절차 | 만기 평가 배치: 종가 수신 → 조건 평가 → 상환금 확정 → 출금 지시 자동 실행 |

| 환헤지 | Currency Hedge | 해외 자산 투자 시 환율 변동 위험을 선물환 등으로 줄이는 전략 | 헤지 비율·만기·선물환 계약 별도 테이블. 헤지 효과 측정 및 롤오버 스케줄러 |
| 환오픈 | Currency Open | 환헤지를 하지 않고 환율 변동 위험에 그대로 노출되는 상태 | 환오픈 펀드 클래스(`hedge_type=OPEN`) 구분. 환율 변동 손익을 기준가에 반영 |
| 듀레이션 | Duration | 채권 가격의 금리 민감도를 나타내는 지표. 단위: 년 | 채권 ETF·채권형 펀드 필수 지표. 포트폴리오 가중평균 듀레이션 일별 계산 |
| 크레딧 스프레드 | Credit Spread | 국채 대비 회사채 등 위험 자산의 추가 수익률(금리 차이) | 신용등급별 스프레드 기준 테이블. 스프레드 확대 시 채권 평가손 자동 반영 |
| AUM | Assets Under Management | 운용 중인 자산 총액. 펀드·랩·신탁 합산 | 일별 AUM 스냅샷. 상품·채널·운용사별 집계 리포트 API |
| 비용비율 | Expense Ratio | 펀드 운용에 소요된 총비용을 평균 순자산으로 나눈 비율 | 연환산 계산. 클래스별 분리 공시. TER과 동의어로 사용되기도 함 |
| 수익자 | Beneficiary | 펀드 수익증권 보유자. 이익 분배·환매금 수령 권리를 가짐 | 수익자 명부 관리. 분배금 지급 및 공시 수신 대상 목록 추출 기반 |
| 수익자총회 | Beneficiary Meeting | 펀드 중요 사항(합병·해지 등) 결의를 위한 수익자 모임 | 총회 소집 통보 자동화. 의결권 집계(`vote_count`) 시스템 연동 |
| 펀드합병 | Fund Merger | 복수의 펀드를 하나로 통합하는 절차 | 합병 비율 계산, 좌수 전환, 구 펀드 해지 처리 트랜잭션 원자성 보장 |
| 펀드해지 | Fund Liquidation | 펀드를 청산하고 투자자에게 잔여 자산을 분배하는 절차 | 보유 자산 전량 매도 → 비용 정산 → 잔여금 분배 순서 보장. 해지 완료 상태 전환 |
| 환매중단 | Suspension of Redemption | 시장 급락·유동성 위기 등 사유로 환매를 일시 중단하는 조치 | `redemption_suspended` 플래그. 중단 기간 동안 환매 요청 거부 및 대기열 관리 |
| 신탁원본 | Trust Principal | 신탁 계약 시 위탁자가 맡긴 최초 원금 | 원본 대비 수익률 계산 기준. 추가 납입 시 원본 누적 관리 |
| 신탁이익 | Trust Income | 신탁 운용으로 발생한 이자·배당·매매차익 등 수익 | 이익 유형별(`interest`/`dividend`/`capital_gain`) 분류. 세금 원천징수 계산 기반 |
