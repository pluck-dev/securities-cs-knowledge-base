# 94. 접근성과 포용적 디자인 심화

> 접근성은 선택 기능이 아니라 더 많은 사용자가 제품을 사용할 수 있게 하는 품질 기준입니다.

## 1. 접근성의 범위

- 시각 장애.
- 청각 장애.
- 운동 장애.
- 인지 장애.
- 일시적 장애.
- 환경적 제약.

좋은 접근성은 키보드 사용자, 모바일 사용자, 느린 네트워크 사용자에게도 도움이 됩니다.

## 2. WCAG 기본

WCAG의 네 원칙:

- Perceivable: 인식 가능.
- Operable: 조작 가능.
- Understandable: 이해 가능.
- Robust: 견고함.

색 대비, 텍스트 대체, 키보드 접근, 오류 안내, 의미 있는 구조가 핵심입니다.

## 3. 키보드와 포커스

- 모든 기능은 키보드로 접근 가능해야 한다.
- focus order가 시각 순서와 맞아야 한다.
- focus trap은 modal 안에서만 적절히 사용한다.
- focus indicator를 제거하지 않는다.
- skip link가 긴 페이지에 유용하다.

## 4. 스크린리더

- semantic HTML 우선.
- aria-label은 보이는 텍스트와 충돌하지 않게.
- live region은 신중히 사용.
- 버튼/링크 역할을 정확히.
- 로딩/오류 상태를 읽을 수 있게 제공.

## 5. 폼 접근성

- label 연결.
- 필수/선택 표시.
- 오류 메시지와 필드 연결.
- 입력 예시와 형식 안내.
- 자동완성 속성.
- 모바일 키보드 타입.

## 6. 체크리스트

- [ ] 키보드만으로 주요 흐름을 완료할 수 있다.
- [ ] 색 대비 기준을 만족한다.
- [ ] 이미지 alt 정책이 있다.
- [ ] 폼 오류가 스크린리더에 전달된다.
- [ ] 자동 접근성 테스트와 수동 테스트를 함께 한다.
- [ ] 디자인 시스템 컴포넌트에 접근성 규칙이 포함된다.
