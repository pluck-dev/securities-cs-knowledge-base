# 05. 증권사 실무 환경 (규제 / 보안 / 문화)

다른 IT 업계와 가장 다른 부분입니다. 미리 알아두면 입사 후 당황하지 않습니다.

## 규제 산업이라는 것

- **금융감독원, 한국거래소(KRX) 규정**을 따라야 합니다. 코드 한 줄도 컴플라이언스 영향을 받습니다.
- 거래 기록, 접근 로그, 변경 이력이 **감사(audit) 대상**입니다. "왜 이 값이 이렇게 바뀌었나"를 추적할 수 있어야 합니다.
- 규정이 바뀌면(예: 세법, 호가단위 개편) 코드도 따라 바뀝니다.

## 보안이 매우 빡셉니다

| 항목 | 흔한 정책 |
|------|----------|
| **망분리** | 개발망과 인터넷망이 물리적/논리적으로 분리 |
| **외부 접근 차단** | 깃허브/외부 사이트 접근 제한되는 경우 많음 |
| **반출 통제** | USB 금지, 소스코드 외부 반출 엄격 금지 |
| **계정/권한** | 운영 서버 접근은 별도 승인 절차 |

> 입사 첫날 **회사 보안 정책부터 확인**하세요. 인터넷 검색조차 제한될 수 있어, 사내 위키/문서가 1차 자료가 됩니다.

## 레거시 + 신규 공존

- 수십 년 된 자바/C 시스템 위에 코틀린 신규 서비스를 얹는 형태가 많습니다.
- "왜 이렇게 오래된 코드가 있지?" 당황 금지 — **안정적으로 돌고 있으니 함부로 안 건드리는 것**입니다.
- 신규 코틀린 서비스는 보통 이 레거시와 연동(API, 메시지 큐, DB 공유)됩니다.

## 배포가 신중합니다

- 실서비스 장애 = 고객 돈 사고이므로 배포 절차가 엄격합니다.
- 보통 개발 → 검증(스테이징) → 운영 단계를 거치고, 운영 배포는 **장 마감 후**나 주말에 합니다.
- 변경 승인, 롤백 계획, 사전 공유가 필수인 경우가 많습니다.

## 시간이 곧 부하 패턴

- **장 시간(09:00~15:30)**이 시스템 부하의 피크입니다. 특히 **9시 동시호가**에 주문 폭증.
- **장 마감 후**에는 정산/대사/리포트 같은 **배치(batch)** 작업이 크게 돕니다.
- 그래서 "낮엔 실시간, 밤엔 배치"라는 하루 리듬이 있습니다.

## 신입이 빠르게 신뢰받는 법

1. **금액 계산** — `BigDecimal`, 반올림 정책 정확히.
2. **동시성/정합성** — 잔고 안 꼬이게 트랜잭션 이해.
3. **시간(영업일) 처리** — T+2, 휴장일 계산 실수 안 하기.
4. **솔직함** — "투자자로서는 알지만 개발로는 처음"이라고 말하기. 도메인 아는 신입은 환영받습니다.
