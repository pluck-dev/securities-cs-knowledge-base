# 78. 개발자 CS 커버리지 감사표

> 이 문서는 “무엇이 아직 빠졌는가?”를 계속 점검하기 위한 자체 검증 기준입니다.

## 1. 현재 커버리지 판단

현재 58~77번 문서는 핵심 CS, 네트워크, 백엔드, 프론트엔드, 데이터베이스, 리눅스, 인프라, SRE, 보안, 아키텍처, 품질, 성능, 메시징, 디버깅, 협업을 다룹니다.

하지만 실무 개발자 커리큘럼을 더 방대하게 만들려면 다음 계층이 더 필요합니다.

- 컴파일러와 빌드 도구 내부.
- 분산 합의와 복제 알고리즘.
- 데이터베이스 저장 엔진 내부.
- 관측성 심화와 OpenTelemetry.
- 브라우저 엔진과 JavaScript 런타임 내부.
- 모바일/네이티브 클라이언트.
- AI/ML 시스템과 벡터 검색.
- 실시간 협업 시스템.
- 개인정보/규제/데이터 거버넌스.
- 제품 분석과 실험 플랫폼.
- DDD와 모듈러 아키텍처.
- 공급망 보안과 릴리즈 엔지니어링.

## 3. 깊이 기준

각 주제는 세 단계로 점검합니다.

1. **개념 이해**: 용어와 기본 동작을 설명할 수 있다.
2. **실무 적용**: 어떤 상황에서 쓰고 어떤 trade-off가 있는지 안다.
3. **장애 대응**: 문제가 생겼을 때 지표, 로그, 도구로 좁힐 수 있다.

문서가 단순 정의에 머물면 부족합니다. 각 문서에는 “주의점”, “운영 지표”, “체크리스트”가 들어가야 합니다.
