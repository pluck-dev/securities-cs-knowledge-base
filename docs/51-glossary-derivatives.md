# 51. 파생상품 심화 용어집

증권사 백엔드 개발자가 코스피200 선물·옵션 및 관련 파생상품 시스템을 구축할 때 반드시 숙지해야 할 핵심 용어를 팀별로 정리한 심화 용어집이다. 증거금 산출 엔진, 평가 모델, 리스크 관리 시스템 개발 시 즉시 참조할 수 있도록 개발 관점의 포인트를 함께 기술한다.

---

## 선물(Futures)

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 선물계약 | Futures Contract | 기초자산을 미래의 특정 시점에 사전에 합의한 가격으로 매매하기로 약정한 표준화된 계약 | 계약 마스터 테이블에 기초자산코드, 결제방식, 거래승수, 호가단위 등을 필드로 관리 |
| 기초자산 | Underlying Asset | 선물·옵션 계약의 기준이 되는 자산(예: 코스피200지수, 국채, 달러) | `underlying_code` FK로 상품 계층 구조 설계; 지수 기초자산은 실시간 지수 피드와 연계 필요 |
| 결제월 | Contract Month | 선물의 최종 결제가 이루어지는 월 (예: 3월·6월·9월·12월 쿼터리) | 종목코드 생성 규칙에 결제월 코드(H·M·U·Z 등 CME 관례 또는 거래소 자체 코드) 포함 |
| 최근월물 | Front Month / Nearby Month | 만기가 가장 가까운 선물 종목 | 롤오버 시점 자동 변경 배치 필요; 스프레드 계산의 기준 종목으로 사용 |
| 원월물 | Deferred Month / Back Month | 최근월물보다 만기가 먼 선물 종목 | 캘린더스프레드 포지션 관리 시 최근월물과 원월물 쌍(pair)으로 엔티티 구성 |
| 만기 | Expiration / Maturity | 선물계약이 종료되는 날짜 | `expiry_date` 필드; 만기 당일 장 마감 전후 처리(최종정산가 수신, 포지션 청산) 배치 설계 |
| 최종거래일 | Last Trading Day / LTD | 해당 선물을 거래할 수 있는 마지막 영업일 (코스피200선물: 결제월 두 번째 목요일) | LTD 기준으로 주문 접수 차단 로직 및 당일 만기 알림 푸시 트리거 |
| 최종결제일 | Final Settlement Day | 최종거래일 다음 영업일; 실제 차금이 결제되는 날 | 결제 배치 스케줄러에 T+1 로직; 공휴일 처리 캘린더 테이블 필수 |
| 현금결제 | Cash Settlement | 실물인수도 없이 최종결제가격과 매매가격의 차액만 현금으로 정산하는 방식 | 코스피200선물·옵션은 현금결제; 정산차금 = (최종결제가격 - 거래가격) × 거래승수 × 수량 |
| 실물인수도 | Physical Delivery | 기초자산을 실제로 주고받는 결제 방식 (주식선물, 일부 채권선물) | 인수도 가능 수량 확인, 결제대금 계산, 예탁기관 연계 API 필요 |
| 정산가격 | Daily Settlement Price | 장 종료 후 산출되는 당일 공식 가격; 일일정산의 기준 | 거래소 정산가 수신 후 전일 정산가 대비 차이로 평가손익 계산; 미수신 시 fallback 로직 필요 |
| 일일정산 | Daily Mark-to-Market / Daily Settlement | 매일 정산가격 기준으로 포지션 손익을 계산하여 증거금 계좌에서 차감·가산하는 절차 | 장 종료 배치에서 정산차금 = (당일정산가 - 전일정산가) × 거래승수 × 보유수량 자동 계산 |
| 베이시스 | Basis | 선물가격과 현물가격의 차이 (Basis = Futures Price − Spot Price) | 베이시스 모니터링 지표; 음수 전환 시 백워데이션 신호로 활용 가능 |
| 콘탱고 | Contango | 선물가격이 현물가격보다 높은 정상적 시장 상태 | 이론가 계산 시 보유비용(이자·배당) 양수 구간; 캘린더스프레드 매도 전략 분석에 활용 |
| 백워데이션 | Backwardation | 선물가격이 현물가격보다 낮은 역전 상태; 수요 급증·공급 부족 시 발생 | 롤오버 비용 역전 상황; 리스크 시스템에서 경보 발생 조건으로 설정 |
| 이론가 | Theoretical Price / Fair Value | 무차익거래 조건하에서 보유비용 모델로 산출된 선물의 적정 가격 | Fair Value = Spot × (1 + r − d)^t; r=무위험이자율, d=배당수익률, t=잔존만기(년) |
| 보유비용 | Cost of Carry | 현물을 보유하는 데 드는 비용(이자 − 배당); 선물 이론가의 핵심 변수 | 이자율·배당 마스터 데이터 관리; 국채금리 피드 연동 |
| 롤오버 | Rollover | 만기 도래 선물을 청산하고 원월물로 포지션을 이전하는 거래 | 롤오버 비용 = 원월물 매수가 − 최근월물 청산가; 자동 롤오버 배치 또는 고객 알림 설계 |
| 캘린더스프레드 | Calendar Spread | 동일 기초자산의 서로 다른 만기 선물을 동시에 매수·매도하는 전략 | 스프레드 종목 코드 별도 관리; 스프레드 전용 증거금(할인율 적용) 산출 로직 분리 |
| 미결제약정 | Open Interest / OI | 아직 청산되지 않은 선물·옵션 계약의 총 수량 | 매일 장 마감 후 OI 집계 배치; 포지션 집중도·유동성 지표로 리스크 대시보드에 표시 |
| 거래승수 | Contract Multiplier | 1계약이 나타내는 기초자산의 수량 또는 금액 (예: 코스피200선물 = 250,000원/포인트) | 손익·증거금 계산 시 반드시 곱해야 하는 상수; 상품 마스터에 `multiplier` 필드로 관리 |
| 호가단위 | Tick Size / Price Step | 선물 가격이 변할 수 있는 최소 단위 (예: 코스피200선물 0.05pt) | 주문 검증 시 `(price % tick_size == 0)` 체크; 소수점 처리 시 BigDecimal 사용 권장 |
| 코스피200선물 | KOSPI 200 Futures | 코스피200지수를 기초자산으로 하는 표준 선물계약 (승수 250,000원) | KRX 표준 코드 체계(예: 101W…); 만기 쿼터리 + 시리얼 6개월치 상장 |
| 미니코스피200선물 | Mini KOSPI 200 Futures | 코스피200선물의 1/5 크기 계약 (승수 50,000원); 개인투자자 접근성 향상 목적 | 표준 선물과 별도 종목코드; 증거금은 표준 선물의 1/5 수준으로 비례 산출 |
| 주식선물 | Individual Stock Futures / SSF | 개별 주식을 기초자산으로 하는 선물; 실물인수도 방식 | 기초 주식 배당락·액면분할 이벤트 시 행사가격·계약수량 조정(Corporate Action) 처리 로직 필요 |
| 국채선물 | Korea Treasury Bond Futures / KTB | 3년·5년·10년 국채를 기초자산으로 하는 선물; 금리 리스크 헤지 수단 | 채권가격-금리 역관계; 가격을 `100 − 금리` 형태로 표현; DV01 기반 리스크 산출 |
| 달러선물 | USD/KRW Futures | 미 달러화 대비 원화 환율을 기초자산으로 하는 통화선물 (승수 10,000달러) | 환율 피드(한국은행·외환시장) 연동; 외화 증거금 계좌와 원화 계좌 간 환산 로직 필요 |

## 옵션(Options)

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 콜옵션 | Call Option | 기초자산을 행사가격에 살 수 있는 권리 | 내재가치 = max(S − K, 0); 콜 포지션은 델타 양수 |
| 풋옵션 | Put Option | 기초자산을 행사가격에 팔 수 있는 권리 | 내재가치 = max(K − S, 0); 풋 포지션은 델타 음수 |
| 행사가격 | Strike Price / Exercise Price (K) | 옵션 보유자가 기초자산을 매수(콜) 또는 매도(풋)할 수 있는 미리 정한 가격 | 옵션 종목코드에 행사가격 인코딩; 신규 행사가 자동 상장 시 마스터 테이블 자동 생성 배치 |
| 등가격 | At-The-Money / ATM | 행사가격이 현재 기초자산 가격과 동일하거나 가장 가까운 상태 | 변동성스마일 기준점; ATM 내재변동성이 보통 가장 낮음; 빠른 조회를 위해 인덱싱 |
| 내가격 | In-The-Money / ITM | 즉시 행사 시 이익이 발생하는 상태 (콜: S>K, 풋: S<K) | 자동행사 대상 여부 판별에 사용; KRX 기준 내재가치 > 0이면 자동행사 |
| 외가격 | Out-Of-The-Money / OTM | 즉시 행사 시 손실이 발생하는 상태 (콜: S<K, 풋: S>K) | 만기 시 가치 0으로 소멸; 포지션 청산 처리 배치에서 제외 대상 |
| 프리미엄 | Option Premium | 옵션 매수자가 매도자에게 지급하는 옵션 가격 | 프리미엄 = 내재가치 + 시간가치; 체결가격을 DB에 저장 시 소수점 4자리 이상 유지 |
| 시간가치 | Time Value / Extrinsic Value | 프리미엄에서 내재가치를 뺀 부분; 만기가 가까울수록 감소 | 세타(Theta) 측정 대상; 만기 시 0으로 수렴하므로 만기별 시간가치 감소 곡선 저장 |
| 내재가치 | Intrinsic Value | 즉시 행사 시 실현 가능한 이익 (음수는 0으로 처리) | 콜: max(S−K, 0); 풋: max(K−S, 0); 실시간 계산 함수로 제공 |
| 만기 | Expiration Date | 옵션의 권리가 소멸하는 날짜 | 위클리옵션은 매주 목요일 만기; 만기 복수 관리를 위한 캘린더 테이블 필요 |
| 유럽형옵션 | European Option | 만기일에만 행사 가능한 옵션 (코스피200옵션이 해당) | 블랙숄즈 모델 직접 적용 가능; 조기행사 프리미엄 계산 불필요 |
| 미국형옵션 | American Option | 만기일 이전 언제든 행사 가능한 옵션 (주식옵션 일부) | 이항모형 또는 LSM 몬테카를로 사용; 조기행사 여부 매일 판단 로직 필요 |
| 행사/배정 | Exercise / Assignment | 옵션 매수자가 권리를 행사하면(Exercise), 매도자에게 의무가 배정(Assignment)됨 | 배정 알고리즘(랜덤 또는 비례); 배정 결과를 매도자 계좌에 즉시 반영 |
| 자동행사 | Automatic Exercise | 만기 시 내가격 옵션을 별도 행사 신청 없이 자동으로 행사하는 제도 | KRX: 내재가치 ≥ 0.01pt이면 자동행사; 만기 당일 배치에서 자동행사 대상 필터링 |
| 코스피200옵션 | KOSPI 200 Options | 코스피200지수를 기초자산으로 하는 옵션 (승수 250,000원, 유럽형, 현금결제) | 행사가격 간격 2.5pt(ATM 근방) / 5pt(원방); 실시간 종목 수 수백 개 관리 |
| 위클리옵션 | Weekly Option | 매주 목요일 만기가 도래하는 단기 옵션 | 매주 신규 행사가 자동 상장 배치; 짧은 만기로 세타 감소 속도 빠름 |
| 미니옵션 | Mini Option | 표준 옵션의 1/5 크기 계약 (승수 50,000원) | 표준·미니 옵션 간 델타헤지 수량 비율 5:1 |
| 내재변동성 | Implied Volatility / IV | 시장에서 거래되는 옵션 프리미엄으로부터 역산된 변동성 | 뉴턴-랩슨 또는 Brent 방법으로 반복 계산; 수렴 실패 케이스 예외 처리 필수 |
| 역사적변동성 | Historical Volatility / HV | 과거 기초자산 수익률의 표준편차로 측정한 변동성 (예: 20일, 60일) | `STDDEV(log(S_t/S_{t-1})) × sqrt(252)` 연율화; 롤링 윈도우 계산 배치 |
| 변동성스마일 | Volatility Smile | 동일 만기 내에서 행사가격별로 내재변동성이 U자 또는 스큐 형태를 띠는 현상 | 변동성 서피스 보간(interpolation) 로직; 행사가·만기 2차원 그리드로 저장 |
| 변동성스큐 | Volatility Skew | 변동성스마일이 한쪽으로 기울어진 형태; 주식 옵션에서는 풋 쪽 IV가 더 높은 경향 | 25델타 리스크 리버설(RR)·버터플라이(BF)로 스큐 파라미터 관리 |

## 그릭스(Greeks)·평가

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 델타 | Delta (Δ) | 기초자산 가격이 1단위 변할 때 옵션 가격의 변화량; 콜 0~1, 풋 −1~0 | 포지션 델타 = Δ × 계약수 × 거래승수; 실시간 합산으로 방향성 리스크 모니터링 |
| 감마 | Gamma (Γ) | 기초자산 가격이 1단위 변할 때 델타의 변화량; 볼록성 측정 | 감마 × (ΔS)² / 2 = 이차 손익; ATM 근방·만기 근접 시 급증하므로 리스크 한도 별도 설정 |
| 세타 | Theta (Θ) | 시간이 1일 경과할 때 옵션 가격의 변화(시간가치 감소); 통상 음수 | 일일 세타 = (옵션가격 변화) / (1/252); 옵션 매수 포지션 보유비용으로 반영 |
| 베가 | Vega (ν) | 내재변동성이 1%p 변할 때 옵션 가격의 변화량 | 변동성 리스크(Vega Risk) 한도; 단위 주의: IV 1% 변화 기준 또는 1%p 기준 혼용 주의 |
| 로(rho) | Rho (ρ) | 무위험이자율이 1%p 변할 때 옵션 가격의 변화량 | 단기 옵션에선 미미하나 장기 옵션(LEAPS)에서 유의미; 금리 변동 시나리오 스트레스테스트에 포함 |
| 델타헤지 | Delta Hedge | 포지션의 델타를 0으로 만들어 기초자산 방향성 리스크를 제거하는 전략 | 헤지 수량 = −(포지션 델타) / (선물 델타≒1); 재조정(리밸런싱) 주기 설정 필요 |
| 감마스캘핑 | Gamma Scalping | 감마가 큰 옵션 매수 포지션에서 기초자산 가격 변동 시 지속적으로 델타헤지하여 수익을 얻는 전략 | 헤지 체결 빈도·비용 최적화 알고리즘; P&L 귀속(attribution) 로직 설계 필요 |
| 블랙숄즈모형 | Black-Scholes Model / BSM | 유럽형 옵션의 이론가를 산출하는 확률론적 가격결정 모형 | `d1=(ln(S/K)+(r+σ²/2)t)/(σ√t)`, `C=S·N(d1)−K·e^{−rt}·N(d2)`; 소수점 정밀도 double 이상 권장 |
| 이항모형 | Binomial Tree Model | 기초자산 가격이 매 기간 상승(u) 또는 하락(d)한다고 가정하는 이산 모형; 미국형 옵션 평가 가능 | 트리 노드 수 = (N+1)(N+2)/2; N=100 이상 시 메모리·속도 최적화 필요(배열 재활용) |
| 몬테카를로시뮬레이션 | Monte Carlo Simulation | 난수를 이용해 기초자산 경로를 수만 번 시뮬레이션하여 옵션 가격을 추정하는 방법 | 경로 의존형(아시안·배리어) 옵션에 적합; 분산감소기법(안티세테틱, 제어변량) 적용으로 수렴 속도 향상 |
| 변동성서피스 | Volatility Surface | 행사가격(또는 델타)과 만기를 두 축으로 하는 3차원 내재변동성 곡면 | SVI(Stochastic Volatility Inspired) 또는 삼차 스플라인 보간; 아비트라지 프리 조건 검증 필수 |
| 민감도분석 | Sensitivity Analysis / Scenario Analysis | 시장 변수(가격·변동성·금리)를 변화시켜 포트폴리오 손익에 미치는 영향을 계산하는 분석 | 가격 ±1%, 변동성 ±1%p, 금리 ±25bp 등 표준 시나리오 그리드 사전 정의 |
| 옵션가격모형 | Option Pricing Model | 옵션 이론가를 산출하기 위한 수리·통계 모형의 총칭 | 모형별 파라미터 관리(σ, r, q); 모형 선택 전략을 상품 마스터에 매핑 |
| 위험중립가치평가 | Risk-Neutral Valuation | 모든 자산이 무위험이자율로 성장한다고 가정하는 프라이싱 프레임워크 | BSM·이항모형·몬테카를로 모두 이 프레임워크 위에서 구현; 실세계 확률과 혼동 금지 |
| 파이 | Phi (φ) / Charm | 시간과 기초자산 가격 변화에 따른 델타의 2차 편미분; 일부 시스템에서 추적 | 고빈도 델타 재조정 시스템에서 내일의 델타 예측에 활용 |
| 볼가 | Volga / Vomma | 변동성 변화에 대한 베가의 2차 편미분; 변동성 볼록성 측정 | 변동성 리스크 정밀 관리 시스템에서 2차 베가 리스크로 추적 |
| 바나 | Vanna | 기초자산 가격 변화에 대한 베가의 편미분(=변동성 변화에 대한 델타의 편미분) | 크로스 그릭; 델타헤지 + 변동성 헤지를 동시에 수행하는 시스템에서 교차항으로 관리 |

## 증거금 산출

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 개시증거금 | Initial Margin / IM | 파생상품 포지션을 개설할 때 납부해야 하는 최소 증거금 | 거래소 고시 증거금율 × 계약가치; 산출 후 잔고 부족 시 주문 거부 반환 코드 설계 |
| 유지증거금 | Maintenance Margin / MM | 포지션 유지를 위해 계속 보유해야 하는 최소 증거금 수준 (개시증거금보다 낮음) | 일반적으로 개시증거금의 70~80%; 잔고 < 유지증거금 시 마진콜 발생 트리거 |
| 추가증거금 | Margin Call / Variation Margin | 잔고가 유지증거금 미만으로 떨어질 때 추가로 납부를 요구받는 증거금 | 마진콜 발생 시 SMS/이메일/앱 푸시 발송 및 입금 기한(예: 다음 영업일 11시) 관리 |
| 위탁증거금 | Customer Margin / Client Margin | 고객이 증권사에 예치하는 증거금; 거래소 증거금과 구분 | 증권사가 거래소에 예치하는 증거금(위탁분)과 고객 계좌 증거금을 별도 관리 |
| 순위험증거금 | Net Risk Margin | 포지션 간 상계 효과를 반영한 순 위험 기반 증거금 | 롱·숏 상계, 헤지 인정 여부를 포함한 포트폴리오 수준 계산; SPAN 엔진 활용 |
| SPAN증거금 | Standard Portfolio Analysis of Risk / SPAN | CME가 개발한 포트폴리오 기반 증거금 산출 방식; 16개 리스크 시나리오 최대 손실 기반 | KRX의 COMS(코스피200 계열) 기반; 시나리오 배열 16개(±가격 변화 × ±변동성 변화) 병렬 계산 |
| COMS | COMS (KRX 자체 증거금 시스템) | KRX가 SPAN 방식을 변형·적용한 코스피200 파생상품 증거금 산출 시스템 | KRX COMS 파일(.cms)을 수신·파싱하여 위험계수 테이블 업데이트; 일일 1회 이상 갱신 |
| 포트폴리오증거금 | Portfolio Margin | 개별 포지션이 아닌 전체 포트폴리오의 순위험을 기준으로 산출하는 증거금 | 전략 상계(스프레드·헤지) 인정으로 증거금 절감; 복잡도 증가 → 별도 계산 엔진 필요 |
| 위험계수 | Risk Factor / Risk Array | 각 시나리오에서의 손익 변화 배열; SPAN에서 계약 1단위당 손익 (예시: 시나리오 8번 = −125,000원) | KRX 배포 파일에서 파싱; 16개 시나리오 × 계약수 × 위험계수 행렬 연산 |
| 사전증거금 | Pre-Trade Margin Check | 주문 체결 전에 증거금 충족 여부를 실시간으로 확인하는 절차 | 주문 수신 → 증거금 계산 → 부족 시 거부; 레이턴시 목표 1ms 이하 (캐시 적극 활용) |
| 장중증거금 | Intraday Margin | 장 중 시장 변동성 확대 시 거래소가 임시로 부과하는 추가 증거금 | KRX 긴급 고시 수신 후 즉시 적용; 고객 통보 및 미납 시 강제청산 로직 연동 |
| 증거금률 | Margin Rate / Margin Ratio | 계약가치 대비 증거금 비율 (예: 코스피200선물 개시증거금률 약 7.5%) | `margin_rate` 마스터 테이블; 거래소 고시 변경 시 자동 반영 배치 + 변경 이력 관리 |
| 대용증거금 | Securities-in-Lieu / Substitute Margin | 현금 대신 유가증권(주식·채권·ETF 등)으로 납부하는 증거금 | 대용가격(시세의 70~80%) 적용; 종목별 대용비율 마스터 관리; 담보 관리 시스템 연동 |
| 일일정산차금 | Daily Settlement Variation | 일일 정산가 변동으로 발생하는 손익 차금; 매일 계좌에 반영 | 차금 = (당일정산가 − 전일정산가) × 승수 × 수량; 양수=이익가산, 음수=손실차감 |
| 결제차금 | Final Settlement Variation | 최종결제일에 최종결제가격 기준으로 발생하는 마지막 정산 차금 | 최종결제가 = 당일 코스피200 최종 산출값; 포지션 전량 청산 처리와 함께 배치 실행 |
| 증거금 부족 | Margin Deficiency | 예치 증거금이 요구 증거금에 미달하는 상태 | 부족액 = max(0, 요구증거금 − 예치증거금); 실시간 모니터링 후 알림·강제청산 임계값 설정 |
| 강제청산 | Forced Liquidation / Liquidation Call | 증거금 부족 해소 불가 시 증권사가 고객 포지션을 강제로 청산하는 절차 | 청산 우선순위(OTM→ITM, 손실 큰 순 등) 정책 정의; 청산 체결가 기록·통보 의무 |
| 현금증거금 | Cash Margin | 현금으로만 납부하는 증거금; 대용증거금과 구분 | 파생결제 계좌(현금전용) 분리 관리; 출금 제한 로직(증거금 묶음) 구현 |

## 전략·기타 파생

| 용어 | 영문/약어 | 뜻 | 개발 포인트 |
|------|-----------|-----|-------------|
| 불스프레드 | Bull Spread | 기초자산 가격 상승을 예상할 때 낮은 행사가 콜 매수 + 높은 행사가 콜 매도로 구성하는 전략 | 동일 만기 2개 행사가 콜 페어 관리; 최대 이익·최대 손실 사전 계산 후 고객 표시 |
| 베어스프레드 | Bear Spread | 기초자산 가격 하락을 예상할 때 높은 행사가 풋 매수 + 낮은 행사가 풋 매도로 구성하는 전략 | 풋스프레드 증거금 = max(0, 행사가 차이 − 수취프리미엄) × 승수 |
| 스트래들 | Straddle | 동일 행사가격의 콜·풋을 동시에 매수(롱스트래들) 또는 매도(숏스트래들)하는 변동성 전략 | 손익분기점 = 행사가 ± 프리미엄 합계; 변동성 거래 시스템에서 IV 기준 진입 신호 생성 |
| 스트랭글 | Strangle | 콜과 풋을 서로 다른 행사가(OTM)로 동시에 매수·매도하는 전략; 스트래들보다 비용 낮음 | 2개 행사가의 OTM 조건 검증; 손익 계산 시 양 방향 손익분기점 별도 산출 |
| 버터플라이 | Butterfly Spread | 3개 행사가를 이용한 전략; 중간 행사가 옵션 2계약 매도 + 양쪽 행사가 각 1계약 매수 | 3-레그 주문 동시 체결 로직; 버터플라이 전용 스프레드 증거금 산출 |
| 콘도르 | Condor | 4개 행사가를 이용한 전략; 버터플라이보다 이익 구간이 넓고 최대 이익은 낮음 | 4-레그 포지션 관리; 구성 옵션 중 일부 만기 전 청산 시 잔여 레그 리스크 재계산 |
| 커버드콜 | Covered Call | 기초자산 현물을 보유하면서 콜옵션을 매도하는 전략; 프리미엄 수입으로 보유 비용 절감 | 현물 포지션과 옵션 포지션을 연계; 현물 없이 콜 매도 시 나체매도(Naked Call)로 전환 경보 |
| 프로텍티브풋 | Protective Put | 기초자산 현물을 보유하면서 풋옵션을 매수하는 전략; 하방 리스크 보험 역할 | 풋 프리미엄을 보험료로 인식; 현물 평가손 + 풋 평가익 통합 손익 표시 |
| 차익거래 | Arbitrage | 시장 간 또는 이론가 대비 가격 불일치를 이용해 무위험 이익을 추구하는 거래 | 선물 이론가 vs 실제가 비교; 괴리율 임계값 초과 시 자동 주문 생성 알고리즘 필요 |
| 프로그램매매 | Program Trading | 일정 조건(선물-현물 괴리율 등)이 충족되면 시스템이 자동으로 바스켓 주문을 실행하는 매매 | 코스피200 구성 종목 바스켓 매매; 괴리율 산출 + 거래 비용 차감 후 순이익 양수 시 트리거 |
| 헤지거래 | Hedge Trade | 보유 포지션의 리스크를 상쇄하기 위해 반대 방향 포지션을 취하는 거래 | 헤지 목적 포지션에 `hedge_flag` 부여; 증거금 상계 인정 여부 판별에 활용 |
| 투기거래 | Speculative Trade | 가격 변동에서 이익을 얻으려는 방향성 거래; 시장 유동성 공급 역할 | 투기·헤지 구분은 신고 기준; 시스템상 포지션 한도(Position Limit) 별도 적용 가능 |
| ELW | Equity Linked Warrant / ELW | 특정 주식·지수를 기초자산으로 하는 주식워런트증권; 증권사가 발행하는 장내 파생상품 | 발행사(LP) 호가 제공 의무; 내재변동성·델타 실시간 산출 및 투자자 공시 필요 |
| 양매도 | Short Straddle / Short Strangle | 콜과 풋을 동시에 매도하는 전략; 변동성 감소 기대 시 사용 | 매도 포지션으로 무한 손실 가능; 증거금 요구 최대; 리스크 한도 초과 모니터링 중요 |
| 양매수 | Long Straddle / Long Strangle | 콜과 풋을 동시에 매수하는 전략; 큰 방향성 변동 기대 시 사용 | 최대 손실 = 지불한 프리미엄 합계; 손익분기점 양방향 산출 후 고객 UI 표시 |
| 합성포지션 | Synthetic Position | 옵션 조합으로 선물·현물과 동일한 손익 구조를 만드는 포지션 | 합성 롱: 콜 매수 + 풋 매도(동일 행사가); 풋-콜 패리티 arbitrage 모니터링 대상 |
| 델타중립 | Delta Neutral | 포트폴리오의 총 델타가 0이 되도록 포지션을 구성하여 방향성 리스크를 제거하는 상태 | 총 델타 = Σ(개별 델타 × 계약수 × 승수); 목표 범위(예: ±50) 설정 후 이탈 시 알림 |
| 풋-콜 패리티 | Put-Call Parity | 동일 행사가·만기의 콜·풋 프리미엄과 현물가·할인인수 간의 항등 관계: C − P = S − K·e^{−rt} | 패리티 위반 시 차익거래 기회; 실시간 패리티 모니터링 및 자동 알림 시스템 구현 |
| 변동성차익거래 | Volatility Arbitrage | 옵션 내재변동성과 실현변동성(또는 모델 변동성) 간의 차이를 이용한 차익거래 | IV vs RV 비교 대시보드; 델타헤지 비용 고려 후 순이익 산출 |
| 위험조정수익 | Risk-Adjusted Return | 취한 리스크 대비 수익률 (예: 샤프비율, 소르티노비율) | 전략 성과 분석 모듈; Sharpe = (R_p − R_f) / σ_p; 일별 수익률 시계열 저장 필요 |
